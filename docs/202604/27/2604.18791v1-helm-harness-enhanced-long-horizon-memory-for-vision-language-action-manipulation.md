## 摘要
尽管视觉-语言-动作（VLA）模型在短时程任务中表现强劲，但在长时程操纵任务中却存在系统性失败。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失败；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟、验证鸿沟和恢复鸿沟。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个情景记忆模块（EMM），通过 CLIP 索引的关键帧检索关键任务历史；一个学习型状态验证器（SV），根据观测、动作、子目标和受记忆约束的上下文在执行前预测动作失败；以及一个 Harness 控制器（HC），执行回滚和重新规划。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 上，HELM 将任务成功率比 OpenVLA 提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口扩展到 H=32 仅带来 5.4 个百分点的提升，且相同预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了 CALVIN 上的长时程性能，并显著增强了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们发布了 LIBERO-Recovery，作为一种用于评估长时程操纵中失败恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与整体含义（研究动机和背景）
视觉-语言-动作（VLA）模型（如 OpenVLA, Octo）在短时程任务中表现优异，但在长时程（Long-horizon）操纵任务中会发生系统性失效。作者通过实证研究发现，这种失效并非单纯因为上下文窗口不够长（将窗口从 8 扩展到 64 仅能带来微小提升），而是源于当前“反应式执行”模式下的三个结构性缺陷：
*   **记忆鸿沟 (Memory Gap, $F_M$)**：固定窗口丢弃了已完成子目标的证据，导致机器人重复已完成的动作。
*   **验证鸿沟 (Verification Gap, $F_V$)**：动作在执行前缺乏可行性检查，导致错误动作静默执行并传播误差。
*   **恢复鸿沟 (Recovery Gap, $F_R$)**：一旦发生错误，模型会在损坏的状态下继续尝试后续步骤，引发连锁反应。

### 2. 方法论：HELM 框架
HELM 是一个模型无关（Model-agnostic）的增强框架，通过三个核心组件闭合上述鸿沟：
*   **情节记忆模块 (EMM)**：维护一个基于 CLIP 索引的关键帧存储库。它在子目标完成、检测到失败或固定步长时写入关键帧。检索时，通过当前观测与历史关键帧的余弦相似度提取最相关的上下文，并将其序列化为文本注入 VLA 的输入。
*   **状态验证器 (SV) - 核心贡献**：这是一个轻量级的 MLP 模块，用于在执行前预测动作失败概率 $p_{fail}$。其关键创新在于**受记忆约束的输入**：它同时接收当前观测、拟执行动作、当前子目标以及从 EMM 检索到的历史上下文。
*   **Harness 控制器 (HC)**：负责协调执行循环。如果 SV 预测失败概率超过阈值，HC 会触发恢复机制。
    *   **回滚 (Rollback)**：利用 EMM 中的检查点恢复到之前的正确状态。
    *   **前向恢复 (Forward Recovery)**：在无法回滚的环境中，根据当前损坏状态重新规划。

### 3. 实验设计
*   **数据集与场景**：
    *   **LIBERO-LONG**：包含 10 个任务，平均 5.8 个子目标。
    *   **CALVIN (ABC → D)**：评估在未见环境中的长程链式任务完成能力。
    *   **LIBERO-Recovery (新提出)**：专门用于评估失败恢复能力的协议，通过在子目标边界随机注入扰动（如物体位移、夹具状态翻转）来测试。
*   **对比方法 (9 个基准)**：包括原始 OpenVLA/Octo、长上下文版本 (H=32/64)、Oracle 记忆、基于规则的验证器、集成不确定性 (Ensemble)、相同数据预算下的 LoRA 微调、以及 Reflexion（自我反思）等。

### 4. 资源与算力
*   **算力设备**：所有实验均在单张 **NVIDIA A100 (80GB)** GPU 上运行。
*   **训练时长**：状态验证器 (SV) 的训练需要约 5 万个 rollout 步骤的数据，训练时间约为 **2 小时**。
*   **推理开销**：HELM 引入的额外推理延迟约为 **12ms/step**，仅比原始 VLA 骨干网络增加约 15% 的开销。

### 5. 实验数量与充分性
*   **实验规模**：LIBERO-LONG 评估了 500 个回合，CALVIN 评估了 1000 个链条，每个实验运行 3 个随机种子并报告标准差。
*   **充分性**：实验设计非常全面。作者不仅对比了性能，还通过消融实验验证了 EMM 和 SV 的耦合效应（证明没有记忆，SV 的效果会大幅下降）。
*   **客观性**：通过引入“相同数据预算的 LoRA”基准，证明了 HELM 的提升并非仅仅来自额外的训练数据，而是来自架构设计的优越性。

### 6. 主要结论与发现
*   **结构性提升**：在 LIBERO-LONG 上，HELM 将 OpenVLA 的成功率从 58.4% 提升至 **81.5%**（+23.1 pp）。
*   **长上下文非灵丹妙药**：单纯增加上下文长度（H=32）仅提升 5.4 pp，证明了引入外部显式记忆和验证机制的必要性。
*   **SV 的优越性**：学习到的 SV 优于基于规则的检查（+8.4 pp vs +6.8 pp），且比集成方法（Ensemble）更高效。
*   **故障覆盖**：HELM 将记忆失效减少了 76%，验证失效减少了 61%，恢复失效减少了 82%。

### 7. 优点与亮点
*   **模型无关性**：可以无缝套用在 OpenVLA、Octo 等不同架构上，具有很强的通用性。
*   **执行前验证**：不同于事后反思（Post-hoc reflection），SV 在动作发生前拦截错误，极大地保护了任务状态不被破坏。
*   **低成本高收益**：相比于全量微调大模型，训练一个轻量级 SV 的成本极低，但对长程任务的鲁棒性提升巨大。

### 8. 不足与局限
*   **数据依赖性**：SV 的训练仍需要针对特定环境收集约 5 万步的 rollout 数据，限制了其在完全陌生环境下的零样本（Zero-shot）部署能力。
*   **回滚假设**：在真实机器人场景中，某些物理动作是不可逆的（如打破杯子），此时回滚机制失效，需依赖效果稍弱的前向恢复。
*   **仿真与现实差距**：实验主要在仿真环境（LIBERO, CALVIN）中完成，真实世界的传感器噪声和视觉偏移可能会影响 CLIP 检索和 SV 的准确性。
*   **子目标分解依赖**：虽然占比很小（4.3%），但 VLA 生成的子目标分解质量仍是性能的一个潜在瓶颈。
