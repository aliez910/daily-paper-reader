## 摘要
视觉-语言-动作（VLA）模型有望实现通用的机器人操纵，但通常作为短时界策略进行训练和部署，这些策略假设最新的观测结果足以进行动作推理。这一假设在非马尔可夫长时界任务中失效，因为在这些任务中，与任务相关的证据可能会被遮挡或仅出现在轨迹的早期阶段，且杂乱的环境和干扰物会使细粒度的视觉定位变得脆弱。我们提出了 CodeGraphVLP，这是一个分层框架，通过将持久的语义图状态与可执行的基于代码的规划器以及进度引导的视觉语言提示相结合，实现了可靠的长时界操纵。语义图在部分可观测性下维护与任务相关的实体和关系。合成的规划器在该语义图上执行，以进行高效的进度检查，并输出子任务指令以及与子任务相关的物体。我们利用这些输出构建抑制杂波的观测结果，使 VLA 执行器专注于关键证据。在真实世界的非马尔可夫任务中，CodeGraphVLP 相比于强大的 VLA 基线和支持历史信息的变体，提高了任务完成率，同时与 VLM 在环规划相比，显著降低了规划延迟。我们还进行了广泛的消融研究，以确认各组件的贡献。

## Abstract
Vision-Language-Action (VLA) models promise generalist robot manipulation, but are typically trained and deployed as short-horizon policies that assume the latest observation is sufficient for action reasoning. This assumption breaks in non-Markovian long-horizon tasks, where task-relevant evidence can be occluded or appear only earlier in the trajectory, and where clutter and distractors make fine-grained visual grounding brittle. We present CodeGraphVLP, a hierarchical framework that enables reliable long-horizon manipulation by combining a persistent semantic-graph state with an executable code-based planner and progress-guided visual-language prompting. The semantic-graph maintains task-relevant entities and relations under partial observability. The synthesized planner executes over this semantic-graph to perform efficient progress checks and outputs a subtask instruction together with subtask-relevant objects. We use these outputs to construct clutter-suppressed observations that focus the VLA executor on critical evidence. On real-world non-Markovian tasks, CodeGraphVLP improves task completion over strong VLA baselines and history-enabled variants while substantially lowering planning latency compared to VLM-in-the-loop planning. We also conduct extensive ablation studies to confirm the contributions of each component.

## 1. 核心问题与研究动机
*   **核心问题**：现有的视觉-语言-动作（VLA）模型通常遵循**马尔可夫假设**，即仅根据当前观测进行决策。但在长程（Long-horizon）机器人任务中，这一假设会失效，原因包括：
    *   **非马尔可夫依赖**：关键信息可能被遮挡，或仅出现在过去的观测中。
    *   **视觉干扰**：环境中的杂乱物体（Clutter）容易导致 VLA 模型的视觉定位（Grounding）出错。
    *   **规划延迟**：传统的 VLM 在线规划（VLM-in-the-loop）推理速度慢，难以满足闭环控制的实时性要求。
*   **研究背景**：论文旨在结合结构化状态表示与程序化规划，提升 VLA 模型在复杂、非马尔可夫环境下的鲁棒性与效率。

## 2. 方法论
CodeGraphVLP 采用分层架构，核心思想是将环境状态显式化，并将规划逻辑代码化。
*   **持久化语义图状态 (Persistent Semantic-Graph State)**：
    *   **构建**：利用实例分割模型（YOLOE）和 VLM 识别任务相关物体，通过多视图关联和几何规则建立包含实体属性及关系（如 `in`, `on`, `near`）的图结构 $G_t$。
    *   **更新**：使用视频目标跟踪器（Cutie）在线更新物体掩码，并定期通过分割模型修正漂移或发现新物体。
*   **代码即规划器 (Code-as-Planner)**：
    *   **合成**：在任务开始前，利用 LLM（如 GPT-5）根据任务指令和初始图状态一次性生成 Python 规划代码。
    *   **执行**：代码通过查询语义图来检查任务进度（利用 `graph.task_memory` 记录已完成步骤），输出当前子任务指令及相关的物体集合 $O_{rel}$。
*   **进度引导的去干扰提示 (Clutter-Free Prompting)**：
    *   **视觉提示**：根据 $O_{rel}$ 对原始图像进行掩码处理，仅保留相关物体，黑化无关区域，减少视觉干扰。
    *   **语言提示**：将长程指令简化为具体的子任务指令（如“拿起黑色杯子”），降低 VLA 执行器的推理难度。

## 3. 实验设计
*   **实验场景**：三个真实世界的桌面操纵任务，均具有非马尔可夫特性：

    2.  **Place-and-Stack**：将方块放入杯中（方块被遮挡）后，叠放另一个杯子。
    3.  **Swap Cups**：在三个盘子间交换两个杯子的位置（涉及中间缓冲区和顺序约束）。

## 6. 主要结论与发现
*   **性能领先**：CodeGraphVLP 在所有非马尔可夫任务中均显著优于基线，平均成功率达到 **81.7%**，而最强基线仅为 56.7%。

## 7. 优点
*   **状态显式化**：通过语义图解决了长程任务中的遮挡和记忆丢失问题。
*   **计算高效**：一次性代码合成避免了昂贵的实时大模型调用。
*   **抗干扰能力强**：创新的去干扰视觉提示有效解决了 VLA 模型在复杂背景下的定位难题。

## 8. 不足与局限
*   **基础模型依赖**：语义图的准确性和代码生成的逻辑严密性高度依赖于底层 VLM/LLM 的性能。
*   **场景局限性**：目前仅在桌面操纵任务中验证，对于更广阔的开放世界或高度动态的环境，语义图的维护成本和代码的复杂性可能会增加。
*   **代码验证缺失**：论文提到未来需要加入自动验证机制，以防止生成的代码出现逻辑错误。
