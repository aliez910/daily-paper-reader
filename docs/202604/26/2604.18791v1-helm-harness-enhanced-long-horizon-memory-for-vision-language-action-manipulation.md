## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操作任务中却存在系统性失效。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失效问题；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟（memory gap）、验证鸿沟（verification gap）和恢复鸿沟（recovery gap）。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个情景记忆模块（EMM），通过 CLIP 索引的关键帧检索关键任务历史；一个学习型状态验证器（SV），根据观察、动作、子目标和记忆调节的上下文，在执行前预测动作失败；以及一个控制框架控制器（HC），负责执行回滚和重新规划。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 数据集上，HELM 将任务成功率比 OpenVLA 提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口延长至 H=32 仅带来 5.4 个百分点的提升，且在相同预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了在 CALVIN 上的长程表现，并显著增强了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们发布了 LIBERO-Recovery，作为一种用于评估长程操作中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与整体含义（研究动机和背景）
视觉-语言-动作（VLA）模型（如 OpenVLA, Octo）在短程任务中表现出色，但在长程操作任务中成功率骤降。作者指出，这种失效并非单纯因为上下文窗口不够长，而是源于三个结构性缺陷：
*   **记忆鸿沟（Memory Gap, $F_M$）：** 固定窗口丢弃了已完成子目标的证据，导致机器人重复已完成的动作。
*   **验证鸿沟（Verification Gap, $F_V$）：** 动作是反应式生成的，缺乏执行前的可行性检查，导致错误动作静默执行并累积。
*   **恢复鸿沟（Recovery Gap, $F_R$）：** 一旦发生错误，模型会在损坏的状态下继续尝试后续步骤，引发连锁反应。

### 2. 方法论：核心思想与技术细节
HELM 提出了一个模型无关的框架，通过三个专用组件增强现有的 VLA 模型：
*   **情节记忆模块 (EMM)：** 维护一个由 CLIP 索引的关键帧存储库。它在子目标完成、检测到失败或每隔 20 步时写入快照。检索时，通过 CLIP 嵌入的余弦相似度提取最相关的历史上下文，并将其序列化为文本注入 VLA 的语言输入。
*   **状态验证器 (SV) —— 核心贡献：** 一个轻量级的 3 层 MLP。它在执行前接收（当前观察、拟执行动作、当前子目标、记忆上下文）作为输入，预测动作失败的概率 $p_{fail}$。关键创新在于 SV 是**记忆调节**的，即判断当前动作是否合理需参考历史记忆。
*   **控制框架控制器 (HC)：** 负责协调循环。如果 SV 预测失败，HC 会触发**回滚（Rollback）**（返回到记忆中的上一个检查点）或**前向恢复（Forward Recovery）**（从当前损坏状态重新规划）。

### 3. 实验设计
*   **数据集/场景：**
    *   **LIBERO-LONG：** 包含 10 个任务，平均 5.8 个子目标。
    *   **CALVIN (ABC → D)：** 评估在未见环境中的长程链式操作。
    *   **LIBERO-Recovery (新提出)：** 专门用于评估故障恢复的协议，在子目标边界注入随机扰动（如物体位移或夹爪状态翻转）。
*   **对比方法（共 9 种基准）：** 包括原始 OpenVLA（不同上下文长度 H=8~64）、Oracle 记忆（真值注入）、基于规则的验证器、集成不确定性（Ensemble）、同等数据预算下的 LoRA 微调、以及 Reflexion（自我反思）等。

### 4. 资源与算力
*   **硬件：** 使用了单张 **NVIDIA A100 80GB GPU**。
*   **训练时长：** 状态验证器（SV）的训练需要约 5 万步的 rollout 数据，在 A100 上训练耗时约 **2 小时**。
*   **推理开销：** 每步增加约 **12 毫秒** 的延迟，比单纯运行 VLA 主干网络增加约 15% 的计算开销。

### 5. 实验数量与充分性
实验设计非常充分且具有说服力：
*   **多维度对比：** 不仅对比了 SOTA 模型，还通过消融实验拆解了 EMM、SV 和回滚机制的各自贡献。
*   **机制分析：** 深入探讨了检索方法（CLIP vs 随机 vs 学习型）、失败标签的时间跨度、记忆容量 $k$ 的影响。
*   **客观性：** 引入了人工标注研究（Cohen’s $\kappa=0.81$）来验证失败分类的准确性，并使用了 3 个随机种子报告标准差。

### 6. 主要结论与发现
*   **长上下文并非万能：** 将 OpenVLA 的上下文从 8 增加到 64，成功率仅提升 6.7%，而 HELM 提升了 **23.1%**（58.4% → 81.5%）。
*   **SV 的优越性：** 学习到的 SV 优于基于规则的检查，且在性能上接近 5 倍推理成本的集成模型（Ensemble），但效率更高。
*   **记忆的重要性：** 移除记忆上下文后，SV 的性能显著下降，证明了“预判失败”必须依赖历史信息。
*   **模型无关性：** HELM 在 Octo 模型上同样实现了 21.6% 的大幅提升，证明了其普适性。

### 7. 优点与亮点
*   **结构化解决长程问题：** 针对性地定义并解决了 VLA 执行循环中的三个具体鸿沟。
*   **预执行验证：** 不同于事后反思（Post-hoc reflection），SV 在错误发生前拦截，极大地保护了任务状态。
*   **高效性：** 采用轻量级 MLP 配合冻结的 VLA 主干，避免了昂贵的大模型全量微调。
*   **新基准：** 贡献了 LIBERO-Recovery 协议，填补了机器人领域缺乏标准化故障恢复评估的空白。

### 8. 不足与局限
*   **数据依赖：** SV 的训练需要针对特定环境收集约 5 万步的 rollout 数据，限制了其完全零样本（Zero-shot）部署的能力。
*   **回滚假设：** 回滚机制假设动作是可逆的（如放开物体），在某些不可逆的现实场景（如液体倾倒）中可能失效（虽然论文提出了前向恢复变体作为补充）。
*   **Sim-to-Real 挑战：** 实验全部在仿真环境完成，现实世界中的传感器噪声和视觉偏移可能会影响 CLIP 检索和 SV 的准确性。
*   **子目标分解：** 依赖 VLA 生成子目标序列，若初始分解错误（约 4.3% 的概率），后续恢复也会受限。
