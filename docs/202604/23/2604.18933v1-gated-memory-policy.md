## 摘要
机器人操纵任务表现出不同的记忆需求，从不需要记忆的马尔可夫（Markovian）任务到依赖于跨越单次或多次交互尝试的历史信息的非马尔可夫（non-Markovian）任务。令人惊讶的是，由于分布偏移和过拟合，简单地延长视觉运动策略（visuomotor policy）的观测历史往往会导致性能显著下降。为了解决这些问题，我们提出了门控记忆策略（Gated Memory Policy, GMP），这是一种能够同时学习何时调用记忆以及调用什么记忆的视觉运动策略。为了学习何时调用记忆，GMP 采用了一种学习到的记忆门控机制，仅在必要时选择性地激活历史上下文，从而提高了鲁棒性和反应能力。为了高效地学习调用什么记忆，GMP 引入了一个轻量级的交叉注意力（cross-attention）模块，用于构建有效的潜在记忆表示。为了进一步增强鲁棒性，GMP 在历史动作中注入扩散噪声（diffusion noise），从而在训练和推理过程中减轻对噪声或不准确历史信息的敏感性。在我们提出的非马尔可夫基准测试 MemMimic 上，GMP 相比长历史基线方法实现了 30.1% 的平均成功率提升，同时在 RoboMimic 的马尔可夫任务中保持了具有竞争力的性能。所有代码、数据和实地部署说明均可在我们的项目网站 https://gated-memory-policy.github.io/ 上获取。

## Abstract
Robotic manipulation tasks exhibit varying memory requirements, ranging from Markovian tasks that require no memory to non-Markovian tasks that depend on historical information spanning single or multiple interaction trials. Surprisingly, simply extending observation histories of a visuomotor policy often leads to a significant performance drop due to distribution shift and overfitting. To address these issues, we propose Gated Memory Policy (GMP), a visuomotor policy that learns both when to recall memory and what to recall. To learn when to recall memory, GMP employs a learned memory gate mechanism that selectively activates history context only when necessary, improving robustness and reactivity. To learn what to recall efficiently, GMP introduces a lightweight cross-attention module that constructs effective latent memory representations. To further enhance robustness, GMP injects diffusion noise into historical actions, mitigating sensitivity to noisy or inaccurate histories during both training and inference. On our proposed non-Markovian benchmark MemMimic, GMP achieves a 30.1% average success rate improvement over long-history baselines, while maintaining competitive performance on Markovian tasks in RoboMimic. All code, data and in-the-wild deployment instructions are available on our project website https://gated-memory-policy.github.io/.

### 2. 方法论：Gated Memory Policy (GMP)
GMP 围绕“何时调用记忆”和“调用什么记忆”两个核心问题展开：
*   **核心思想**：引入一个**记忆门控（Memory Gate）**来按需激活历史信息，并使用**交叉注意力（Cross-Attention）**高效提取历史特征。
*   **关键技术细节**：
    *   **记忆门控机制**：使用一个 MLP 预测二值门控信号 $\mu_t$。如果 $\mu_t=0$，则跳过历史信息的交叉注意力计算，仅依赖当前观测。
    *   **门控校准（Calibration）**：这是 GMP 的核心创新。通过在验证集上对比“无记忆策略”和“全记忆策略”的动作预测误差（$\delta_t$ vs $\delta_{mem}$），如果无记忆时的误差显著大于有记忆时，则将该时刻标记为“需要记忆”。随后冻结门控权重，再训练最终策略。
    *   **历史动作噪声注入**：在训练和推理时，对历史动作序列注入扩散调度噪声（Diffusion-scheduled noise）。这能增强模型对历史误差的鲁棒性，防止过拟合。
    *   **高效交叉注意力**：将历史图像和动作编码为 Token 并缓存。通过交叉注意力而非自注意力处理历史，使计算复杂度随历史长度线性增长。
    *   **重叠轨迹训练（Overlapped Trajectory Training）**：为了解决长历史训练时的显存瓶颈，通过在 Batch 内复用相同图像的特征，实现端到端的视觉编码器训练。

### 6. 主要结论与发现
*   **性能提升**：在非马尔可夫任务上，GMP 比长历史基线平均成功率提高 **30.1%**。
*   **通用性**：在马尔可夫任务（RoboMimic）中，GMP 通过关闭门控，保持了与无历史策略相当的顶尖性能，避免了性能退化。
*   **记忆长度**：GMP 能够处理长达 **6000 帧（约 10 分钟）**的历史上下文，且推理延迟保持在可接受范围内（0.16s）。
*   **鲁棒性**：噪声注入显著提升了策略在面对推理时历史轨迹误差时的稳定性。

### 7. 优点（亮点）
*   **按需调用**：通过门控机制解决了“历史信息是双刃剑”的问题，实现了反应式（Reactive）和记忆式（Memory-based）行为的平衡。
*   **自监督校准**：无需人工标注何时需要记忆，通过预测误差自动发现关键记忆时刻。
*   **计算高效**：缓存 Token 和交叉注意力的设计，使其比传统的长上下文 Transformer 更适合高频机器人控制。
*   **真机部署**：在复杂的野外环境和动态任务（如投掷、翻转）中表现出极强的泛化能力。

### 8. 不足与局限
*   **有限窗口**：虽然支持长历史，但本质上仍是固定窗口的注意力机制，无法实现真正的“无限”记忆。
*   **校准依赖**：门控的质量高度依赖于验证集（Dval）的覆盖范围和质量，如果验证集无法反映任务的记忆需求，门控可能失效。
*   **误差代理的局限**：使用动作预测误差作为记忆需求的唯一指标，在具有高度内在歧义或极高不确定性的任务中可能不够准确。
*   **训练流程较长**：需要先训练两个基础策略进行校准，再训练最终策略，相比端到端训练增加了流程复杂度。
