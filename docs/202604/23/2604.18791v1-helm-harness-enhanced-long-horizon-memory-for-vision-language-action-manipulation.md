## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操控任务中却存在系统性失效。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失效问题；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟（memory gap）、验证鸿沟（verification gap）和恢复鸿沟（recovery gap）。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个通过 CLIP 索引关键帧检索关键任务历史的情景记忆模块（EMM）；一个学习得到的状态验证器（SV），它根据观测、动作、子目标和基于记忆的上下文在执行前预测动作失败；以及一个执行回滚和重规划的引导控制器（HC）。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 数据集上，HELM 将任务成功率比 OpenVLA 提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口延长至 H=32 仅带来 5.4 个百分点的提升，且在相同预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了在 CALVIN 上的长程任务性能，并显著提升了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们还发布了 LIBERO-Recovery，这是一个用于评估长程操控中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与整体含义
*   **研究背景**：视觉-语言-动作（VLA）模型（如 OpenVLA）在短程任务中表现出色，但在长程操控任务（步骤多、子目标复杂）中成功率大幅下降。
*   **核心发现**：研究表明，长程任务的失败并非单纯因为上下文窗口（Context Window）不够长。即使将历史长度增加 4-8 倍，性能提升依然有限。
*   **核心问题**：论文识别出 VLA 模型在执行循环中的三个结构性缺陷：
    1.  **记忆鸿沟 (Memory Gap)**：固定窗口丢弃了已完成子目标的证据，导致模型重复执行已完成的动作。
    2.  **验证鸿沟 (Verification Gap)**：模型反应式地生成动作，缺乏在执行前对动作可行性的校验。
    3.  **恢复鸿沟 (Recovery Gap)**：一旦发生错误，模型会在损坏的状态下继续执行，导致连锁失败。

### 2. 方法论
HELM 提出了一个模型无关的框架，通过三个专用组件增强现有的 VLA 模型：
*   **情节记忆模块 (EMM)**：
    *   维护一个基于 CLIP 索引的关键帧存储库。
    *   **写入策略**：在子目标完成、检测到失败或每隔固定步数（20步）时记录状态。
    *   **检索**：通过余弦相似度检索最相关的历史关键帧，并将其作为结构化文本注入 VLA 的语言输入中。
*   **状态验证器 (SV) —— 核心贡献**：
    *   这是一个轻量级的 MLP 模块，用于在执行前预测失败概率 $p_{fail}$。
    *   **输入**：当前观测、检索到的记忆上下文、拟执行动作、当前子目标。
    *   **关键点**：SV 的预测是“记忆条件化”的，即它能根据历史判断当前动作是否合理（例如：如果记忆显示杯子已放好，再次放置动作将被判定为失败）。
*   **安全控制器 (HC)**：
    *   管理子目标堆栈。
    *   如果 SV 预测失败概率超过阈值，HC 会触发**回滚 (Rollback)**（回到之前的检查点）或**重规划**。

### 3. 实验设计
*   **数据集/场景**：
    1.  **LIBERO-LONG**：包含 10 个长程任务，平均 5.8 个子目标。
    2.  **CALVIN (ABC → D)**：评估在未见环境中的长程任务链完成情况。
    3.  **LIBERO-Recovery (本文提出)**：专门用于评估故障恢复的协议，在子目标边界随机注入物体位移或夹爪状态翻转等扰动。
*   **对比方法**：
    *   基础模型：OpenVLA (H=8, 32, 64), Octo。

*   **硬件**：单张 **NVIDIA A100 (80GB)** GPU。
*   **训练时长**：状态验证器 (SV) 的训练非常高效，在收集的 50K 步 Rollout 数据上仅需约 **2 小时**。

### 5. 实验数量与充分性
*   **实验规模**：在 LIBERO-LONG 上进行了 500 次评估回合，在 CALVIN 上进行了 1000 次任务链评估，并使用了 3 个随机种子。
*   **消融实验**：非常详尽，分别验证了 EMM、SV、HC 以及回滚机制的贡献，甚至对比了不同的记忆检索方法（随机、最近、CLIP、学习型）。
*   **公平性**：通过设置“同预算 LoRA 微调”基准，证明了性能提升来自于 HELM 的结构化设计，而非仅仅因为接触了更多的训练数据。

### 6. 主要结论与发现
*   **性能飞跃**：在 LIBERO-LONG 上，HELM 将 OpenVLA 的成功率从 **58.4% 提升至 81.5%**（提升 23.1 个百分点）。
*   **结构化优势**：单纯增加上下文长度（H=32）仅提升 5.4 个百分点，证明了“记忆+验证”机制比“长序列输入”更有效。
*   **恢复能力**：在注入扰动的测试中，HELM 的恢复成功率 (RSR) 远超基准模型，证明了其在动态环境下的鲁棒性。
*   **模型无关性**：该框架在 Octo 模型上同样有效，提升了 21.6 个百分点。

### 7. 优点
*   **即插即用**：不需要对庞大的 VLA 骨干模型进行昂贵的重新训练或微调。
*   **闭环验证**：引入了机器人领域急需的“执行前校验”机制，有效拦截了物理不可行或语义错误的动作。
*   **高效性**：SV 模块极轻量，适合实时控制。
*   **诊断深入**：论文对长程任务失败的原因进行了深入的分类学分析（FM/FV/FR），具有很高的学术参考价值。

### 8. 不足与局限
*   **数据依赖性**：SV 的训练仍需要目标环境的少量 Rollout 数据，这限制了其在完全陌生环境下的零样本部署能力。
*   **回滚限制**：在现实世界中，某些动作是不可逆的（如打破鸡蛋），此时“回滚”策略失效。虽然论文提出了“前向恢复”变体，但效果不如回滚。
*   **仿真局限**：实验主要在仿真环境中完成，真机部署时可能面临视觉噪声和更复杂的物理交互挑战。
