## 摘要
在机器人学习中，跨不同本体和环境扩展训练数据集已成为学习可泛化机器人策略的主流范式。这些策略通常通过行为克隆来模仿从预先收集的演示中的动作。然而，由于机器人动作与数据收集机器人的动力学相关联，不同的机器人可能需要不同的动作来实现相同的运动。这种差异阻碍了策略在不同机器人上的训练和部署。为了解决这个问题，我们提出使用笛卡尔状态差作为跨机器人的通用动作表示，并引入了状态预测与自适应命令执行（SPACE）框架。SPACE在三个层面上处理机器人动力学变化：跨不同本体、同一本体的不同硬件单元以及单个机器人在运行过程中的变化。它由两个组件组成：(i) 一个笛卡尔状态差策略，用于预测几何末端执行器位移；(ii) 动作适配器，将预测的笛卡尔状态差转换为机器人特定的控制命令。实验表明，在从跨不同本体和同一本体不同硬件单元收集的数据中进行学习时，SPACE显著优于直接预测控制命令的策略。SPACE在部署时的动力学变化下也保持鲁棒，包括控制频率、物体重量和控制器增益的变化。项目页面可在 http://haeone.site/space-website/ 获取。

## Abstract
In robot learning, scaling training datasets across diverse embodiments and environments has become a dominant paradigm for learning generalizable robot policies. These policies are commonly trained via behavior cloning to imitate actions from pre-collected demonstrations. However, since robot actions are tied to the dynamics of the data collection robot, different robots may require different actions to achieve the same motion. This discrepancy hinders both policy training and deployment across diverse robots. To address this, we propose using Cartesian state delta as a universal action representation across robots, and introduce State Prediction and Adaptive Command Execution (SPACE) framework. SPACE handles robot dynamics variation at three levels: across different embodiments, across hardware units of the same embodiment, and within a single robot during operation. It consists of two components: (i) a Cartesian state delta policy that predicts geometric end-effector displacement, and (ii) Action Adapter, which converts the predicted Cartesian state delta into robot-specific control commands. Experiments show that SPACE substantially outperforms policies that directly predict control commands when learning from data collected across different embodiments and across hardware units of the same embodiment. SPACE also remains robust under dynamics shifts at deployment, including changes in control frequency, object weight, and controller gains. ## 2. 论文提出的方法论

### 核心思想
将策略输出定义为**笛卡尔状态增量** Δpₜ = pₜ₊₁ – pₜ（即末端执行器在笛卡尔空间中的实际位移），这是一个与机器人具体动力学无关的几何量。然后，通过一个轻量级的**动作适配器**将预测的 Δp 映射到目标机器人当前动力学条件下的控制命令 uₜ。

### 关键技术细节
1. **笛卡尔状态增量策略（Cartesian State Delta Policy）**
   - 训练目标：最大化演示数据中观测到状态增量的似然：
     ```
     max θ  E[log πθ(Δpₜ | oₜ)]
     ```
     其中观测 oₜ = {pₜ, Iₜ, lₜ} 包含末端位姿、图像和语言指令。
   - 与直接预测控制命令的策略不同，状态增量可以从任何提供末端位姿的数据集中获得，不受控制命令来源的限制（例如人类手持夹具数据、动觉示教数据等）。

2. **动作适配器（Action Adapter）**
   - **离线校准**：在目标机器人上执行随机脚本策略，收集M条轨迹（M=10, 长度K=50），构成校准数据集 D_cal = {(Δp, u)}。通过最小二乘拟合线性模型：
     ```
     min_{W0,b0} Σ_{(Δp,u)∈D_cal} ||W0Δp + b0 - u||²
     ```
     得到初始参数 (W0, b0)。
   - **在线自适应**：在策略部署过程中，使用**最小均方算法（LMS）** 持续更新适配器参数。每步执行命令后，观测实际运动 Δp_t^{obs}，计算预测误差 e_t = W_t Δp_t^{obs} + b_t – û_t，然后更新：
     ```
     W_{t+1} ← W_t - μ e_t (Δp_t^{obs})ᵀ
     b_{t+1} ← b_t - μ e_t
     ```
     学习率 μ=0.2。

3. **算法流程（伪代码见附录 A）**
   - 收集校准数据 → 初始化Action Adapter → 策略输出目标状态增量 Δp^{target} → 适配器生成命令 û → 执行并观测实际增量 → 计算误差 → 更新适配器参数。

### 公式
- 行为克隆目标：`max θ E[log πθ(Δpₜ | oₜ)]`
- 适配器最小二乘目标：`min Σ ||WΔp + b - u||²`
- LMS更新：`W ← W - μ e (Δp^{obs})ᵀ`，`b ← b - μ e`

## 6. 论文的主要结论与发现

- **跨本体学习**：SPACE 在UR5+FR3联合训练的三个任务上平均比直接预测控制命令的策略**高30%**；在零样本从UR5迁移到FR3的任务（Cloth）上从0%提升到**92%**；在UMI+FR3联合训练任务上提升**50%**。
- **跨硬件单元学习**：在DROID多硬件数据集上，SPACE 在两个任务上分别**高出80%和84%**；在单机迁移实验中，控制命令策略在新硬件上成功率从98%骤降到18%，而SPACE仍保持**84%**。
- **动力学变化鲁棒性**：控制频率从15Hz提升到30Hz时，SPACE保持成功率（约94%），而控制命令策略从98%降至50%；负载从90g增加到530g时，SPACE成功率为92%，控制命令策略为0%；增益变化（0.5×, 1.5×）下，SPACE仍维持>95%成功率，而控制命令策略降至25%或0%。
- **消融验证**：Action Adapter的**离线初始化和在线更新缺一不可**：仅离线时成功率从96%降至15%；仅在线（无校准）为0%；仅在线连续累积多个rollout后也仅30%。
- 直接使用状态增量策略而不加Action Adapter**在所有任务上成功率为0%**，说明必须经过适配器转换。

## 7. 优点

- **创新性**：将动作表示从控制命令解耦为末端位移（状态增量），并配合简单有效的线性适配器在线自适应，解决了多机器人动力学不一致的核心痛点。
- **完整性**：方法论在三个层次（跨本体、跨硬件、运行时）均得到验证，且每个层次都展示了显著优势，逻辑自洽。
- **实用性与简洁性**：Action Adapter仅需1分钟随机校准数据，在线更新计算量小（LMS算法），易于集成到现有策略框架（如π0.5）。
- **广泛的适用性**：支持从缺乏控制命令的数据（如UMI、动觉示教）中学习；与模态无关，可应用于任何提供末端位姿的机器人。
- **实验设计严谨**：包含丰富的消融和基线对比（不同动作空间、增益调优、增量累积），结果清晰展示各组件贡献。
- **开源与可复现**：提供项目页面和算法伪代码，便于复现。

## 8. 不足与局限

- **缺乏力感知**：笛卡尔状态增量只描述运动，无法表达施加力的大小。对于需要恒力或力控的任务（如插拔、表面擦拭），仅依赖位置增量可能不足。论文在局限性中承认了这一点。
- **控制模态转换限制**：当前方法假设目标机器人支持笛卡尔增量控制命令。对于只支持关节空间命令的机器人，可能需要额外的逆运动学转换，论文未探索。
- **校准依赖随机轨迹**：校准数据需随机探索动作空间，可能不适用于执行安全要求极高或禁止随机运动的场景。校准时长虽短，但增加了额外部署步骤。
- **在线自适应范围有限**：LMS更新假设动力学变化相对平滑且局部，对于剧烈突变（如突然施加外力）可能不够鲁棒；仅用历史一步误差更新，收敛速度可能受限。
- **实验局限性**：
  - 所有实验仅在Franka Research 3机器人上进行，未在其他型号（如UR5、KUKA）验证反向迁移或联合训练的效果。
  - 任务均为桌面拾放类操作，未涉及更复杂的如开门、组装等精细或长任务。
  - 跨本体实验用了UR5数据，但FR3作为目标机器人，未测试UR5作为目标时SPACE的适用性。
  - DROID实验仅筛选了“marker”相关子集，未在完整DROID数据集上验证缩放性。
  - 动力学变化实验（频率、载荷、增益）仅测试了单一任务PnP Box，泛化性有待进一步验证。
- **理论基础相对简单**：Action Adapter采用线性模型+LMS，面对强非线性或时变动力学可能不足，更复杂的模型（如概率模型或隐状态估计）可能进一步提升鲁棒性。
