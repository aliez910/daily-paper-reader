## 摘要
视觉-语言-动作（VLA）模型中的跨具身迁移仍然具有挑战性，因为低级状态和动作空间在不同机器人平台上有根本区别。我们观察到，操作背后的高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在具身之间是基本共享的。基于这一观察，我们提出了ZR-0，一个26亿参数端到端VLA模型，它使用密集具身思维链（ECoT）监督来在视觉语言模型（VLM）内对齐跨具身表示。ZR-0采用双流架构：一个预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而基于扩散Transformer的动作专家（系统1）通过流匹配产生连续动作块。这两个组件通过交叉注意力耦合，并带有一个注意力掩码，该掩码将动作专家限制为仅输入提示特征，使得在推理时可以完全跳过ECoT生成而没有任何性能损失。ZR-0在ProcCorpus-60M上进行预训练，这是一个大规模数据集，包含来自超过40万条轨迹的大约6000万帧（约1000小时），密集的ECoT标注覆盖了96.8%的帧。我们在三个模拟基准测试上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和人形（RoboCasa GR-1桌面）具身，以及在xArm平台上的真实世界实验，展示了在所有设置中的强大性能。代码和模型检查点可在 https://github.com/RUCKBReasoning/ZR-0 获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

# 论文总结：ZR-0——使用密集具身思维链监督训练视觉-语言-动作模型

## 1. 核心问题与研究动机
- **问题**：跨具身（cross-embodiment）迁移是视觉-语言-动作（VLA）模型面临的根本挑战。不同机器人平台在运动学配置、控制接口、传感器设置上差异巨大，导致低维状态和动作空间异构且物理含义不同。现有的零填充、归一化、固定语义维度等方法仅缓解格式差异，却无法实现深层的**语义对齐**——即让模型学到共享的、可迁移的表示，而非仅仅拟合统一架构下的具身特定模式。
- **关键观察**：虽然低维动作空间各异，但操作背后的**高级认知过程**（场景感知、物体识别、任务进度估计、子任务分解等）在跨具身中基本共享。因此，作者提出利用这种共享认知作为监督信号来对齐表示。

## 2. 方法论：核心思想与关键技术
### 2.1 整体架构（双流 System 1 / System 2）
- **System 2（VLM）**：基于预训练的 Qwen3-VL-2B-Instruct，接收任务指令 `l` 和当前图像观测 `o_t`，生成结构化的**具身思维链（ECoT）** 推理 `r_t`。训练时采用标准的下一个词预测损失 (L_ntp)。
- **System 1（DiT 动作专家）**：基于 Diffusion Transformer (DiT)，接收 VLM 输出的特征 `f_t` 与机器人状态 `s_t`，通过**流匹配（flow matching）** 生成连续动作块 `A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]`。动作专家包括状态编码器、动作编码器、多层 DiT 块和动作解码器（均为 MLP）。
- **耦合方式**：DiT 块中的交叉注意力层使用 VLM 特征作为 Key/Value，但施加**注意力掩码**，使动作专家**只能关注输入提示（指令和图像）对应的特征，忽略 ECoT 词元**。这一设计使得推理时无需生成 ECoT，单次前向传播即可为动作专家提供所需特征。
- **架构细节**：DiT 块内采用 1:3 的自注意力与交叉注意力比率（相比 GR00T N1 的 1:1 更强调跨模态交互）。

### 2.2 训练目标
- 联合优化两个目标：
  - **下一个词预测损失（L_ntp）**：用于 ECoT 序列生成，仅更新 VLM 参数。
  - **流匹配损失（L_fm）**：用于动作块回归，梯度通过 VLM 特征反向传播更新整个模型（VLM + 动作专家）。
  - 总损失：`L = L_ntp + α L_fm`，预训练时 α=5，后训练时 α=1。
- 噪声调度：flow matching 时间步 τ 从 Beta(1.5, 1.0) 采样，强调更噪声的时刻。

### 2.3 ECoT 组成（共 6 部分）
- **场景描述**：文本描述当前视觉场景 → 增强物体识别。
- **进展评估**：评估已完成内容 + 完成指示符（Yes/No） → 感知任务进度。
- **未来计划**：自然语言描述剩余步骤 → 长期规划。
- **待做动作（To-Do Actions）**：分解为原子子任务（如“Grasp the blue plate”），使用具身无关的格式（动词+宾语+介词短语） → **跨具身对齐的关键**。
- **目标物体**：边界框 JSON 格式 → 视觉定位监督。
- **离散动作**：FAST 分词器产生的具身特定离散动作标记 → 连接高层推理与低层连续控制。

### 2.4 推理流程
- 不生成 ECoT：直接利用 VLM 对输入提示的前向传播特征，初始化高斯噪声并迭代去噪（流匹配的欧拉积分），经过 N 步得到动作块。单块 A6000 上约 90 ms，支持实时控制。

- **ECoT 监督有效**：在 LIBERO 上移除 ECoT 导致成功率下降 2.1 个百分点，证明 action-only 监督不足以充分对齐 VLM 表示。

## 7. 优点
- **密集 ECoT 监督的思想新颖**：利用跨具身共享的高层认知过程作为监督信号，推动表示对齐，而不是依赖复杂的格式统一。
- **双流架构解耦设计**：System 2 提供语义理解，System 1 负责连续控制，通过交叉注意力耦合；注意力掩码策略使推理零额外开销。
- **大规模预训练数据 ProcCorpus-60M**：提供密集 ECoT 注释（96.8% 帧覆盖率），涵盖广泛机器人数据集，利于学习通用物理常识。
