## 摘要
视觉-语言-动作（VLA）模型在跨实体迁移方面仍然具有挑战性，因为机器人平台之间的低级状态和动作空间根本不同。我们观察到，操作背后高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在不同实体间是基本共享的。基于这一观察，我们提出了ZR-0，一个拥有26亿参数的端到端VLA模型，它使用密集的具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨实体表示。ZR-0采用双流架构：一个预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而一个基于扩散Transformer的动作专家（系统1）通过流匹配产生连续的动作片段。这两个组件通过交叉注意力耦合，并带有一个注意力掩码，限制动作专家仅输入提示特征，从而在推理时可以完全跳过ECoT生成，而没有任何性能损失。ZR-0在ProcCorpus-60M上进行了预训练，这是一个大规模数据集，包含来自超过40万个轨迹的大约6000万帧（约1000小时），密集的ECoT注释覆盖了96.8%的帧。我们在三个模拟基准上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和人形（RoboCasa GR-1 Tabletop）实体，以及在xArm平台上的真实世界实验，展示了在所有设置上的强大性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：视觉-语言-动作（VLA）模型在跨机器人平台（跨实体）迁移时面临根本性挑战，因为不同机器人的**低级状态和动作空间**（如关节角度、动作表示方式、自由度等）存在巨大差异，难以直接对齐。
- **关键观察**：尽管底层控制细节不同，但操作过程中**高级认知过程**（如场景感知、物体识别、任务规划、子任务分解）在不同实体间是共享的。
- **整体含义**：如果能够通过训练让模型学习到这种**实体无关的高层推理能力**，就可以促进跨实体的知识迁移，从而构建通用的机器人策略。论文提出使用**密集的具身思维链（Embodied Chain-of-Thought, ECoT）** 作为监督信号，在视觉-语言模型（VLM）内部对齐跨实体表示，使模型能够学习到可迁移的高层理解，而无需在推理时付出额外代价。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **ZR-0模型架构**：基于“系统1/系统2”认知框架的双流架构：
  - **System 2（VLM）**：预训练视觉-语言模型（基于Qwen3-VL-2B-Instruct），处理任务指令和图像观测，生成结构化的ECoT推理序列。训练期间，VLM通过下一个词预测损失（L_ntp）学习生成包含场景描述、进度评估、未来计划、子任务分解、目标物体包围框、离散动作的ECoT。
  - **System 1（动作专家）**：基于扩散Transformer（DiT）的生成模型，接收VLM的最后一层隐藏状态和机器人状态，通过流匹配（Flow Matching）生成连续的H步动作片段（chunk）。DiT模块采用“1个自注意力 + 3个交叉注意力”的结构，交叉注意力中动作专家作为query，VLM特征作为key/value。
  - **交叉注意力掩码 (Attention Mask)**：关键创新点 —— 动作专家**只允许关注VLM的输入提示（任务指令和图像）对应的特征，而忽略ECoT生成的文本token的特征**。这使得推理时VLM只需要一次前向传播计算输入提示的特征即可，完全跳过ECoT的生成，实现零额外推理开销。
- **ECoT的六组件及作用**：
  - 场景描述（Scene Description）：增强物体识别能力。
  - 进度评估（Progress Assessment）：感知任务进度，判断是否完成。
  - 未来计划（Future Plan）：长时程推理和规划。
  - 待办动作（To-Do Actions）：将未来计划分解为原子化的、实体无关的子任务（如“Grasp the blue plate”），是跨实体对齐的关键。
  - 目标物体（Target Objects）：以包围框形式指明操作对象，增强视觉定位。
  - 离散动作（Discrete Actions）：使用FAST分词器产生的实体特有离散动作token，作为高层推理与底层连续动作的桥梁。
- **训练目标**：
  - 下一个词预测损失（L_ntp）：仅更新VLM，监督ECoT生成。
  - 流匹配损失（L_fm）：更新整个模型（VLM + 动作专家），监督动作片段生成。噪声时间步τ从Beta(1.5, 1.0)分布采样，强调高噪声步。
  - 总损失：L = L_ntp + α L_fm（α控制权衡，预训练α=5，后训练α=1）。
- **推理流程**：输入任务指令、当前图像、机器人状态，VLM一次性前向传播提取特征（不生成ECoT文本），动作专家通过欧拉积分逐步去噪从高斯噪声生成动作片段，执行后重新规划。生成一个动作片段约需90ms（A6000 GPU），适合实时部署。

### 7. 优点
- **方法巧妙**：利用智能推理（ECoT）作为中间监督，却通过注意力掩码避免了推理开销，平衡了表示增强与效率。
- **架构灵活**：双流设计使得VLM可以任意更换更强大的预训练模型，动作专家独立处理连续动作，易于扩展。
- **数据丰富**：预训练数据集ProcCorpus-60M规模大且带密集ECoT注释（96.8%覆盖），是模型成功的关键支撑。

- **开源贡献**：代码和模型权重开源，便于社区复现和进一步发展。
- **跨实体对齐的新范式**：从“格式化统一”转向“语义对齐”，通过高层推理实现实体无关的知识传递，为VLA跨实体学习提供了新思路。

### 8. 不足与局限
- **预训练数据规模相对较小**：约1000小时，比π0（超1万小时）、LingBot-VLA（约2万小时）等少一个量级，限制了某些低频技能的覆盖和精细动作性能（如真实世界Hang Cups任务不如π0.5）。
- **ECoT注释成本高**：密集注释需要强VLM前向推理，计算资源消耗大，且当前对均匀注释，未优化信息帧选择。论文自身提出这是一个待解决的效率问题。
- **某些技能表现不佳**：在RoboCasa GR-1的“Close”类任务（关柜门、抽屉）和多阶段交互任务上明显落后于专门方法，可能因为预训练语料中此类数据稀少，ECoT学习到的对齐表示不足。
- **消融不全面**：仅对比了有无ECoT，没有更深入分析ECoT各组件（如场景描述、目标物体）的单独贡献，也没有研究不同α加权或不同ECoT长度的影响。
- **真实实验规模有限**：仅4个任务且每个任务10次试验，虽然使用了任务进度评分，但统计稳定性不如仿真；任务挑战性总体中等，缺乏高难度长时程真实任务。
- **未评估泛化到全新实体**：虽然预训练包含多实体，但下游微调均在已知实体类型上进行，未直接测试零样本跨实体（如从单臂到人形不做额外微调），论文称之为“跨实体适应”而非“零样本迁移”。
