## 摘要
视觉-语言-动作（VLA）模型中的跨具身迁移仍然具有挑战性，因为低级状态和动作空间在不同机器人平台上根本不同。我们观察到，操作背后的高级认知过程，包括场景感知、目标识别、任务规划和子任务分解，在不同具身之间很大程度上是共享的。基于这一观察，我们提出了ZR-0，一个拥有26亿参数的端到端VLA模型，它使用密集具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨具身表示。ZR-0采用双流架构：预训练VLM（系统2）在训练期间生成结构化的ECoT推理，而基于扩散Transformer的动作专家（系统1）通过流匹配产生连续的动作块。这两个组件通过交叉注意力耦合，注意力掩码限制动作专家仅输入提示特征，使得在推理时可以完全跳过ECoT生成且没有任何性能损失。ZR-0在ProcCorpus-60M上进行了预训练，这是一个大规模数据集，包含来自超过40万条轨迹的大约6000万帧（约1000小时），密集ECoT注释覆盖了所有帧的96.8%。我们在三个仿真基准上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和人形（RoboCasa GR-1桌面）具身，并在xArm平台上进行了真实世界实验，展示了在所有设置中的强劲性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

## 1. 核心问题与整体含义（研究动机和背景）
- **跨具身迁移的挑战**：视觉-语言-动作（VLA）模型在不同机器人平台之间迁移时，由于低层状态/动作空间异构（如自由度数量、坐标系、控制接口等），导致难以共享知识。现有方法主要采用格式级对齐（如零填充、显式维度映射），但无法消除语义鸿沟。
- **关键观察**：操作背后的高层认知过程（场景感知、目标识别、任务规划、子任务分解）在不同具身之间是共享的。因此，如果能让模型学习到这些具身无关的推理能力，就能实现更有效的跨具身迁移。
- **核心思路**：提出 **ZR-0**，一个 2.6B 参数的 VLA 模型，通过**密集具身思维链（Dense Embodied Chain-of-Thought, ECoT）监督**，在视觉-语言模型（VLM）的表示层对齐跨具身的知识，使得低层动作预测可以从高质量的高层推理特征中获益。

## 2. 方法论：核心思想、关键技术细节
- **总体架构**：双流设计，受 System 1 / System 2 认知框架启发。
    - **System 2（VLM）**：预训练 VLM（Qwen3-VL-2B-Instruct），处理任务指令与多视角图像，在训练时生成结构化的 ECoT 推理序列；ECoT 包含 6 个组件：
        1. **场景描述**：提升目标识别。
        2. **进度评估**：感知任务完成情况。
        3. **未来计划**：长时程推理与规划。
        4. **待做动作**：具身无关的子任务分解（动词+对象+介词短语）。
        5. **目标对象**：边界框，增强视觉定位。
        6. **离散动作**：FAST 分词器产生的具身相关离散 token，桥接高层推理和低层控制。
    - **System 1（动作专家）**：基于 Diffusion Transformer (DiT) 的专家模块，通过流匹配（Flow Matching）直接输出**连续动作块**（chunk size H）。
- **耦合方式**：VLM 的最后一层隐藏状态通过**交叉注意力**馈入动作专家。关键设计：在交叉注意力中施加**注意力掩码**，使动作专家只能访问 VLM 的**输入提示特征**（指令 + 图像），而不能访问 ECoT token。这使得推理时完全可以跳过 ECoT 生成，仅需一次前向通过 VLM 即可为动作专家提供全部所需特征。
- **训练目标**：联合优化两个损失。
    - 下一个 token 预测损失（`L_ntp`）：用于 ECoT 推理生成，仅更新 VLM。
    - 流匹配损失（`L_fm`）：拟合去噪向量场 `(A_t - ε)`，梯度同时传播至动作专家和 VLM（通过 VLM 特征）。
    - 总损失：`L = L_ntp + α * L_fm`，训练时 `α = 5`。
- **推理机制**：无需自回归生成 ECoT；从高斯噪声初始化动作块，通过 Euler 积分迭代去噪（通常 4 步），每步直接使用 VLM 的输入特征。在 A6000 上单个动作块生成约 90 ms，适合实时部署。

## 7. 优点
- **跨具身对齐创新**：首次将**密集、结构化、具身无关的 ECoT 推理**作为监督信号，有效对齐 VLM 表示，超越了格式级的对齐方法。
- **高效的双流设计**：注意力掩码设计使得训练时可利用完整 ECoT 信息，推理时完全跳过自回归生成，兼顾了表示质量和推理效率。
- **大规模 ECoT 数据构建**：ProcCorpus-60M（96.8% 帧有注释）为方法提供了坚实基础，且开源。
- **系统的实验验证**：覆盖三个模拟具身 + 真实平台，与多个现代 VLA 基线对比，消融实验清晰证明了 ECoT 的关键作用。
- **泛化能力**：在域随机化场景下鲁棒性强（RoboTwin 2.0 中清场/随机差异仅 0.72%），体现预训练带来的视觉不变性。

## 8. 不足与局限
- **数据规模有限**：预训练数据约 1000 小时，远小于 π0（>10k h）、LingBot-VLA（~20k h）等，导致部分技能（如关门）覆盖不足，影响下游性能（RoboCasa 中 Close 类任务低于 JoyAI-RA）。
- **高精度操作偏弱**：真实实验中 Hang Cups 任务落后 π0.5（70.0 vs 85.0），说明 ECoT 对精细运动精度的帮助有限，可能更依赖大规模动作监督。
- **ECoT 标注成本高**：对每帧进行 VLM 标注消耗大量计算；论文虽在 Discussion 中提及未来可挑选关键帧，但未在当前实现。
- **真实实验规模小**：每任务仅 10 次，统计变异大，且仅在一个具体平台（xArm）上评估，跨真实场景的泛化尚不明确。
- **未涉及零样本迁移或跨 benchmark 微调**：所有下游实验都使用对应 benchmark 的微调数据，没有测试预训练后直接零样本操控的能力。
- **潜在偏差风险**：ECoT 自动标注可能引入 VLM 自身的错误（幻觉、错误推理），进而影响训练；论文未对此进行纠偏分析。
