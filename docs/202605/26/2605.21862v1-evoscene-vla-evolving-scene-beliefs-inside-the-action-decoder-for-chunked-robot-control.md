## 摘要
分块视觉-语言-动作（VLA）策略预测多步机器人控制，每次更新仅基于当前视觉观察。然而，机器人的动作会导致接触、遮挡和物体运动，后续决策所依赖的几何结构可能在下一视觉更新到来之前发生变化。空间VLA改善了当前帧的几何信息，时间VLA则聚合了历史帧，但两者均未在分块之间维护一个经动作更新的场景先验。我们主张在控制调用之间维护一个持久性的经动作更新的场景状态，并提出了EvoScene-VLA。其循环场景前缀在分块之间传递具有几何感知的场景状态。每次视觉-语言模型（VLM）调用时，VLM将当前观察中的场景信息与来自上一分块的经动作更新的先验相结合；动作解码器输出下一动作分块以及一个紧凑的场景更新。该更新成为下一先验，当下一调用到来时，VLM将其与新观察进行校正。因此，每次控制调用都从一个既反映最近动作又包含最新视觉证据的场景先验开始。训练期间，场景预测器提供未来场景令牌目标，几何锚定则将场景槽与冻结的深度和3D教师对齐。部署时我们舍弃这两个模块。在31个RoboTwin任务中，EvoScene-VLA在固定评估中将平均成功率从87.2%提升至89.1%，在随机评估中从86.1%提升至88.5%。在Galaxea R1-Lite真实机器人上，EvoScene-VLA优于所有基线。

## Abstract
Chunked vision-language-action (VLA) policies predict multi-step robot controls, conditioning each update on the current visual observation alone. Yet robot actions cause contact, occlusion, and object motion, and the geometry that later decisions depend on can change before the next visual update arrives. Spatial VLAs improve current-frame geometry. Temporal VLAs aggregate past frames. Neither maintains an action-updated scene prior across chunks. We argue for a persistent action-updated scene state across control calls, and introduce EvoScene-VLA. Its recurrent scene prefix carries a geometry-aware scene state across chunks. At each vision-language model (VLM) call, the VLM combines scene information from the current observation with the action-updated prior from the previous chunk; the action decoder outputs both the next action chunk and a compact scene update. This update becomes the next prior, which the VLM corrects against the new observation when the next call arrives. Each control call therefore starts from a scene prior that reflects both recent actions and fresh visual evidence. During training, \textbf{Scene Predictor} supplies future scene-token targets, and Geometric Anchor aligns scene slots with frozen depth and 3D teachers. We discard both modules at deployment. On 31 RoboTwin tasks, EvoScene-VLA raises average success from 87.2% to 89.1% in fixed evaluation and from 86.1% to 88.5% in randomized evaluation. On the Galaxea R1-Lite real robot, EvoScene-VLA outperforms all baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有分块视觉-语言-动作（VLA）策略每步仅依赖当前视觉观测预测多步动作。然而，机器人执行的动作会引起接触、遮挡和物体运动，导致决策所依赖的场景几何在下一视觉更新到来前就已改变。空间VLA（如SpatialVLA）只改善当前帧几何，时间VLA（如MemoryVLA）仅聚合历史观测，均未在分块之间维护一个**经动作更新的场景先验**。
- **动机**：策略需要一种**持久、可经动作更新、并能被新观测校正**的场景表示，以避免每个新分块被迫从零推理场景状态。
- **整体含义**：本文提出将动作解码器视为自然更新场景状态的位置，通过**循环场景前缀**跨分块传递场景信念，使每个控制调用都从一个融合了近期动作和最新视觉证据的先验开始。

## 2. 方法论

### 核心思想
- **循环场景前缀**：在VLM前缀中增加两类槽：
    - **观测槽**：从当前多视图图像提取几何证据。
    - **先验槽**：继承上一分块经动作解码器去噪后输出的场景状态。
- **联合动作-场景去噪**：动作解码器在单次流匹配（flow-matching）过程中同时去噪动作块和匹配的场景块，去噪后的场景令牌在分块末尾写入，成为下一分块的先验。
- **训练辅助模块（推理时丢弃）**：
    - **几何锚定**：两级锚定——局部深度锚定（通过跨视图掩码深度重建，监督每个观测槽的几何）和全局3D锚定（通过冻结的3D基础模型特征蒸馏到聚合场景表示）。
    - **场景预测器**：以当前场景表示和动作序列为条件，预测未来关键帧的场景潜在表示，为联合去噪提供目标。

### 关键技术细节
- **VLM前缀排序**：`[图像令牌, 观测槽_{1..V}, 先验槽, 语言令牌]`，采用**非对称注意力掩码**：观测槽可读取对应视图图像，先验槽可读取所有观测槽，但图像/语言令牌不反向关注场景槽，保持预训练通路完整。
- **联合流匹配**：采样共享时间τ，对动作和场景分别构造线性插值，动作专家预测两个速度场，损失包括动作FM损失和场景FM损失。
- **训练总损失**：`L = L_actFM + λ1*L_geo + λ2*L_rep + λ3*L_pred + λ4*L_sceneFM`，前三项用于场景表示、预测和融合。

### 算法流程（文字描述）
- **训练**：给定当前观测、语言、状态、动作块和未来多视角帧 → 形成前缀并VLM前向 → 通过局部深度锚定和全局3D锚定计算两个几何损失 → 场景预测器输出未来场景潜在 → 计算预测损失 → 联合去噪目标，计算动作和场景FM损失 → 反向传播更新全部参数。
- **推理**：加载VLM和动作专家（丢弃几何锚定和场景预测器）→ 每分块：前缀VLM前向，得到当前场景表示→ 初始去噪变量，进行10步欧拉流匹配 → 去噪出动作块和场景块 → 执行动作，场景块最后一帧写回先验槽 → 下一分块循环。

## 8. 不足与局限

- **潜在状态可解释性弱**：循环场景先验为潜在嵌入，无法直接可视化或解释其几何内容，对诊断失败行为不友好。
