## 摘要
视觉-语言-动作（VLA）模型有望实现通用的机器人操控，但通常作为短时界策略进行训练和部署，这些策略假设最新的观测结果足以进行动作推理。这一假设在非马尔可夫长时界任务中失效，因为在这些任务中，与任务相关的证据可能会被遮挡或仅出现在轨迹的早期阶段，且杂乱的环境和干扰物会使细粒度的视觉定位变得脆弱。我们提出了 CodeGraphVLP，这是一个分层框架，通过将持久的语义图状态与可执行的基于代码的规划器以及进度引导的视觉-语言提示相结合，实现了可靠的长时界操控。语义图在部分可观测的情况下维护与任务相关的实体和关系。合成的规划器在语义图上执行，以进行高效的进度检查，并输出子任务指令以及与子任务相关的对象。我们利用这些输出构建抑制杂乱的观测结果，使 VLA 执行器专注于关键证据。在真实世界的非马尔可夫任务中，CodeGraphVLP 相比于强大的 VLA 基准模型和支持历史信息的变体，提高了任务完成率，同时与 VLM 在环（VLM-in-the-loop）规划相比，显著降低了规划延迟。我们还进行了广泛的消融研究，以确认各组件的贡献。

## Abstract
Vision-Language-Action (VLA) models promise generalist robot manipulation, but are typically trained and deployed as short-horizon policies that assume the latest observation is sufficient for action reasoning. This assumption breaks in non-Markovian long-horizon tasks, where task-relevant evidence can be occluded or appear only earlier in the trajectory, and where clutter and distractors make fine-grained visual grounding brittle. We present CodeGraphVLP, a hierarchical framework that enables reliable long-horizon manipulation by combining a persistent semantic-graph state with an executable code-based planner and progress-guided visual-language prompting. The semantic-graph maintains task-relevant entities and relations under partial observability. The synthesized planner executes over this semantic-graph to perform efficient progress checks and outputs a subtask instruction together with subtask-relevant objects. We use these outputs to construct clutter-suppressed observations that focus the VLA executor on critical evidence. On real-world non-Markovian tasks, CodeGraphVLP improves task completion over strong VLA baselines and history-enabled variants while substantially lowering planning latency compared to VLM-in-the-loop planning. We also conduct extensive ablation studies to confirm the contributions of each component.

### CodeGraphVLP 论文结构化总结

#### 1. 核心问题与研究动机
*   **核心问题**：现有的视觉-语言-动作（VLA）模型大多基于**马尔可夫假设**，即认为当前的单帧观测足以决定下一步动作。然而，在长程（Long-horizon）机器人任务中，这一假设会失效，原因包括：
    *   **非马尔可夫依赖**：动作决策往往依赖于过去发生的事情（如：物体被遮挡后的位置、初始状态的记忆）。
    *   **环境干扰**：杂乱场景中的无关物体（干扰项）会误导 VLA 的视觉定位。
    *   **效率瓶颈**：现有的记忆增强型 VLA 算力开销大，而“VLM 在环”规划（反复调用大模型）延迟极高，难以实现闭环控制。
*   **研究动机**：提出一种既能保持长程记忆、又能高效推理、且对杂乱环境鲁棒的分层框架。

#### 2. 方法论
CodeGraphVLP 的核心思想是将**结构化状态表示**、**程序化规划**与**注意力引导的执行**相结合：
*   **持久化语义图（Semantic-Graph State）**：
    *   通过多视角观测初始化，利用实例分割（YOLOE）和 VLM 识别任务相关物体。
    *   建立物体间的空间关系（如 `in`, `on`, `near`），并在执行过程中通过物体跟踪（Cutie）动态更新图状态，确保在部分遮挡下依然能维持环境的“全局观”。
*   **代码即规划器（Code-as-Planner）**：
    *   利用 LLM（如 GPT-4/5）一次性生成针对特定任务的 Python 代码。
    *   该代码作为高层规划器，通过查询语义图来检查任务进度（Progress Check），并输出当前的**子任务指令**及**相关物体列表**。
    *   相比反复调用 VLM，执行 Python 代码几乎不产生延迟。
*   **去噪视觉语言提示（Clutter-Free Prompting）**：
    *   **视觉层面**：根据规划器输出的相关物体，对原始图像进行掩码处理（Masking），只保留关键物体，黑化干扰区域。
    *   **语言层面**：将长程指令简化为具体的子任务指令。
    *   最后将“干净”的图像和指令输入给底层的 VLA 执行器（如 $\pi_0$）。

#### 3. 实验设计
*   **实验场景**：使用 UR10e 机械臂在真实世界桌面环境进行操作。
*   **任务设计（非马尔可夫特性）**：
    1.  **两次取放（Pick-and-Place Twice）**：需记住初始位置以返回原点。
    2.  **放置并堆叠（Place-and-Stack）**：物体放入杯中被遮挡后，需凭记忆操作。
    3.  **交换杯子（Swap Cups）**：涉及中间缓冲区和多步逻辑，需严格跟踪进度。
*   **对比基准（Benchmarks）**：
    *   **主流 VLA 模型**：$\pi_0$, $\pi_0$ FAST, $\pi_{0.5}$, Gr00T N1.5。
    *   **历史增强变体**：支持多帧历史输入的 Gr00T N1.5。
    *   **消融实验**：对比了无代码规划器、无语义图、无视觉掩码等不同配置。

#### 4. 资源与算力
*   **训练配置**：
    *   使用 **4 张 NVIDIA A6000 GPU** 进行微调。
    *   采用 **LoRA**（低秩自适应）技术。
    *   训练迭代次数为 **50,000 次**，Batch Size 为 128。
*   **推理配置**：
    *   推理阶段仅需 **1 张 GPU**。
    *   规划器（代码执行）延迟极低，主要耗时在于 VLA 的前向传播。

#### 5. 实验数量与充分性
*   **实验规模**：收集了约 400 段演示视频用于微调。在三个复杂任务上进行了多次真实机器人测试。
*   **充分性评价**：
    *   **客观性**：通过与多种最先进的（SOTA）开源 VLA 模型对比，证明了其在长程任务上的代差优势。
    *   **公平性**：所有基准模型均在相同的数据集和硬件环境下进行了微调和评估。
    *   **消融实验**：详细分析了语义图和视觉掩码对成功率的贡献，实验逻辑严密。

#### 6. 主要结论与发现
*   **性能提升**：CodeGraphVLP 在所有非马尔可夫任务中的成功率均大幅领先，平均成功率达到 **81.7%**，而最强的历史增强基线仅为 56.7%。
*   **延迟降低**：相比于每步都调用 VLM 进行规划的方法，CodeGraphVLP 的规划延迟从秒级降低到了毫秒级（约 0.3s/step），满足闭环控制需求。
*   **鲁棒性**：视觉掩码技术显著增强了模型在杂乱环境下的泛化能力，减少了因干扰物导致的误操作。

#### 7. 优点（亮点）
*   **架构优雅**：将符号化的代码逻辑与神经网络的感知能力完美结合，解决了 VLA “记不住”和“看花眼”的问题。
*   **高效性**：通过“一次合成，多次执行”的代码规划模式，规避了大模型在线推理的高昂成本。
*   **可解释性**：语义图和 Python 代码提供了清晰的中间推理过程，便于调试和理解机器人的决策逻辑。

#### 8. 不足与局限
*   **依赖基础模型**：语义图的构建和代码的生成高度依赖于底层 VLM/LLM 的性能，若初始识别错误或代码逻辑有误，后续执行将崩溃。
*   **提示词工程**：生成可靠的代码规划器需要精心设计的 Prompt，对于极度复杂的任务，代码的泛化能力可能受限。
*   **应用限制**：目前主要集中在桌面操纵任务，对于需要大规模移动或更复杂动态环境的任务，语义图的维护成本会增加。
