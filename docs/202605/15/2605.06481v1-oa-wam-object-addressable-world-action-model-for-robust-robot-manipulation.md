## 摘要
世界动作模型（WAMs）通过联合预测场景演变和机器人动作来增强视觉-语言-动作（VLA）策略，但现有方法通常将预测的世界表示为整体图像、视频标记或全局潜变量。当指令涉及特定对象时，动作解码器很难处理这些表示，特别是在场景偏移的情况下，对象的身份往往与上下文纠缠在一起。我们提出了 OA-WAM，一种用于鲁棒机器人操作的可对象寻址世界动作模型。OA-WAM 将每一帧分解为 N+1 个槽位（slot）状态，包括一个机器人槽位和 N 个对象槽位。每个槽位包含一个持久的地址向量和一个随时间变化的内容向量，并在块因果序列中与文本、图像、本体感受和过去动作标记融合。世界头预测下一帧的槽位状态，而流匹配动作头在同一次前向传播中解码 16 步连续动作块。通过仅使用地址键路由跨槽位注意力，并在每个 Transformer 层重置地址切片，从而强制实现可寻址性，在不添加额外标记的情况下，将“操作哪个对象”与“该对象当前是什么”分离开来。OA-WAM 在 LIBERO (97.8%) 和 SimplerEnv (79.3%) 上达到了强大的 VLA 和 WAM 基准水平，在最相关的 LIBERO-Plus 几何轴上达到了最先进的性能，并在七轴综合评估中保持竞争力。因果槽位干预测试显示交换绑定余弦值为 0.87，而整体基准模型最高仅为 0.09。这些结果表明，可寻址的对象状态为场景扰动下的鲁棒世界动作建模提供了一个有效的接口。

## Abstract
World Action Models (WAMs) enhance Vision-Language-Action policies by jointly predicting scene evolution and robot actions, but existing methods usually represent the predicted world as holistic images, video tokens, or global latents. These representations are difficult for an action decoder to address when an instruction refers to a particular object, especially under scene shifts where object identity is entangled with context. We propose OA-WAM, an Object-Addressable World Action Model for robust robot manipulation. OA-WAM decomposes each frame into N+1 slot states, with one robot slot and N object slots. Each slot contains a persistent address vector and a time-varying content vector, and is fused with text, image, proprioception, and past-action tokens in a block-causal sequence. A world head predicts next-frame slot states, while a flow-matching action head decodes a 16-step continuous action chunk in the same forward pass. Addressability is enforced by routing cross-slot attention through address-only keys and resetting the address slice at every transformer layer, separating which object to act on from what that object currently is without adding extra tokens. OA-WAM matches strong VLA and WAM baselines on LIBERO (97.8%) and SimplerEnv (79.3%), reaches state-of-the-art performance on the most relevant LIBERO-Plus geometric axes, and remains competitive on the seven-axis aggregate. A causal slot-intervention test yields a swap-binding cosine of 0.87, versus at most 0.09 for holistic baselines. These results suggest that addressable object states provide an effective interface for robust world-action modeling under scene perturbations.

#### 1. 论文的核心问题与整体含义
现有的视觉-语言-动作（VLA）模型和世界动作模型（WAM）在处理机器人操作任务时，通常将场景表示为整体图像、视频标记或全局潜变量。这种“整体性”表示导致目标对象的身份（Identity）与周围上下文（背景、光照、布局）深度纠缠。当场景发生几何偏移（如相机角度变化、物体位置变动）时，模型往往无法准确定位指令中提到的特定对象，导致鲁棒性极差。论文将其归结为缺乏**对象寻址能力（Object Addressability）**，并旨在构建一个能够将“操作哪个对象”与“该对象当前状态”在架构层面解耦的模型。

#### 2. 论文提出的方法论
OA-WAM 的核心思想是将世界模型组织为一组可独立寻址的对象槽位（Slots），并通过架构约束确保动作生成直接挂钩到这些槽位上。

*   **对象槽位分解**：每一帧被分解为 $N+1$ 个槽位（1个机器人槽位，$N$个物体槽位）。每个槽位向量由两部分组成：
    *   **地址向量 ($addr_k$)**：基于语言标签和初始特征计算，在整个情节中保持不变，代表物体身份。
    *   **内容向量 ($content_t^k$)**：随时间变化，代表物体的姿态、外观等动态状态。
*   **关键技术细节**：
    *   **地址专用注意力约束**：在 Transformer 的每一层，跨槽位的注意力键（Key）投影仅读取 $addr$ 子向量，而查询（Query）和值（Value）则读取完整向量。
    *   **残差流重置**：在每个 Transformer 块之后，通过前向钩子将残差流中的地址部分重置为原始地址，防止身份信息在深层网络中发生漂移。
    *   **统一序列建模**：使用 7B 参数的 Chameleon 风格骨干网络，将文本、图像 VQ 码、本体感受、过去动作和对象槽位组合成块因果序列。
*   **预测头**：
    *   **世界头**：回归下一帧每个槽位的内容和姿态。
    *   **动作头**：采用流匹配（Flow-matching）解码 16 步连续动作块。

论文在三个主要的机器人操作基准测试中进行了评估：
*   **LIBERO**：评估分布内（In-distribution）的成功率，包括 Spatial、Object、Goal 和 Long 四个子集。
*   **SimplerEnv (WidowX/Bridge)**：评估模拟到真实的视觉匹配能力。
*   **LIBERO-Plus**：核心鲁棒性测试，包含相机偏移、机器人初始状态、布局、光照、背景、语言描述和传感器噪声 7 个扰动轴。
*   **对比方法**：包括 SOTA VLA 模型（如 OpenVLA, $\pi_0$, $\pi_{0.5}$, InternVLA-M1）和 WAM 模型（如 VLA-JEPA, Cosmos-Policy, WorldVLA）。

#### 5. 实验数量与充分性
实验设计非常充分且具有针对性：

*   **因果干预测试**：进行“槽位交换”实验，通过交换两个物体的地址向量观察机器人轨迹的偏转，这是验证模型内部机制的强有力手段。

#### 6. 主要结论与发现
*   **鲁棒性飞跃**：在 LIBERO-Plus 的几何扰动轴（相机、机器人状态、布局）上，OA-WAM 显著超过了 $\pi_{0.5}$ 等强基准，几何平均成功率提升了 4.8%。
*   **寻址验证**：在因果干预测试中，OA-WAM 的交换绑定余弦值达到 0.87，而所有整体性基准模型均低于 0.09，证明了其确实实现了对象级绑定。
*   **分布内保持**：在提升鲁棒性的同时，OA-WAM 在标准 LIBERO 任务上依然保持了 97.8% 的极高成功率，证明该架构不会损害基础性能。

#### 7. 优点
*   **架构创新**：通过简单的张量级操作（掩码和重置）实现了复杂的对象寻址功能，无需引入大量额外参数。
*   **可验证性**：不同于大多数“黑盒”VLA 模型，OA-WAM 的决策逻辑（即它在看哪个物体）可以通过地址空间进行直接干预和验证。
*   **世界模型集成**：将对象级动力学预测与动作生成结合，提供了更强的时空监督。

#### 8. 不足与局限
*   **感知瓶颈**：模型高度依赖上游的感知系统（SAM3, DINOv3）。在传感器噪声轴上表现较差（-17.1%），主要是因为图像噪声导致槽位提取失败，而非策略本身失效。
*   **实时性挑战**：感知部分的延迟（138ms）远高于骨干网络推理，对于高频闭环控制可能存在压力。
*   **模拟器依赖**：目前所有验证均在模拟器中完成，尚未展示在真实机器人上的部署效果（尽管 SimplerEnv 提供了一定参考）。
*   **假设限制**：模型假设物体间是弱耦合的，在处理物体间有复杂物理遮挡或纠缠的场景时可能存在局限。
