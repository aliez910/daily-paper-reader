## 摘要
跨任务泛化是开放世界机器人操作中的核心挑战，其关键在于从已见任务中提取可迁移的操作知识。最近的上下文学习方法利用已见任务的演示，在不更新参数的情况下为未见任务生成动作。然而，现有方法仅提供低级连续动作序列作为上下文，未能捕捉到可组合的技能知识，导致模型退化为表层的轨迹模仿。我们提出了“分解与重组”（Decompose and Recompose），这是一个使用原子技能-动作对作为中间表示的技能推理框架。我们的方法将已见演示分解为可解释的技能-动作对齐，使模型能够通过组合推理为未见任务重组这些技能。具体而言，我们通过视觉语义检索结合规划代理的技能序列，构建了一个任务自适应的动态演示库，并辅以覆盖感知的静态库以填补缺失的技能模式。这些共同产生了技能全面的演示，明确地激发了关于技能组合和执行顺序的组合推理。在 AGNOSTOS 基准测试和真实世界环境中的实验验证了我们方法的零样本跨任务泛化能力。

## Abstract
Cross-task generalization is a core challenge in open-world robotic manipulation, and the key lies in extracting transferable manipulation knowledge from seen tasks. Recent in-context learning approaches leverage seen task demonstrations to generate actions for unseen tasks without parameter updates. However, existing methods provide only low-level continuous action sequences as context, failing to capture composable skill knowledge and causing models to degenerate into superficial trajectory imitation. We propose Decompose and Recompose, a skill reasoning framework using atomic skill-action pairs as intermediate representations. Our approach decomposes seen demonstrations into interpretable skill--action alignments, enabling the model to recompose these skills for unseen tasks through compositional reasoning. Specifically, we construct a task-adaptive dynamic demonstration library via visual-semantic retrieval combined with skill sequences from a planning agent, complemented by a coverage-aware static library to fill missing skill patterns. Together, these yield skill-comprehensive demonstrations that explicitly elicit compositional reasoning for skill composition and execution ordering. Experiments on the AGNOSTOS benchmark and real-world environments validate our method's zero-shot cross-task generalization capability.

*   **核心问题**：现有的上下文学习（In-Context Learning, ICL）方法（如 X-ICM）仅提供低级的数值动作序列作为上下文。这种方式缺乏显式的技能语义，导致大语言模型（LLM）往往只是在进行表面的轨迹模式匹配（Trajectory Pattern Matching），而非真正的技能推理，难以应对复杂的技能重组需求。

### 2. 论文提出的方法论
该论文提出了 **Decompose and Recompose (DnR)** 框架，核心流程如下：
*   **原子技能收集（Decompose）**：
    *   利用关键帧提取技术（基于夹持器状态变化和运动停止准则）将长演示分割成段。
    *   使用视觉语言模型（VLM，如 Qwen2.5-VL）结合夹持器约束，为每个动作段标注“动词[物体]”格式的原子技能标签（如 `Grasp[cube]`），形成**技能-动作对齐数据**。
*   **双库演示检索（Retrieval）**：
    *   **动态演示库**：结合视觉语义相似度（DINOv3 特征）和规划相似度（利用规划代理预测目标任务的技能序列，计算 Jaccard 相似度）来检索最相关的演示。
    *   **覆盖感知静态库**：为了防止动态检索遗漏关键技能，构建了一个离线静态库。利用 **IDF（逆文档频率）权重**和长度正则化评分，贪婪地补充动态库中缺失的技能模式，确保上下文的技能完备性。

    *   将检索到的“指令-观察-技能序列-动作序列”格式化为 Prompt。
    *   LLM 在推理时参考技能标签理解因果逻辑，但最终仅输出可执行的低级动作序列，确保与机器人控制器的直接兼容。

*   **技能推理的价值**：实验证明，显式的技能标签比纯数值动作更能激发 LLM 的组合推理能力，尤其在处理从未见过的技能组合时。

*   **模型能力正相关**：跨任务泛化表现高度依赖于底层 LLM 的推理能力和视觉编码器的语义提取能力。

### 7. 优点（亮点）
*   **中间表示创新**：引入“原子技能-动作对”作为桥梁，解决了低级动作缺乏语义和高级指令过于抽象之间的断层。
*   **免训练架构**：无需针对特定任务分布进行微调，具有极强的即插即用性和跨领域迁移潜力。

### 8. 不足与局限
*   **3D 空间推理不足**：真实世界实验显示，模型在处理精细的 3D 空间关系（如物体中心定位）和 6-DoF 姿态（如倾斜抓取导致对齐失败）时仍存在局限。

*   **实时性挑战**：依赖大型 LLM 的多轮推理和长上下文处理，可能在对实时性要求极高的动态操作场景中受限。
