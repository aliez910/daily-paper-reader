## 摘要
通用机器人策略日益受益于大规模预训练，但仅靠离线数据不足以实现稳健的现实世界部署。部署后的机器人会遇到分布偏移、长尾失败、任务变化以及人类纠错机会，而固定的演示数据集无法完全捕捉这些情况。我们提出了“边部署边学习”（Learning While Deploying, LWD），这是一个车队级的离线到在线强化学习框架，用于通用视觉-语言-动作（VLA）策略的持续后训练。LWD 从预训练的 VLA 策略开始，通过利用在机器人车队中收集的自主运行和人类干预，闭合了部署、共享物理经验、策略改进和重新部署之间的循环。为了稳定从异构、稀疏奖励的车队数据中学习的过程，LWD 将用于稳健价值估计的分布隐式价值学习（DIVL）与通过伴随匹配进行 Q 学习（QAM）相结合，以便在基于流的 VLA 动作生成器中进行策略提取。我们在由 16 台双臂机器人组成的车队上验证了 LWD，涵盖了八项现实世界的操纵任务，包括语义杂货补货和时长 3-5 分钟的长程任务。随着车队经验的积累，单一通用策略不断改进，平均成功率达到 95%，其中长程任务的提升最为显著。

## Abstract
Generalist robot policies increasingly benefit from large-scale pretraining, but offline data alone is insufficient for robust real-world deployment. Deployed robots encounter distribution shifts, long-tail failures, task variations, and human correction opportunities that fixed demonstration datasets cannot fully capture. We present Learning While Deploying (LWD), a fleet-scale offline-to-online reinforcement learning framework for continual post-training of generalist Vision-Language-Action (VLA) policies. Starting from a pretrained VLA policy, LWD closes the loop between deployment, shared physical experience, policy improvement, and redeployment by using autonomous rollouts and human interventions collected across a robot fleet. To stabilize learning from heterogeneous, sparse-reward fleet data, LWD combines Distributional Implicit Value Learning (DIVL) for robust value estimation with Q-learning via Adjoint Matching (QAM) for policy extraction in flow-based VLA action generators. We validate LWD on a fleet of 16 dual-arm robots across eight real-world manipulation tasks, including semantic grocery restocking and 3--5 minute long-horizon tasks. A single generalist policy improves as fleet experience accumulates, reaching an average success rate of 95%, with the largest gains on long-horizon tasks.

### 论文总结：边部署边学习（Learning While Deploying, LWD）

#### 1. 核心问题与整体含义（研究动机和背景）
论文针对通用机器人策略（如 VLA 模型）在现实部署中面临的挑战：**离线预训练数据无法覆盖所有实际场景**。机器人部署后会遇到分布偏移（Distribution Shift）、长尾失败案例、任务变体以及人类干预机会。
**研究动机**是将“部署”从训练的终点转变为持续改进的起点。通过构建一个“数据飞轮”，利用机器人机群（Fleet）在真实环境中的自主尝试和人类干预数据，实现策略的持续在线进化，使其从“通用”走向“部署就绪”。

#### 2. 方法论：核心思想与关键技术
LWD 提出了一个车队级的离线到在线强化学习框架，包含两个核心算法组件：

*   **分布隐式价值学习（DIVL, Distributional Implicit Value Learning）：**
    *   **核心思想：** 针对机群数据异构、奖励稀疏且包含多种结果（成功、失败、干预）的特点，DIVL 放弃了传统的标量价值估计，转而学习回报的**分布**。
    *   **技术细节：** 采用分类离散化（Categorical Discretization）表示价值分布。利用 IQL 的隐式改进原则，通过提取分布的 $\tau$-分位数（Quantile）作为 TD 目标值，避免了对分布外动作的查询。
    *   **自适应 $\tau$ 策略：** 根据分布的熵（不确定性）动态调整 $\tau$ 值：在不确定性高时更保守，在确定性高时更乐观。
*   **伴随匹配 Q 学习（QAM, Q-learning with Adjoint Matching）：**
    *   **核心思想：** 解决基于流匹配（Flow Matching）的 VLA 策略难以通过链式法则直接反向传播 Critic 梯度的问题。
    *   **技术细节：** QAM 将 Critic 提供的动作梯度转化为流模型路径上的每一步监督信号，通过求解伴随方程实现稳定且高效的策略提取，保留了生成式动作建模的表达力。
*   **训练流程：** 分为离线预训练（在包含演示、历史运行和失败尝试的混合缓冲区上训练）和在线持续训练（机群实时回传数据，与离线数据混合微调）。

#### 3. 实验设计
*   **硬件平台：** 16 台 Agibot G1 双臂协作机器人。
*   **实验场景（8 项任务）：**
    *   **语义杂货补货（4 项）：** 平面货架补货、错位纠正、冰箱补货（含开门）、冷柜补货。侧重语义理解和泛化。
    *   **长程操纵任务（4 项）：** 功夫茶冲泡、鸡尾酒调制、果汁制作、鞋盒打包。每项任务持续 3-5 分钟，包含 5-8 个子步骤。
*   **对比方法（Baselines）：**
    *   **SFT：** 仅使用人类演示数据的监督微调。
    *   **RECAP：** 一种基于优势加权的离线 RL 改进方法。
    *   **HG-DAgger：** 交互式模仿学习，利用人类干预数据。
    *   **LWD (Offline)：** 仅进行离线 RL 阶段的 LWD。

#### 4. 资源与算力
*   **模型架构：** 策略网络基于 PaliGemma（Gemma-2B + SigLIP），Critic 和价值网络基于 Gemma 3-270M 和 SigLIP-So400M。
*   **数据规模：** 离线缓冲区包含 **652.5 小时** 的混合数据。
*   **在线训练：** 每个实验分配 **4 小时** 的墙钟时间，16 台机器人总计贡献约 **60 小时** 的在线交互数据。
*   **系统架构：** 使用分布式 Actor-Learner 架构，Learner 采用多机 SPMD JAX 程序运行。文中未明确具体 GPU 型号，但提到了多机分布式训练。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 16 台机器人的大规模部署，这在真实机器人 RL 研究中属于领先水平。
*   **消融实验：** 针对 DIVL（对比标量回归）和自适应 $\tau$ 策略进行了专门的消融研究。
*   **客观性：** 实验涵盖了短程语义任务和极具挑战性的长程任务，并使用了人类评估员根据预定义准则进行评分，评估指标包括成功率和循环时间（Cycle Time）。实验设计较为全面且贴近实战。

#### 6 主要结论与发现
*   **性能提升：** LWD 在所有任务上均优于基准方法，平均成功率达到 **95%**。
*   **长程任务优势：** 在长程任务中提升最显著（从 SFT 的 0.68 提升至 0.91），证明了 RL 在处理复合误差和稀疏奖励传播方面的优越性。
*   **效率优化：** 相比 SFT，LWD 显著缩短了任务完成的平均循环时间（减少约 23.75 秒），动作更加果断。
*   **价值函数有效性：** 可视化证明 DIVL 学习到的价值函数能够准确追踪长程任务的进度。

#### 7. 优点
*   **闭环进化：** 真正实现了“边部署边学习”，将机群经验转化为策略提升。
*   **算法稳健：** 结合 DIVL 和 QAM，解决了 VLA 模型在离线到在线转换过程中的不稳定性。
*   **任务复杂度：** 成功挑战了如“功夫茶”等极高难度的长程、接触密集型任务。
*   **工程实现：** 展示了支持大规模机器人机群实时训练的分布式数据基础设施。

#### 8. 不足与局限
*   **更新频率：** 目前的在线学习采用固定的实时调度，对于更大规模或更长期的持续学习可能不是最优的。
*   **推理能力：** 长程任务仍依赖简单的语言指令，对于需要复杂逻辑推理或多步纠错的任务，VLA 的推理能力仍有提升空间。
*   **安全性：** 框架目前未显式建模执行安全性（Safety），在无约束的真实家庭环境中部署可能存在风险。
*   **人工依赖：** 虽然是自主收集，但仍需要人类进行最终的成功/失败标注或必要的干预。
