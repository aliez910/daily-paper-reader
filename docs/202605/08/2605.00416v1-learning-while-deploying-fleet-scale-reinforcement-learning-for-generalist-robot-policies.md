## 摘要
通用机器人策略日益受益于大规模预训练，但仅靠离线数据不足以实现稳健的现实世界部署。部署后的机器人会遇到分布偏移、长尾失败、任务变化以及人类纠错机会，而固定的演示数据集无法完全捕捉这些情况。我们提出了“边部署边学习”（Learning While Deploying, LWD），这是一个车队级的离线到在线强化学习框架，用于通用视觉-语言-动作（VLA）策略的持续后训练。LWD 从预训练的 VLA 策略开始，通过利用在机器人车队中收集的自主展开和人类干预，闭合了部署、共享物理经验、策略改进和重新部署之间的循环。为了从异构、稀疏奖励的车队数据中稳定学习，LWD 将用于稳健价值估计的分布隐式价值学习（DIVL）与通过伴随匹配进行 Q 学习（QAM）相结合，用于流式 VLA 动作生成器中的策略提取。我们在由 16 台双臂机器人组成的车队上验证了 LWD，涵盖了八项现实世界操纵任务，包括语义杂货补货和 3-5 分钟的长程任务。随着车队经验的积累，单个通用策略不断改进，平均成功率达到 95%，其中在长程任务上的提升最为显著。

## Abstract
Generalist robot policies increasingly benefit from large-scale pretraining, but offline data alone is insufficient for robust real-world deployment. Deployed robots encounter distribution shifts, long-tail failures, task variations, and human correction opportunities that fixed demonstration datasets cannot fully capture. We present Learning While Deploying (LWD), a fleet-scale offline-to-online reinforcement learning framework for continual post-training of generalist Vision-Language-Action (VLA) policies. Starting from a pretrained VLA policy, LWD closes the loop between deployment, shared physical experience, policy improvement, and redeployment by using autonomous rollouts and human interventions collected across a robot fleet. To stabilize learning from heterogeneous, sparse-reward fleet data, LWD combines Distributional Implicit Value Learning (DIVL) for robust value estimation with Q-learning via Adjoint Matching (QAM) for policy extraction in flow-based VLA action generators. We validate LWD on a fleet of 16 dual-arm robots across eight real-world manipulation tasks, including semantic grocery restocking and 3--5 minute long-horizon tasks. A single generalist policy improves as fleet experience accumulates, reaching an average success rate of 95%, with the largest gains on long-horizon tasks.

### 论文总结：Learning while Deploying (LWD)

#### 1. 核心问题与整体含义
论文针对通用机器人策略（如 VLA 模型）在现实部署中面临的挑战：**离线预训练的局限性**。尽管大规模离线数据能提供基础能力，但无法覆盖部署时的分布偏移（Distribution Shift）、长尾失败案例、环境变化及人类实时纠错。
**核心动机**是将“部署”从训练的终点转变为持续改进的起点。通过构建一个“数据飞轮”，利用机器人车队（Fleet-scale）在真实环境中的自主交互和人类干预数据，实现策略的在线持续进化。

#### 2. 方法论
LWD 提出了一个离线到在线（Offline-to-Online）的强化学习框架，包含两个关键技术组件：

*   **分布隐式价值学习 (DIVL, Distributional Implicit Value Learning)：**
    *   **核心思想：** 改进了隐式 Q 学习（IQL）。由于机群数据具有异构性（不同策略版本、不同场景、稀疏奖励），同一状态-动作对的收益可能呈多峰分布。
    *   **技术细节：** DIVL 不学习标量价值，而是学习收益的**分类分布（Categorical Distribution）**。它通过最小化负对数似然来拟合分布，并提取特定分位数（$\tau$-quantile）作为 TD 目标的引导值。
    *   **自适应策略：** 引入基于分布熵（Entropy）的自适应 $\tau$ 调节机制。当价值估计不确定（熵高）时，采取保守的 $\tau$；当确定时，采取激进的 $\tau$ 以促进改进。
*   **伴随匹配 Q 学习 (QAM, Q-learning with Adjoint Matching)：**
    *   **核心思想：** 解决流匹配（Flow Matching）策略的优化问题。直接对多步去噪过程进行梯度回传计算量大且不稳定。
    *   **技术细节：** QAM 将价值函数的动作梯度（$\nabla_a Q$）转化为流模型每一步的局部回归目标。通过求解伴随方程，将终端奖励信号稳定地注入到 VLA 的动作生成头中。
*   **算法流程：** 
    1. 离线阶段：在混合数据集（演示、历史尝试、失败回放）上预训练策略、评论家（Critic）和价值模型。
    2. 在线阶段：将策略部署至机器人车队，异步收集自主尝试和人类干预数据，存入在线缓存。
    3. 持续更新：利用离线与在线混合数据，通过 DIVL 和 QAM 持续更新共享策略并重新部署。

#### 3. 实验设计
*   **场景与任务：** 在 **16 台 Agibot G1 双臂机器人**上进行了 8 项真实任务测试：
    *   **杂货补货任务（4项）：** 平面货架补货、错位纠正、冷冻柜补货（含开门）、开放式冷柜补货。侧重语义泛化。
    *   **长程操纵任务（4项）：** 功夫茶（泡茶、分茶）、制作鸡尾酒（摇晃、倾倒）、榨汁（切水果、操作机器）、装鞋盒。任务时长达 3-5 分钟，步骤复杂。
*   **对比基线 (Benchmarks)：**
    *   **SFT (Supervised Fine-Tuning)：** 仅使用人类演示数据的标准流匹配策略。
    *   **RECAP：** 一种基于优势加权的离线 RL 改进方法。
    *   **HG-DAgger：** 经典的交互式模仿学习方法，利用人类干预数据。
    *   **LWD (Offline)：** 仅完成离线 RL 阶段的本方法。

#### 4. 资源与算力
*   **模型架构：** 
    *   **VLM Backbone:** 基于 Gemma 3-270M-IT 和 SigLIP-So400M。
    *   **Policy:** 基于 PaliGemma (Gemma-2B) 和 300M 参数的动作专家头。

*   **实验规模：** 涵盖了 8 个差异化明显的真实任务，使用了 16 台机器人的大规模车队，这在同类研究中属于较高规格。
*   **消融实验：** 
    *   对比了 **DIVL vs. 标量回归（Expectile Regression）**，证明了分布学习在长程任务上的显著优势。

#### 6. 主要结论与发现
*   **性能提升：** LWD 在所有任务上均优于基线，平均成功率达到 **95%**。
*   **长程任务优势：** 在长程任务（如功夫茶）中，LWD 的提升最为显著（相比 SFT 提升了约 20-30%），证明了 RL 在处理多步动态规划和错误恢复方面的能力。
*   **效率优化：** 相比 SFT，LWD 显著减少了任务完成的平均循环时间（减少约 23.75 秒），策略执行更加果断，减少了犹豫和重试。
*   **数据飞轮效应：** 随着在线经验的积累，单一个通用策略在多个任务上同步进化，证明了车队级学习的可扩展性。

#### 7. 优点
*   **闭环进化：** 真正实现了“边部署边学习”，将失败案例转化为学习信号。
*   **算法创新：** DIVL 有效解决了异构机器人数据的多峰价值分布问题；QAM 解决了生成式流策略难以通过 RL 稳定微调的难题。
*   **系统工程：** 构建了支持 16 台机器人异步数据上传、版本化快照和策略分发的工业级基础设施。

#### 8. 不足与局限
*   **安全性缺失：** 框架目前未显式建模执行安全性（Safety），在真实复杂环境中可能存在碰撞风险。
*   **更新调度简单：** 在线学习采用固定的实时调度，对于更大规模（如成千上万台）的部署，可能需要更复杂的持续学习策略。
*   **语言理解深度：** 长程任务仍依赖相对简单的短指令，对于需要复杂逻辑推理或多轮对话的任务支持有限。
*   **算力细节不透明：** 缺乏具体的硬件资源消耗数据，难以评估该方案的部署成本。
