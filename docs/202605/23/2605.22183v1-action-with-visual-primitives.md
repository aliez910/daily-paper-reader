## 摘要
视觉-语言-动作（VLA）模型已成为通用机器人操作的一种有前景的范式。当前架构中的常见设计是将语言指令和视觉观察映射到单次前向传播中的动作。虽然概念上简单，但这种表述将指令理解、空间场景理解和运动控制纠缠在单一学习目标中。因此，动作专家必须隐式地重新学习预训练VLM中已经存在的认知和感知能力，这可能限制学习效率和泛化能力。我们提出AVP（基于视觉基元的动作），这是一种端到端架构，实现了这种以视觉基元为中心的接口：VLM推断下一阶段目标并发出视觉基元标记，这些标记调节流匹配动作专家，其监督来自末端执行器运动学。在通用拾取和放置任务上的实际机器人实验表明，AVP相比pi_0.5将成功率提高了27.61%，并优于其他近期方法，在数据效率、空间组合泛化和对象级迁移方面具有一致的增益。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for generalist robotic manipulation. A common design in current architectures maps language instructions and visual observations to actions in a single forward pass. While conceptually simple, this formulation entangles instruction comprehension, spatial scene understanding, and motor control within a single learning objective. As a result, the action expert must implicitly relearn cognitive and perceptual capabilities already present in the pretrained VLM, which can limit both learning efficiency and generalization. We introduce AVP (Action with Visual Primitives), an end-to-end architecture that implements this visual-primitive-centric interface: the VLM infers the next-stage target and emits visual-primitive tokens that condition a flow-matching action expert, with supervision derived from end-effector kinematics. Real-robot experiments on general pick-and-place tasks show that AVP improves the success rate by 27.61% over pi_0.5 and outperforms other recent methods, with consistent gains in data efficiency, spatial-compositional generalization, and object-level transfer.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有视觉-语言-动作（VLA）模型将语言指令理解、空间场景感知与运动控制纠缠在同一个学习目标中。动作专家（action expert）必须隐式地重新学习预训练VLM已经具备的认知和感知能力，导致学习效率低、泛化能力弱（尤其当对象、布局或环境变化时）。
- **研究动机**：如何合理划分VLM与动作专家之间的职责——理想情况下，VLM负责“做什么”和“在哪里”，动作专家专注于“怎么做”。现有方法（如π0.5通过语言子任务、π0.7通过子目标图像、Point-VLA通过外部模型生成视觉提示）各有局限：语言接口难以传达细粒度空间区分，密集像素预测仍使动作专家需重新发现关键信息，外部视觉提示引入多阶段流水线，增加延迟和错误传播。
- **整体含义**：提出一种端到端架构AVP，在VLM与动作专家之间建立以视觉基元（visual primitives）为核心的通迅接口，实现职责明确的分工，从而提升学习效率、数据效率、空间组合泛化和对象级迁移能力。

## 2. 论文提出的方法论

- **核心思想**：将视觉基元（如点、边界框、掩码等）作为VLM与动作专家之间的**显式通信媒质**。VLM负责从语言指令和视觉观察中推断下一阶段目标，并生成视觉基元token；这些token被投影回视觉token空间，作为条件输入到流匹配（flow-matching）动作专家，由其生成机器人动作。监督信号直接来自末端执行器运动学（无需外部感知模块或人工标注）。
- **关键技术细节**：
  - **三组件架构**：预训练VLM、自回归解码器（预测视觉基元）、流匹配动作专家。
  - **视觉基元预测**：给定语言指令`l`和当前观测`o_t`，VLM产生多模态上下文token，视觉基元解码器`Dψ`自回归预测离散化的视觉基元`p_t`（表示下一阶段子任务及其空间目标）。
  - **投影与融合**：将预测的基元投影到视觉token空间得到`z_vp_t`，与原始多模态token融合形成增强的多模态表示`z_aug_t`，供动作专家用于动作预测`a_{t:t+h}`。
  - **训练损失**：总损失`L = L_act + λL_vp`，其中`L_vp`是基元预测的交叉熵损失，`L_act`是标准动作预测损失。`λ`平衡两项。
  - **基元监督来源**：通过手眼标定将末端执行器3D位置投影到2D图像平面，离散化为网格标签，自动从机器人轨迹中提取，无需人工标注。
- **算法流程**（文字说明）：
  1. 输入语言指令`l`、多视角观测`o_t`、机器人本体状态`st`。
  2. VLM解析当前子任务，输出上下文token。
  3. 自回归解码器从上下文中预测离散化视觉基元`p_t`（如目标点/框）。
  4. 将`p_t`投影到视觉token空间，生成`z_vp_t`，与原始token融合。
  5. 流匹配动作专家以`z_aug_t`和`st`为条件，预测未来动作序列`a_{t:t+h}`。
  6. 训练时同时优化基元预测损失和动作预测损失。

## 6. 论文的主要结论与发现

- AVP在所有三个任务上**一致优于**π0.5基线，总体成功率提升27.61%。
- **空间组合泛化**：训练数据中只有间接过渡（A→C→B），未见直接过渡（A→B）。π0.5完全失败（0/8成功），而AVP在直接过渡上达到8/8成功。
- **跨域零样本泛化**：仅在中国象棋数据上训练，AVP能在未见物体、不同背景和指令协议下成功操作（多数场景成功率≥7/8），而π0.5严重退化。
- **消融**：视觉基元类型选择影响性能，框+掩码+记忆机制达到最佳（91%平均成功率），无基元仅有78%。
- **可扩展性**：基元监督来自末端执行器运动学，自动获取；基元形式轻量（点/框），易于扩展到时序（记忆基元、顺序基元），边际成本低。

## 7. 优点

- **明确的职责分解**：VLM负责认知和空间定位，动作专家只负责运动执行，避免了隐式学习复杂映射，提高数据效率和泛化。
- **端到端训练**：内部生成基元，无需外部检测器或分段模型，避免多阶段流水线的延迟和错误传播。
- **监督自动获取**：基元标签从机器人固有运动学自动推导，只需一次手眼标定，成本与标准模仿学习相近，易于扩展到新平台。
- **方法通用性强**：视觉基元形式灵活（点、框、掩码、记忆/顺序基元等），能自然融入长时依赖任务。
- **实验全面**：涵盖多任务、空间组合、跨域零样本泛化、组件消融，验证了核心设计有效性和泛化能力。
