## 摘要
基于便携式演示界面的移动模仿学习面临两个耦合的瓶颈：受运动污染的动作标签以及连续移动基座上由推理引起的执行延迟。近期腕戴式界面降低了桌面数据收集的成本，但单一的腕部视角无法捕捉基站导航所需的全局上下文。添加一个身体安装的摄像头会将人类的行走与手部运动纠缠在一起。同时，生成式策略引入了数百毫秒的推理延迟，在此期间基座会越过预测的航点，迫使在动作拼接处进行向后修正。本文提出了Mobile UMI，一个无需硬件的演示框架，通过三个组成部分解决这两个问题。首先，双摄像头捕捉系统在没有任何机器人的情况下记录以胸部为中心的全局上下文和以腕部为中心的局部交互。其次，基于ChArUco的一次性空间锚点统一了胸部和手部的视觉惯性框架；然后手部姿势相对于胸部重新表达，以提取解耦的SE(3)操作和SE(2)基座轨迹。第三，异步的滚动时域执行器执行在线状态匹配：每个生成的动作块与当前物理姿态重新对齐，使得过期的航点在执行前被丢弃。整个系统在四个长时域家庭任务上进行评估，每个任务100次试验的平均成功率达到83.8%。与ACT和扩散策略的对比实验表明，仅使用胸部相对标签就能缩小大部分差距；在线状态匹配补足了剩余部分。这些结果表明，在测试条件下，对于移动模仿学习，显式的运动学分解结合状态级延迟对齐提供了一种有效的解决方案，而无需对底层策略类进行架构变更。

## Abstract
Mobile imitation learning on portable demonstration interfaces faces two coupled bottlenecks: locomotion-contaminated action labels and inference-induced execution latency on a continuously moving base. Recent wrist-mounted interfaces lower the cost of tabletop data collection, yet a single wrist view does not capture the global context required for base navigation. Adding a body-mounted camera entangles human walking with hand motion. Meanwhile, generative policies introduce hundreds of milliseconds of inference latency, during which the base advances past predicted waypoints, forcing backward corrections at action splices. This paper presents Mobile UMI, a hardware-free demonstration framework that addresses both gaps through three components. First, a dual-camera capture system records chest-centric global context and wrist-centric local interaction without any robot present. Second, a one-shot ChArUco-based spatial anchor unifies the chest and hand visual-inertial frames; the hand pose is then re-expressed relative to the chest to extract decoupled SE(3) manipulation and SE(2) base trajectories. Third, an asynchronous receding-horizon executor performs online state matching: each generated action chunk is realigned with the current physical pose so that expired waypoints are discarded before execution. The full system is evaluated on four long-horizon household tasks, achieving an average success rate of 83.8% over 100 trials per task. Controlled comparisons against ACT and Diffusion Policy show that the chest-relative label alone closes much of the gap; online state matching closes the remainder. These results indicate that, for mobile imitation learning under the tested conditions, explicit kinematic factorization combined with state-level latency alignment provides an effective solution without requiring architectural changes to the underlying policy class.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：移动操作中的模仿学习面临两个耦合的瓶颈：
  1. **动作标签受运动污染**：使用便携式演示接口（如手腕相机）收集数据时，若额外添加胸部相机获取全局上下文，会导致操作者的行走位移与手部运动在全局坐标系中纠缠，策略无法区分有意的操作和步行引起的平移。
  2. **生成式策略推理延迟导致执行不匹配**：如扩散策略等生成模型通常需要数百毫秒的推理延迟，在连续移动的基座上，机器人在推理期间已经向前移动，导致预测的第一个航点落在当前位姿后方，被迫执行向后修正，产生可见的抖动。

- **背景**：现有工作（如UMI）在静态桌面任务中降低了数据收集成本，但扩展到移动操作时存在上述问题。移动操作需要基座导航与手臂协调，传统遥操作昂贵且受限于物理机器人存在。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
提出 **Mobile UMI**，一个无机器人（hardware-free）的演示框架，通过三个组件解耦运动和执行延迟：
1. 双摄像头系统（胸部+手腕）分别采集全局导航上下文和局部操作细节。
2. 基于ChArUco的一次性空间锚点，统一胸部与手部的视觉惯性帧，将手部位姿重新表达为相对于胸部，从而提取 **解耦的SE(3)操作轨迹** 和 **SE(2)基座轨迹**。
3. 异步滚动时域执行器，在每个动作拼接处进行 **在线状态匹配**，丢弃过期航点，避免执行陈旧命令。

### 关键技术细节

- **数据采集系统**：操作者佩戴胸部相机（Orbbec Gemini 2L，RGB+IMU），手持相机-夹爪模块（同型号相机）。利用视觉惯性里程计（VIO, OpenVINS）获取各自轨迹。通过一个静态ChArUco标定板作为空间锚点，求解胸部与手部世界帧之间的变换。

- **解耦动作表示**：
  - 基座导航：胸部轨迹的SE(2)投影（平面平移+朝向）。
  - 手臂操作：相对胸部的手部SE(3)增量变化，去除了行走平移。
  - 夹爪控制：指尖ArUco标记距离映射到[0,1]。
  - 额外将全方向行走的胸部轨迹投影到非完整约束的差分驱动基座上（丢弃侧向速度分量）。

- **跨视角条件扩散策略**：
  - 输入：胸部RGB图像、手部RGB图像、当前位姿状态、历史一步动作。
  - 输出：11维动作向量（基座3维、手臂7维、夹爪1维），预测未来16步的动作块，执行前8步。
  - 使用共享权重的ResNet-18提取视觉特征，结合位姿状态形成条件向量。
  - 采用1D时序U-Net去噪骨干，通过FiLM层注入扩散步长嵌入。
  - 训练时扩散步数K=100，推理时采用DDIM减少到10步，使延迟低于100ms（RTX 4070上）。

- **异步滚动时域执行与延迟补偿**：
  - 异步执行：当前动作块执行时，后台基于最新观测生成新块（预测16步，执行8步）。
  - 在线状态匹配：收到新动作块后，前向积分预测未来状态序列；将当前实测状态（编码器+里程计）与预测状态进行加权最近邻匹配；丢弃匹配索引之前的所有过期航点，从对齐点开始执行；加权距离包括基座SE(2)差异、手臂平移和旋转差异、夹爪开度差异。

## 3. 实验设计

- **场景与任务**：四个长时家庭任务，操作在3×3m地面区域，包含两张桌子（L型布局）：
  1. **茶包放置（Tea Bag Placement）**：导航到桌子，抓取茶包，运送并放入杯子。
  2. **关灯（Turn off the light）**：导航到墙壁开关并将其关闭。
  3. **书本放置（Book placement）**：拿起书本放到目标位置。
  4. **家庭操作（Household Manipulation）**：结合导航和多步骤物体操作的综合任务。

- **数据收集**：每个任务收集的演示数量分别为50、100、100、200次（同一操作者，实验室环境，固定光照500 lux，物体位姿人工重置至名义位置）。

- **基准方法与对比**：
  - **ACT**（Action Chunking Transformer，4层transformer编码器-解码器，chunk size 16，KL权重10）。
  - **Diffusion Policy**（复用Mobile UMI的视觉编码器、FiLM条件和1D U-Net骨干）。
  - 每种baseline分别训练 **全局标签** 和 **胸部相对标签** 两类动作表示。
  - 所有方法共享相同演示、观测和训练计划。

- **评价指标**：100次独立试验的成功率（完整操作序列无误、120秒内完成），每次试验起始基座位姿有±10cm平移和±15°朝向随机扰动。

- **消融实验**：
  - 去除解耦表示（w/o Decoupled Representation）：保持状态匹配但使用全局标签。
  - 去除延迟补偿（w/o Latency Compensation）：使用解耦标签但禁用状态匹配。

## 4. 资源与算力

- **训练硬件**：单张 **NVIDIA RTX 4070（12GB）** GPU。
- **优化器**：AdamW，初始学习率1e-4，权重衰减1e-6，线性预热500步后余弦退火。
- **训练时长**：每任务训练 **20 epochs**（batch size=64），论文未给出总训练时间。
- **推理**：DDIM采样10步，端到端总延迟平均142ms（图像获取33ms、推理87ms、命令下发22ms）。

- **试验数量**：每个任务进行 **100次独立试验**，四个任务共400次。
- **对比基线**：两种流行方法（ACT、Diffusion Policy）各带两种标签，共4个基线变体，加上Mobile UMI完整模型及两个消融，共7行对比。消融实验充分验证了两个主要贡献的作用。
- **公平性**：所有方法使用相同的演示数据、观测和训练计划；对比中明确分离了表示（标签）和策略类的效果；提出了相同的滚动时域执行框架。
- **充分性**：
  - 四项任务覆盖了不同难度的长时操作。
  - 消融实验清晰展示了每个组件的重要性。
  - 但数据集规模较小（50~200次演示），且仅在单一实验室环境（固定光照、固定物体位置模式）中测试，未涉及多场景泛化或真实家庭环境变化（如光照、物体布局变化）。此外，未比较其他延迟补偿方法（如模型预测控制、动态模型）或更复杂的策略架构（如基于Transformer的大模型）。

## 6. 论文的主要结论与发现

- Mobile UMI在四个任务上平均成功率 **83.8%**，显著优于ACT（51.5%使用胸部相对标签）和Diffusion Policy（66.8%使用胸部相对标签）。
- 将动作标签从全局坐标改为 **胸部相对表示**（解耦行走和操作）是性能提升的主要来源：ACT提升9个百分点，Diffusion Policy提升14.3个百分点。
- **在线状态匹配**进一步将成功率从66.8%（Diffusion Policy+胸部标签）提升至83.8%（完整Mobile UMI），消除延迟导致的抖动。
- 解耦表示是必不可少的：去掉它后成功率骤降至18.8%。
- 延迟补偿也是必要的：去掉后成功率从83.8%降至60.0%，最大降幅出现在书本放置和家庭操作任务。
- 结论：在不改变底层策略架构（如扩散模型）的前提下，**显式运动学分解 + 状态级延迟对齐**即可有效解决移动模仿学习的耦合瓶颈。

## 7. 优点

- **硬件无关性**：演示采集无需物理机器人存在，可跨场景收集，降低门槛。
- **解耦表示简洁有效**：通过相对胸部坐标将人类行走与手部操作分离，无需复杂的全身动力学模型。
- **延迟补偿轻量化**：基于最近邻匹配丢弃过期航点，计算开销小，无需明确动力学模型或约束求解器，易于部署。
- **跨视角融合**：双视角（全局+局部）提供导航和操作所需信息，训练时使用条件扩散策略，保留多模态性。
- **全面的消融实验**：清晰量化解耦表示和延迟补偿各自贡献。
- **开源友好**：基于低成本商用传感器（Orbbec、AgileX）和ROS，便于复现。

## 8. 不足与局限

- **实验环境的局限性**：所有演示和测试在同一实验室进行，固定光照（500 lux）和对象布局，未评估光照变化、物体摆放多样性、环境杂乱程度等泛化能力。
- **任务多样性有限**：四个任务均为“搬运-放置”型操作，缺乏动态交互（如推、拉、接触力调节）和重负载操作。
- **解耦表示的动态限制**：不能捕获全身动态力交换（例如基座需要抵抗手臂力矩的情况），对重负载操作可能力不从心。
- **演示数量较少**：最多200次演示，对复杂任务可能不足。论文未研究数据量对性能的影响。
- **延迟补偿仅基于运动学**：未建模物理惯性、响应滞后，在高速/急停场景可能效果下降。
- **对比基线不够新**：与2023年的ACT和Diffusion Policy对比，未涉及2024-2025年更先进的策略（如基于Transformer的扩散模型、3D Diffusion Policy等）。
- **未讨论人体工程学**：手持相机-夹爪模块可能引入人为运动抖动，论文通过Savitzky-Golay滤波处理，但未评估其对演示质量的影响。
