## 摘要
视觉-语言-动作（VLA）模型已成为通用机器人操作领域的一个有前景的范式。当前架构的常见设计是将语言指令和视觉观察映射到单次前向传播中的动作。虽然在概念上简单，但这种表述将指令理解、空间场景理解和运动控制纠缠在单一学习目标中。因此，动作专家必须隐式地重新学习预训练VLM中已存在的认知和感知能力，这可能限制学习效率和泛化能力。我们提出了AVP（视觉基元动作），这是一种端到端架构，实现了这种以视觉基元为中心的接口：VLM推断下一阶段目标并生成视觉基元标记，这些标记作为流匹配动作专家的条件，其监督来自末端执行器运动学。在通用抓取与放置任务上的真实机器人实验表明，AVP相比pi_0.5将成功率提高了27.61%，并优于其他最新方法，在数据效率、空间组合泛化和物体级迁移方面均取得了一致的提升。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for generalist robotic manipulation. A common design in current architectures maps language instructions and visual observations to actions in a single forward pass. While conceptually simple, this formulation entangles instruction comprehension, spatial scene understanding, and motor control within a single learning objective. As a result, the action expert must implicitly relearn cognitive and perceptual capabilities already present in the pretrained VLM, which can limit both learning efficiency and generalization. We introduce AVP (Action with Visual Primitives), an end-to-end architecture that implements this visual-primitive-centric interface: the VLM infers the next-stage target and emits visual-primitive tokens that condition a flow-matching action expert, with supervision derived from end-effector kinematics. Real-robot experiments on general pick-and-place tasks show that AVP improves the success rate by 27.61% over pi_0.5 and outperforms other recent methods, with consistent gains in data efficiency, spatial-compositional generalization, and object-level transfer.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：当前 Vision-Language-Action (VLA) 模型将语言指令和视觉观测直接映射为动作，导致动作专家必须隐式学习认知与感知能力（如空间理解、场景解析），而 VLM 的预训练知识未能被充分利用。这种纠缠的学习目标限制了数据效率、泛化能力，尤其在空间布局或物体外观发生变化时表现不佳。
- **研究动机**：探索 VLM 与动作专家之间的合理职责分工 —— VLM 负责“做什么、在哪里”，动作专家专注于“如何执行”。现有方法（如规划中心、视觉提示）虽引入中间表示，但存在依赖外部模型、级联误差、延迟较高等问题。
- **整体含义**：提出一种端到端架构 AVP，通过视觉基元作为显式的 VLM↔动作专家通信接口，实现解耦学习，提升操控成功率和泛化能力。

## 2. 方法论

### 核心思想
- 在 VLM 和动作专家之间引入 **视觉基元（Visual Primitives）**—— 由 VLM 内部生成的空间接地 token（如点、边界框、掩码等），作为动作专家的显式条件输入。
- 视觉基元的监督直接从机器人末端执行器运动学自动获取，无需手动标注或外部感知模型。

### 关键技术细节
1. **VLM 推理阶段**：给定语言指令 `l` 和观测 `o_t`，VLM 产生多模态上下文 token。
2. **视觉基元解码器**：自回归预测离散化的视觉基元 `p_t`（编码下一阶段子任务及空间目标）。
3. **投影模块**：将视觉基元投影到视觉 token 空间，得到 `z_t^{vp}`，并融合到原始多模态表示形成增强表示 `z_t^{aug}`。
4. **动作专家**：基于增强表示和本体感知状态 `s_t`，使用流匹配（Flow Matching）预测动作序列 `a_{t:t+h}`。
5. **训练损失**：联合优化 `L = L_act + λ L_vp`，其中 `L_act` 为动作预测损失，`L_vp` 为视觉基元预测交叉熵损失。
6. **监督自动生成**：通过三步骤——（i）从夹爪状态变化提取交互关键帧；（ii）从机器人本体获得末端执行器 3D 位置；（iii）利用相机标定将 3D 点投影到图像平面并离散化为基元标签。

## 6. 主要结论与发现

- **性能提升**：在所有三个任务上，AVP 显著优于 π0.5 基线。中国象棋任务平均成功率提升 27.61%（62.67%→90.28%），通用物体任务提升 21.22%（64.96%→86.18%），多米诺任务提升 6.25%（81.94%→88.19%）。
- **泛化能力**：在空间组合泛化（A→B 直接移动）上，基线完全失效（0%），而 AVP 达到 83%；在零样本跨域泛化中，AVP 能成功处理多数未见物体，而基线在分布偏移下显著退化。
- **视觉基元有效性**：消融实验表明，框+掩码+记忆三要素组合实现最佳性能（91% 平均成功率），且掩码透明度影响放置精度。
- **延迟**：AVP 推理延迟 0.27s（中国象棋），远低于级联视觉提示方法（Point-VLA 37.32s），略高于单次策略 π0.5（0.16s）。

## 7. 优点

- **端到端设计**：无需外部检测器、分割器或视觉提示生成器，降低模块依赖和级联误差。
- **自动标注**：视觉基元监督完全从末端执行器运动学自动生成，避免昂贵的人工空间标注。
- **解耦认知与动作**：VLM 专注于 what/where，动作专家专注于 how，提升学习效率和泛化。
- **丰富的泛化能力**：在空间组合和跨域物体上均表现出显著优势，有助于实际部署。
- **实验全面**：涵盖多个任务、多种指标和消融，结论可靠性高。

## 8. 不足与局限

- **额外延迟**：两阶段推理（基元预测+动作生成）比单次策略增加约 0.11s，在实时要求高的场景中可能受限。
- **标定敏感性**：视觉基元监督依赖精确的手眼标定，相机外参漂移或物理干扰会直接影响基元质量。
- **任务局限**：实验仅涵盖桌面抓取放置，未验证在移动、变形物体、非结构场景等复杂任务中的表现。
- **统计稳健性**：泛化实验样本量较小（例如仅 8 个物体一次性测试），缺乏重复试验和置信区间，可能高估泛化能力。
- **训练算力未报告**：缺乏训练所需的 GPU 总数和时间的明确信息，难以复现或比较计算开销。
- **对比基线选择性**：主要对比 π0.5，虽涉及 DM0/LDA 但样本少，未与最新 VLA 模型（如 π0.7）在主实验直接比较。
