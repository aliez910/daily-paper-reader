## 摘要
视觉-语言-动作（VLA）模型已成为通用机器人操作的一种有前景的范式。当前架构中的常见设计是将语言指令和视觉观测通过单次前向传递映射为动作。虽然概念上简单，但这种表述将指令理解、空间场景理解和运动控制纠缠在单一学习目标中。因此，动作专家必须隐式地重新学习预训练VLM中已有的认知和感知能力，这可能限制学习效率和泛化能力。我们提出了AVP（基于视觉基元的动作），这是一种端到端架构，实现了以视觉基元为中心的接口：VLM推断下一阶段目标并发射视觉基元标记，这些标记条件化一个流匹配动作专家，监督信号来自末端执行器运动学。在通用拾取和放置任务上的真实机器人实验表明，AVP相比pi_0.5提高了27.61%的成功率，并优于其他近期方法，在数据效率、空间组合泛化和物体级迁移方面持续取得增益。

## Abstract
Vision-Language-Action (VLA) models have emerged as a promising paradigm for generalist robotic manipulation. A common design in current architectures maps language instructions and visual observations to actions in a single forward pass. While conceptually simple, this formulation entangles instruction comprehension, spatial scene understanding, and motor control within a single learning objective. As a result, the action expert must implicitly relearn cognitive and perceptual capabilities already present in the pretrained VLM, which can limit both learning efficiency and generalization. We introduce AVP (Action with Visual Primitives), an end-to-end architecture that implements this visual-primitive-centric interface: the VLM infers the next-stage target and emits visual-primitive tokens that condition a flow-matching action expert, with supervision derived from end-effector kinematics. Real-robot experiments on general pick-and-place tasks show that AVP improves the success rate by 27.61% over pi_0.5 and outperforms other recent methods, with consistent gains in data efficiency, spatial-compositional generalization, and object-level transfer.

## 2. 方法论：核心思想与关键技术细节

**核心思想**：引入**视觉基元（Visual Primitives）** 作为 VLM 与动作专家之间的显式通信媒介——VLM 输出空间化的视觉基元（如点、边界框），动作专家依据这些基元生成动作。

**技术架构（AVP）**：包含三个模块：
- **预训练 VLM**：理解语言指令和多视角观测，预测下一阶段子目标任务及其对应的视觉基元。
- **自回归解码器**：将 VLM 的输出解码为离散化的视觉基元标记 \(p_t\)。
- **动作专家**：基于融合了视觉基元标记的图像特征，使用流匹配（flow-matching）模型预测动作序列。

**关键流程**：
1. 给定语言指令 \(l\) 和观测 \(o_t\)，VLM 输出多模态上下文令牌。
2. 视觉基元解码器 \(D_\psi\) 自回归预测下一步视觉基元 \(p_t\)。
3. 投影模块将 \(p_t\) 映射到视觉令牌空间，得到 \(z^\text{vp}_t\)，与原始多模态令牌融合，形成增强表示 \(z^\text{aug}_t\)。
4. 动作专家 \(\pi_\theta\) 以 \(z^\text{aug}_t\) 和机器人状态 \(s_t\) 为条件，输出动作 \(a_{t:t+h}\)。

\[
\mathcal{L} = \mathcal{L}_\text{act} + \lambda \mathcal{L}_\text{vp}

- \(\mathcal{L}_\text{vp}\)：视觉基元预测的交叉熵损失，**监督信号直接来自末端执行器运动学**（通过手眼标定将 3D 位置投影到 2D 图像网格）。

## 6. 主要结论与发现

- AVP **在所有三个任务上全面优于 π0.5**，整体成功率提升 **27.61%**（中国象棋：90.28% vs 62.67%）。
- 在 **空间组合泛化** 上，AVP 能零样本执行训练中未见的直接转移（8/8 成功），而 π0.5 完全失败（0/8）。
- 在 **跨领域零样本泛化** 上，AVP 对未见物体保持高成功率（如白色布+直接转移场景下 7/8），π0.5 则急剧下降。
- 消融研究：**视觉基元（尤其点或框+掩码）显著提升性能；添加记忆（跨帧持久化）效果最佳**。
- 表明显式解耦视觉基元能提升动作专家的学习效率与泛化能力。

## 7. 优点

- **方法创新**：首次提出将视觉基元作为 VLM 与动作专家之间的显式内部通信接口，避免了外部感知模块的依赖和级联误差。
- **监督便捷**：基元标签直接从末端执行器运动学投影得到，无需人工标注或额外感知模型，可低成本扩展到新平台。
- **端到端优化**：视觉基元生成与动作预测联合训练，增强了表示的对齐和鲁棒性。
- **泛化能力突出**：在空间组合和物体零样本迁移上表现优异，表明模型学到了物体无关的空间操控技能。
- **实时性较好**：推理延迟 0.27s，远低于基于 API 的 Point-VLA（37.32s），可部署于真实机器人。

## 8. 不足与局限

- **推理延迟增加**：两阶段推理（先预测基元，再生成动作）相比单遍策略（如 π0.5 的 0.16s）增大了延迟（0.27s），可能影响高速任务。
- **依赖精确标定**：视觉基元监督依赖于手眼标定，相机外参漂移或物理扰动会引入误差。
- **平台单一**：仅在 AgileX Piper 双机械臂平台上验证，未在多种机器人形态或真实工业场景测试，泛化性有待在更异构的数据集上验证。
- **对比不完全**：Point-VLA 仅测试子集，且部分基线（DM0、LDA）在统一动作空间下可能未进行最优适配，结论可能存在细微偏差。
- **未系统研究缩放性**：论文虽讨论了低监督成本和时间扩展性，但未给出更大规模实验（如更多任务、更长 horizon）的明显证据。
