## 摘要
便携式演示界面上的移动模仿学习面临两个耦合的瓶颈：行走干扰的动作标签以及连续移动基座上推理导致的执行延迟。近年来，腕戴式界面降低了桌面数据收集的成本，但单一的腕部视角无法捕捉基座导航所需的全局上下文。添加身体安装的摄像头会使人行走与手部运动纠缠在一起。同时，生成式策略引入了数百毫秒的推理延迟，在此期间基座会越过预测的路径点，迫使在动作拼接处进行向后修正。本文提出Mobile UMI，一个无需硬件的演示框架，通过三个组件解决这两个问题。首先，双摄像头采集系统记录以胸部为中心的全局上下文和以腕部为中心的局部交互，无需真实机器人介入。其次，基于一次性ChArUco的空间锚点统一了胸部和手部的视觉惯性坐标系；然后将手部姿态相对于胸部重新表达，以提取解耦的SE(3)操作和SE(2)基座轨迹。第三，异步滚动时域执行器进行在线状态匹配：每个生成的动作片段与当前物理姿态重新对齐，以便在执行前丢弃过期的路径点。整个系统在四个长期家庭任务上进行了评估，每个任务100次试验平均成功率达到83.8%。与ACT和扩散策略的对比实验表明，仅使用胸部相对标签就能缩小大部分差距；在线状态匹配则弥补了剩余部分。这些结果表明，在测试条件下的移动模仿学习中，显式运动学分解结合状态级延迟对齐提供了有效的解决方案，而无需改变底层策略类的架构。

## Abstract
Mobile imitation learning on portable demonstration interfaces faces two coupled bottlenecks: locomotion-contaminated action labels and inference-induced execution latency on a continuously moving base. Recent wrist-mounted interfaces lower the cost of tabletop data collection, yet a single wrist view does not capture the global context required for base navigation. Adding a body-mounted camera entangles human walking with hand motion. Meanwhile, generative policies introduce hundreds of milliseconds of inference latency, during which the base advances past predicted waypoints, forcing backward corrections at action splices. This paper presents Mobile UMI, a hardware-free demonstration framework that addresses both gaps through three components. First, a dual-camera capture system records chest-centric global context and wrist-centric local interaction without any robot present. Second, a one-shot ChArUco-based spatial anchor unifies the chest and hand visual-inertial frames; the hand pose is then re-expressed relative to the chest to extract decoupled SE(3) manipulation and SE(2) base trajectories. Third, an asynchronous receding-horizon executor performs online state matching: each generated action chunk is realigned with the current physical pose so that expired waypoints are discarded before execution. The full system is evaluated on four long-horizon household tasks, achieving an average success rate of 83.8% over 100 trials per task. Controlled comparisons against ACT and Diffusion Policy show that the chest-relative label alone closes much of the gap; online state matching closes the remainder. These results indicate that, for mobile imitation learning under the tested conditions, explicit kinematic factorization combined with state-level latency alignment provides an effective solution without requiring architectural changes to the underlying policy class.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心挑战**：移动模仿学习（Mobile Imitation Learning）面临两个耦合的瓶颈：
  - **动作标签污染**：在便携式演示界面（如胸戴相机+手持模组）中，操作员的行走会同时移动躯干和手部，导致采集的全局手部姿态混杂了步行运动，策略难以区分有意的手臂运动和步行引起的平移。
  - **推理延迟问题**：生成式策略（如扩散策略）通常消耗数百毫秒的推理时间，在此期间连续移动的基座会向前运动，使得新生成的动作轨迹的第一个路径点已经位于机器人当前位置之后，执行时迫使向后修正，产生可见的机械抖动（轨迹回滚）。

- **研究目标**：开发一个无需机器人硬件参与的演示采集框架，能够从人类演示中提取解耦的基座导航和手臂操作动作，并解决推理延迟带来的执行问题，从而在移动平台上实现高效、平滑的模仿学习。

- **总体意义**：该方法不改变底层策略架构，仅通过显式运动学分解和状态级延迟对齐，即可在长时家庭任务中显著提升成功率，为移动型服务机器人的模仿学习提供了一种低成本、可扩展的实用方案。

## 2. 提出的方法论

### 2.1 核心思想

- **硬件无关的演示采集**：使用一个胸部相机（全局导航视角）和一个手持相机-夹爪模组（局部操作视角），操作员自由走动完成演示，无需真实机器人。
- **动作解耦**：通过空间锚定统一两个相机的坐标系，将手部姿态重新表达为相对于胸部坐标系，从而将步行运动与手臂操作分离，得到解耦的 SE(2) 基座轨迹和相对 SE(3) 手臂轨迹。
- **延迟补偿**：部署时采用异步滚动时域执行，并在每次动作片段拼接时进行在线状态匹配，丢弃因推理延迟而过期的路径点，避免轨迹回滚。

### 2.2 关键技术细节

1. **双相机系统与视觉惯性里程计（VIO）**：胸部相机和手持相机均使用 Orbbec Gemini 2L（RGB 1280×800 @30Hz + 6轴IMU @200Hz），通过 OpenVINS 分别估计各自在局部世界坐标系中的姿态轨迹 T_Wi^Ii(t) ∈ SE(3)。

2. **空间锚定（Spatial Anchoring）**：在演示场景中放置一个静态 ChArUco 标定板。每帧检测标定板姿态 T_Ci^tag(t)，结合相机-IMU 外参 T_Ci^Ii 得到板在各自世界坐标系中的估计 T_Wi^tag(t)。对所有有效帧取最小二乘平均得到鲁棒估计，进而计算两个相机世界坐标系之间的变换 T_Wc^Wh。

3. **解耦动作表示（Decoupled Action Representation）**：将手部姿态相对于胸部重新表达：
   - T_Cc^Ch(t) = [T_Wc^Cc(t)]^{-1} * T_Wc^Ch(t)
   - 该变换消除了共享的步行运动，仅保留手臂动作。
   - 最终策略学习三个分支：
     - **基座导航**：胸部轨迹的 SE(2) 地面投影。
     - **手臂操作**：相对手部姿态的增量 SE(3)。
     - **抓手控制**：指尖 ArUco 标记距离映射为 [0,1] 开度。

4. **非完整映射**：采集的基座轨迹是全向的（SE(2)），但执行端是差分驱动（非完整）。训练前丢弃侧向速度分量 v⊥，仅保留前进/旋转 (v_t, ω_t)。操作协议限制操作员仅向前/后/原地转向，实际 v⊥ 的 0.99 分位数 <3cm/s；部署时侧向速度 >5cm/s 被低通滤波裁剪。

5. **条件扩散策略架构**：
   - 观测：双 RGB 图像 + 自身状态（基座 SE(2)、相对手 SE(3)、抓手开度 + 历史增量）。
   - 动作向量：Δx,Δy,Δθ（基座3维）+ Δp(3), Δq(4)（手臂7维）+ g（抓手1维）= 11维。
   - 预测长度为 Tp=16 的动作片段，每次执行前 Ta=8 步。
   - 视觉编码器：共享 ResNet-18 + 全局平均池化，融合 proprio 状态形成全局条件向量。
   - 条件注入：扩散步 k 经正弦编码+MLP，通过 FiLM 层注入去噪骨干的每个残差块。
   - 去噪骨干：1D 时间 U-Net（编码-解码结构）。
   - 训练：100 步扩散（余弦平方方差调度），推理时 DDIM 将步数降至 10 步。

6. **异步滚动时域执行 + 状态匹配**：
   - 异步运行：当前动作片段执行时，后台生成新片段。
   - 状态匹配：收到新动作序列后，从观测时刻状态进行前向运动学积分得到预测状态序列 {ŝ_{t0+iδ}}。
   - 查询实时物理状态 s_now（编码器+里程计）。
   - 通过加权最邻近匹配找到最佳索引 i*，最小化：
     i* = argmin_i [ w_b * d_SE2(̂b_i, b_now) + w_t * ‖̂p_i - p_now‖² + w_r * d_SO3(̂R_i, R_now) + w_g * (ĝ_i - g_now)² ]
   - 丢弃 i < i* 的路径点，从 i* 开始执行。

## 6. 主要结论与发现

- **整体性能**：Mobile UMI 在四个任务上平均成功率为 83.8%（茶包 95%，关灯 90%，书本 80%，家庭操作 70%），显著优于最佳基线（扩散策略+胸部相对标签 66.8%）。
- **解耦表示的贡献**：从全局标签切换为胸部相对标签，ACT 提升 9.0pp（42.5%→51.5%），扩散策略提升 14.3pp（52.5%→66.8%），说明解耦动作标签是改善性能的最重要因素。
- **延迟补偿的贡献**：在解耦标签基础上加入状态匹配，成功率从 60.0% 提升至 83.8%（+23.8pp），说明延迟补偿对平滑执行至关重要，尤其在需要基座快速移动的任务（书本放置、家庭操作）中效果显著。
- **状态匹配的可解释性**：平均每次拼接丢弃 3.2 个路径点（对应 0.3-0.4s 轨迹），实际丢弃数量与延迟量一致。
- **关键发现**：无需修改底层策略架构（ACT 或扩散策略），仅通过显式运动学分解和状态级延迟对齐即可获得显著收益。

## 7. 优点

1. **无需机器人的演示采集**：允许在任意环境低成本、无线、自由地收集数据，避免了传统遥操作的物流和技能门槛。
2. **清晰的解耦思路**：利用 ChArUco 锚定进行空间对齐，将手部姿态相对于胸部表达，自然分离步行和操作，方法简洁且有效。
3. **实用的延迟补偿**：状态匹配仅需编码器和里程计信息，无需动力学模型或优化求解器，计算轻量，可在嵌入式控制器中实时运行。
4. **全面的消融验证**：分别量化了解耦表示和延迟补偿各自的增益，证明两者缺一不可。
5. **性能优异**：在 4 个长时家庭任务上 >80% 成功率，且基线对比公平（相同演示/观测/调度）。
6. **可复现性**：全部使用开源工具（OpenVINS、OpenCV、PyTorch）和常见硬件（Orbbec 相机、AgileX 机器人），有助于社区复现。

## 8. 不足与局限

1. **实验覆盖有限**：

2. **潜在偏差**：
   - 演示者仅有一人（单操作员），可能隐含特定习惯，策略可能对该演示者过拟合。
