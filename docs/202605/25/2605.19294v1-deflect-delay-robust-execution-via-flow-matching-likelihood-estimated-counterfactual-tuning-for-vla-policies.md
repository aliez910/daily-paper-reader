## 摘要
视觉-语言-动作（VLA）策略通常以异步推理方式部署：机器人执行先前预测的动作块，同时模型计算下一个动作块。这导致了预测与执行之间的错位：动作块依赖于推理开始前的观测，但在物理状态已经向前漂移几个控制步时执行；在Kinetix上，当推理周期覆盖多达七个控制步时，简单的异步翻转成功率从89%骤降至1%以下。我们提出了DEFLECT，一种完全离线的训练后微调方法，通过将延迟本身转换为无标签的偏好信号，作为现有异步VLA栈的近乎即插即用的升级：从冻结的参考策略构建反事实的更新/过时动作对，并通过隐式流匹配似然比代理在部署时条件下进行评分，无需人工标签、奖励模型或在线 rollout。DEFLECT显著扩展了异步VLA控制的可用延迟范围，在高延迟场景（5-7个控制步）中获得+6.4的成功率提升，在最长延迟下转移到真实规模VLA时获得+4.6的提升，并在两个真实机器人任务（双臂传送带拾放和反应性打地鼠）中持续改进。

## Abstract
Vision-Language-Action (VLA) policies are typically deployed with asynchronous inference: the robot executes a previously predicted action chunk while the model computes the next one. This creates a prediction-execution misalignment: the chunk is conditioned on the observation taken before inference began, but executes in a physical state that has already drifted forward by several control steps; naive asynchronous rollover collapses from 89% to under 1% on Kinetix as the inference cycle covers up to seven control steps. We introduce DEFLECT, a fully offline post-training refinement that applies as a near drop-in upgrade to existing async-VLA stacks by converting latency itself into a label-free preference signal: counterfactual fresh/stale action pairs are constructed from a frozen reference policy and scored under the deployment-time conditioning via an implicit flow-matching likelihood-ratio surrogate, with no human labels, reward models, or online rollouts. DEFLECT substantially extends the usable delay envelope of async VLA control, with +6.4 success-rate gain in the high-latency regime (5-7 control steps), +4.6 when transferred to a real-scale VLA at the longest delay, and consistent improvements on two real-robot tasks (a bimanual conveyor pick-and-place and a reactive whack-a-mole).

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将异步推理中的延迟转化为**无标签的偏好信号**（latency as counterfactual supervision）。通过构建“新鲜条件（fresh conditioned）”与“陈旧条件（stale conditioned）”下的反事实动作对，并将两者在部署时实际条件（混合条件 \(c_{\text{mix}}\)）下用隐式流匹配似然比代理进行评分，从而实现无需人工标签、奖励模型或在线rollout的离线偏好优化（DPO）。
- **关键技术细节**：
  1. **反事实偏好对构造**：从一个冻结的参考VLA政策 \(\pi_{\text{ref}}\) 出发，使用相同的采样噪声，在新鲜条件 \(c_{\text{fresh}}^t = (s_{t+d}, o_{t+d})\) 下生成优选动作 \(A^+\)，在陈旧条件 \(c_{\text{stale}}^t = (s_t, o_t)\) 下生成被拒动作 \(A^-\)。\(c_{\text{mix}}^t = (\hat{s}_{t+d}, o_t)\) 模拟部署时条件（自主感知回滚状态+过去的环境观测）。
  2. **流匹配似然代理**：因为流匹配政策不提供解析的动作密度，使用每个样本的流匹配损失（flow-matching loss）作为负对数似然的代理，即 \(r_\theta(A|c,l) = -\mathbb{E}_{\tau,\epsilon}[\|v_\theta(\tau A + (1-\tau)\epsilon, c, l, \tau) - (A-\epsilon)\|^2_2]\)。
  3. **DPO目标调整**：将两个候选动作在 \(c_{\text{mix}}\) 下用上述代理评分，形成参考校准的DPO边际，损失函数为：
     \[
     \mathcal{L}_{\text{DPO}}(\theta) = -\log\sigma\left( \beta\left[ (r_\theta(A^+)-r_\theta(A^-)) - (r_{\text{ref}}(A^+)-r_{\text{ref}}(A^-)) \right] \right)
     \]
     同时保留一个SFT锚项 \(\mathcal{L}_{\text{FM}}\) 在专家动作块上，防止崩溃。总损失为 \(\mathcal{L} = \lambda_{\text{SFT}}\mathcal{L}_{\text{FM}} + \lambda_{\text{DPO}}\mathcal{L}_{\text{DPO}}\)。
  4. **训练完全离线**：使用预先收集的轨迹数据，不需要在线交互；推理时仍使用与基础政策完全相同的ODE求解器，无额外计算开销。
