## 摘要
自主运行的机器人可能需要基于不再可见的证据做出决策。我们研究延迟证据任务，其中早期线索在后续决策点之前消失，因此视觉上相似的观测可能需要不同的动作。在这些设置中，当前观测不是控制的充分状态。我们提出基于轨迹路由的因果证据（TRACE），一种用于视觉运动模仿策略的记忆框架。TRACE将任务相关的视觉和机器人状态证据（如物体身份、目标选择或路径相关状态）存储在一个固定大小的潜在记忆中，该记忆在长回合中保持有界。TRACE不是通过原始时间或手动提供的任务标签来索引记忆，而是使用路径签名：已执行的机器人状态轨迹的紧凑、顺序敏感特征。这些签名不存储视觉线索本身；相反，它们提供轨迹条件键，用于写入和检索在线索可见时存储的证据。当机器人后来遇到模糊的观测时，策略以TRACE记忆为条件，以恢复缺失的上下文并选择正确的分支。TRACE通过轻量级适配器附加到策略上，而不改变策略主干、动作头或模仿目标。在具有视觉模糊分支点的现实世界长视野操作任务中，TRACE在分支选择和任务成功率上优于替代基线，包括短时历史和循环记忆。项目页面：https://jeong-zju.github.io/trace

## Abstract
Robots under autonomous operation may require decisions based on evidence that is no longer visible. We study delayed-evidence tasks, where an early cue disappears before a later decision point, so visually similar observations can require different actions. In these settings, the current observation is not a sufficient state for control. We introduce TRAjectory-routed Causal Evidence (TRACE), a memory framework for visuomotor imitation policies. TRACE stores task-relevant visual and robot-state evidence, such as object identity, target choice, or route-dependent state, in a fixed-size latent memory that remains bounded over long episodes. Instead of indexing memory by raw time or manually provided task labels, TRACE uses path signatures: compact, order-sensitive features of the executed robot-state trajectory. These signatures do not store the visual cue itself; rather, they provide trajectory-conditioned keys for writing and retrieving the evidence stored when the cue was visible. When the robot later reaches an ambiguous observation, the policy conditions on TRACE memory to recover the missing context and choose the correct branch. TRACE attaches through lightweight adapters to policies, without changing the policy backbone, action head, or imitation objective. Across real-world long-horizon manipulation tasks with visually ambiguous branch points, TRACE improves branch selection and task success over alternative baselines, including short-history and recurrent memory. ## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在机器人延迟证据（delayed-evidence）操作任务中，一个早期线索（如物体身份、起始侧）在后续决策点之前消失，导致视觉上相似的观测需要不同动作。传统视觉运动模仿策略假设当前观测或短时窗口是充分状态，因此在这种“分支点”处无法正确选择动作。
- **研究动机**：机器人需要一种因果的、固定预算的在线记忆，能够在线索可见时存储关键证据，并在决策点恢复该证据以解决歧义。现有方法如循环记忆或通用记忆可能被覆盖、稀释或与动作无关。
- **整体含义**：提出TRACE框架，通过轨迹路径签名（path signatures）作为索引键，将视觉和状态证据写入固定大小的潜在记忆槽，并在歧义时刻读取，从而在不改变基策略骨干或损失函数的前提下提升任务成功率。

## 2. 论文提出的方法论

### 核心思想
- 将记忆内容与记忆访问分离：当前观测提供“记住什么”，已执行的机器人状态轨迹决定“存在哪里、怎么读”。
- 使用**路径签名**（depth-p 截断签名）作为轨迹派生的紧凑、顺序敏感特征，作为读写键。路径签名对时间重参数化和偏移不变，但对顺序敏感，适合表示执行历史。
- 采用**固定数量的潜在记忆槽**（K=4或6），每个槽存储证据，通过轨迹路由来选择性写和读。

### 关键技术细节
- **状态表示**：多视角RGB图像（3个视角） + 17维机器人状态（含基座、双臂关节位置），状态经过掩码和标准化。
- **路径签名计算**：在线流式计算机器人状态轨迹的深度p签名（p=3，特征维度5219），并计算增量δ。两者分别映射为嵌入gt和Δgt，再组合成轨迹键qt。
- **槽记忆写过程**：
  - 将当前观测编码为et（池化的视觉特征 + 状态嵌入）。
  - 利用轨迹键qt计算路由得分，通过softmax选择写入哪些槽（权重ω）。
  - 生成候选写入内容，并用门控机制更新选中的槽。
- **阅读过程**：当前观测和轨迹参数形成查询，对更新后的槽做注意力读出，得到zmem。
- **策略适配器**：
  - TRACE输出共享状态（记忆槽、读出、签名嵌入）传入轻量适配器，转换成基策略的条件输入。
  - 对于回归策略（ACT），适配器将槽编码为记忆token，连接到注意力上下文。
  - 对于扩散策略，适配器将记忆池化并映射为全局条件向量。
  - 基策略的动作表示、损失函数不变。
- **训练与推理一致性**：训练时使用固定预算的因果历史窗口扫描，推理时每一步应用相同更新器。

## 6. 论文的主要结论与发现

- **TRACE显著提升延迟证据任务性能**：平均阶段进度从基线的25%～50%提升至约60%～69%。在Book、Laundry、Medicine等强延迟依赖任务上增益最大。
- **跨策略家族通用**：对回归和扩散两种基策略均可实现一致提升，仅需更换适配器，共享相同的内存更新器。
- **优于通用记忆机制**：相比于GRU、Transformer上下文、LRU、检索提示等记忆方法，TRACE在平均进度上高出13～23个百分点，尤其突出对有序历史利用的特异性。
- **诊断证实历史证据使用**：顺序反转大幅降低路由一致性和分支一致性，而顺序保持的变换（时间重采样、速度抖动）影响很小，说明TRACE的读出来源于有序历史而非决策点观测。
- **长时稳定性**：在2×扩展历史长度下，分支决策一致性仍保持90%以上，槽覆盖广泛，未发生覆盖崩溃。

## 7. 优点

- **方法创新**：将路径签名首次引入机器人记忆的读写路由，紧凑且对轨迹顺序敏感，适合延迟证据场景。
- **模块化设计**：TRACE以轻量适配器附加，不改变基策略骨干、动作头或损失函数，易于集成到不同策略族。
- **在线可用**：记忆更新为因果、流式，训练与推理一致，无需测试时演示检索或未来信息。
- **诊断工具完善**：设计路由相似性和分支一致性指标，直观展示记忆行为与历史证据的依赖关系。
- **真实世界实验落地**：在5个实际机器人任务上验证，包含视觉歧义和长视距操作，结果具有说服力。

## 8. 不足与局限

- **签名维度爆炸**：深度p=3时签名维度已达5219；若p=4则88000+，增加计算和存储压力。论文仅使用p=3，未探索log签名等压缩手段。

- **实验规模有限**：仅5个桌面操作任务，缺少更多样化场景（移动操作、人形机器人）及仿真基准（如CALVIN、FurnitureBench）。VLA基线均经过微调，但可能存在训练不充分导致比较不公的风险。
- **算力报告不足**：未公开训练GPU型号、数量、总时间，降低复现便利性。仅提供推理延迟数据。
- **分支定义依赖手工设计**：任务的分支点、阶段划分是人为定义，虽符合场景，但能否自动发现分支模式未讨论。
