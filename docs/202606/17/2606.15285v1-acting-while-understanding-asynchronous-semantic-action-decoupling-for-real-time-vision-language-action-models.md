## 摘要
视觉-语言-动作模型（VLA）在机器人操作中展示了强大的任务理解和泛化能力，然而全模型推理的高计算成本限制了其在低延迟、高频闭环控制中的部署。我们提出了一种异步语义-动作解耦框架，沿着现有VLA的内部语义-动作接口将语义理解与动作生成分离，无需重新设计视觉-语言骨干网络或引入外部规划器。低频理解模块异步更新可重复使用的语义条件，而高频动作模块持续输出控制动作，无需重复调用完整模型。为了缓解过时语义与当前执行状态之间的时间不匹配，我们进一步引入了历史动作条件和时间错位训练，提供短时间内执行上下文，并在过时语义条件下提高反馈控制鲁棒性。在LIBERO上使用π0.5和UniVLA的实验，以及使用UniVLA的实际机器人部署表明，所提出的框架实现了高达35.6 Hz的服务端动作模块推理吞吐量，并提供了一条低侵入性路径，无需以控制频率运行完整VLA推理即可实现高频闭环控制。

## Abstract
Vision-Language-Action models (VLAs) have demonstrated strong task understanding and generalization in robotic manipulation, yet the high computational cost of full-model inference limits their deployment in low-latency, high-frequency closed-loop control. We propose an asynchronous semantic-action decoupling framework that separates semantic understanding from action generation along the internal semantic-action interface of existing VLAs, without redesigning the vision-language backbone or introducing an external planner. A low-frequency understanding module asynchronously updates reusable semantic conditions, while a high-frequency action module continuously outputs control actions without repeatedly invoking the full model. To mitigate the temporal mismatch between stale semantics and the current execution state, we further introduce historical action conditioning and time-misalignment training, which provide short-horizon execution context and improve feedback control robustness under stale semantic conditions. Experiments on LIBERO with $π_{0.5}$ and UniVLA, together with real-robot deployment using UniVLA, show that the proposed framework achieves up to 35.6 Hz server-side action-module inference throughput and offers a low-intrusion path to high-frequency closed-loop control without running full VLA inference at control rate.

## 2. 方法论

- **核心思想**：将VLA抽象为语义理解模块 \(U_\theta\) 和动作生成模块 \(G_\phi\)，利用两者之间的中间语义条件（intermediate semantic condition）作为可复用缓存。理解模块以低频（约3–5 Hz）异步更新语义缓存，动作模块在每个控制步（20 Hz）直接读取最新缓存的语义条件、当前机器人状态和近期动作历史，重新规划动作块，并通过ACT风格的时间集成（temporal integration）输出最终动作。
- **关键技术细节**：
  - **语义‑动作接口**：对于UniVLA，复用最后一层视觉与输出 token 的隐藏状态；对于π0.5，复用完整的语言‑视觉条件 KV‑cache。
  - **历史动作条件**：向动作模块提供过去约1秒的执行动作历史（长度根据骨干网络的动作块大小设置，如UniVLA为8步，π0.5为10步，真实机器人上为20步），通过轻量投影层（UniVLA）或额外条件嵌入（π0.5）注入，帮助在陈旧语义下获取局部执行进度信息。
  - **时间错位训练**：训练时模拟语义延迟，对每个动作预测步骤随机采样延迟 \(\Delta \sim \text{Uniform}\{0,\dots,H-1\}\)（\(H\) 为动作块长度），将模型暴露于过时语义条件下进行训练。损失函数为 \(\mathcal{L} = \mathbb{E}_{t,\Delta}[\ell_{\text{base}}(G_\phi(z_{t-\Delta}, s_t, h_t), A_t)]\)，其中 \(\ell_{\text{base}}\) 保持原骨干的训练目标（token预测、流匹配或动作回归）。
  - **推理流程**：理解模块后台持续更新缓存；动作模块在每个控制步使用最新缓存、当前状态和实际执行的历史动作重新规划整个动作块，再通过ACT指数加权（衰减系数0.1）合并重叠预测。

## 6. 主要结论与发现

- **异步语义‑动作解耦可在不显著牺牲成功率前提下大幅提升动作推理频率**：UniVLA-Async在服务端达到35.6 Hz动作模块推理吞吐量（同步UniVLA仅3.4 Hz），且真实机器人成功率平均93.3%优于同步UniVLA的78.3%（可能受益于更高频率的重规划）。
- **语义缓存直接复用（naive cache reuse）会导致性能明显下降**（LIBERO-Long从92.0降至85.2），说明单纯的缓存不足以维持稳定控制。
- **历史动作条件和时间错位训练两者互补**：仅历史动作恢复至86.4，仅延迟训练恢复至89.0，两者联合恢复至91.2，接近同步基线92.0，表明显式建模训练‑测试时域错位是关键，历史动作提供局部执行上下文作为辅助。
- **异步解耦与时间集成结合能够实现连续的、高频的状态反馈控制**，不需要全模型以控制频率运行，为大型VLA的实时闭环部署提供了低侵入性解决方案。

## 7. 优点

- **低侵入性**：在不重新设计视觉‑语言骨干网络、不引入外部规划器或复杂训练流水线的前提下，仅通过修改动作侧（添加历史动作条件）和训练策略（延迟模拟）实现异步解耦，易于集成到现有VLA。
- **跨骨干泛化**：在两种代表不同架构（UniVLA使用隐藏状态、π0.5使用KV‑cache）的VLA上均验证有效，证明了方法的通用性。
- **重点关注实际部署中的语义时域错配问题**：不是简单地复用缓存，而是通过历史动作和延迟训练显式增强动作模块在过时语义下的鲁棒性，并设计消融实验证明其贡献。
- **实验覆盖全面且控制严谨**：包含仿真与真实、多平台、多任务、多种对比基线，受控比较确保公平性；对于降低训练预算的π0.5变体结果保持客观，不做过度宣称。
- **推理吞吐量提升显著**：UniVLA-Async将动作侧推理从3.4 Hz提升至35.6 Hz，完全满足20 Hz控制循环需求，且时间开销主要在理解侧（约319 ms）而非动作侧（约4.6 ms）。

## 8. 不足与局限

- **依赖内部语义‑动作接口**：需要VLA中存在可复用的中间条件（如隐藏状态或KV‑cache），不同骨干的接口形式可能不同，限制了方法的直接迁移性。
- **语义陈旧仍存在风险**：当环境快速变化、执行阶段切换或不确定性激增时，固定频率的语义刷新可能不足，过时的语义仍可能误导控制。
- **实验覆盖有限**：仅聚焦于单臂操作任务，未验证在动态操作、移动操作、多机器人协作或高接触力交互等场景下的有效性；未测试更长动作历史窗口或自适应刷新策略。
- **未进行π0.5异步变体的真实机器人闭环实验**：仅报告了服务端吞吐量，无法证明其在真实控制循环中的稳定性。
- **消融分析范围受限**：仅在LIBERO-Long和UniVLA上进行，未探索不同历史长度、不同刷新间隔或在不同骨干上的消融。
- **时序集成和通信开销分析**：论文报告中动作模块延迟理想（4.6 ms），但实际端到端延迟包含通信（10–15 ms）和系统调度抖动，对高频控制的影响可进一步量化。
