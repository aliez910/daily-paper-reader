## 摘要
视觉-语言-动作（VLA）模型在机器人操作中展现出强大潜力，但在边缘硬件上的实时部署仍具挑战。本工作中，我们识别出VLM视觉和上下文token是部署延迟的主要来源：对于GEMM主导的投影算子，当模型维度固定时，计算量随输入token数量线性增长。基于此观察，我们提出RhinoVLA——一种与慧溪R1边缘SoC协同设计的面向部署的VLA模型。RhinoVLA采用令牌高效的Qwen3-VL骨干网络和连续动作专家，在保留预训练多模态能力的同时，降低了VLM侧的token和计算负担。为支持跨机器人学习，RhinoVLA进一步引入统一接口，结合视图注册表、72维物理状态-动作槽空间和机器人实例LoRA，使异构机器人观测和动作模式能够对齐到共享策略下。在部署方面，RhinoVLA通过硬件感知编译、混合精度执行和并行视觉编码进行优化。实验表明，在相似参数规模下，RhinoVLA的下游性能与π0.5相当，同时在慧溪R1上达到11.69 Hz的端到端推理频率，满足10 Hz实时闭环控制目标。该项目将开源在https://github.com/HuixiAI/RhinoVLA。

## Abstract
Vision-Language-Action (VLA) models have shown strong potential for robotic manipulation, but real-time deployment on edge hardware remains challenging. In this work, we identify VLM visual and context tokens as a major source of deployment latency: for GEMM-dominated projection operators, computation grows linearly with the number of input tokens when model dimensions are fixed. Motivated by this observation, we propose RhinoVLA, a deployment-oriented VLA model co-designed with the Huixi R1 edge SoC. RhinoVLA adopts a token-efficient Qwen3-VL backbone and a continuous Action Expert, reducing the VLM-side token and computation burden while preserving pretrained multimodal capability. To support cross-robot learning, RhinoVLA further introduces a unified interface that combines View Registry, 72D physical state-action slot space, and robotinstance LoRA, allowing heterogeneous robot observations and action schemas to be aligned under a shared policy. On the deployment side, RhinoVLA is optimized through hardware-aware compilation, mixed-precision execution, and parallel visual encoding. Experiments show that RhinoVLA achieves downstream performance comparable to π0.5 at a similar parameter scale, while reaching 11.69 Hz end-to-end inference on Huixi R1, meeting the 10 Hz real-time closedloop control target. The project will be open-sourced at https://github.com/HuixiAI/RhinoVLA.

# RhinoVLA 技术报告 详细中文总结

## 一、核心问题与研究背景

- **核心问题**：视觉-语言-动作（VLA）模型在边缘硬件（如机器人机载芯片）上实时闭环推理延迟过高，难以满足 10 Hz 的控制要求。具体瓶颈在于：VLM 中的视觉和上下文 token 数量巨大，导致 MLP 投影算子（GEMM 为主）的计算量与 token 数线性增长，成为运行时的主要开销。
- **背景**：现有 VLA 模型（如 π0.5、GR00T N1）在 NVIDIA Jetson AGX Orin 上分析显示，VLM 骨干和动作专家共占总延迟的 90% 以上，而 VLM 延迟的约 75% 来自 MLP 的线性投影（gate_proj, up_proj, down_proj）。因此，降低 token 数量及协同优化芯片利用率是突破部署瓶颈的关键方向。

## 二、方法论

### 2.1 核心思想
- **Token 高效 VLM 骨干**：采用 Qwen3‑VL‑2B（每张 256×256 图像仅 64 个融合视觉 token，对比 PaliGemma 的 256 个），从源头减少 MLP GEMM 计算量。
- **统一跨机器人接口**：通过**视图注册表**、**72 D 物理状态‑动作槽空间**和**机器人实例 LoRA**，将异构机器人的观测和动作模式对齐到共享策略。
- **软硬件协同部署优化**：在慧溪 R1（7 nm，500 TOPS INT8）上结合硬件感知编译、混合精度（W8A16）和并行视觉编码，实现 11.69 Hz 端到端推理。

### 2.2 关键技术细节
- **VLM 与 Action Expert 解耦**：
  - VLM 骨干（2.13 B 参数）提供视觉‑语言理解，KV 缓存最后 18 层暴露给 Action Expert。
  - Action Expert（0.40 B 参数，18 层 Transformer）基于 Qwen 兼容的注意力/MLP 结构，以流匹配生成连续动作块。
- **视图注册表**：预处理时将每个相机的角色（如 `[head|rgb]`）作为显式标记插入图像前，消除数据集特定顺序依赖。
- **72 D 统一槽空间**：定义固定物理语义（如 D0‑D6 左臂关节弧度、D14‑D15 平行夹爪开合比、D48‑D50 头颈部旋转），附带二进制状态/动作掩码，仅对有效槽计算监督损失。
- **机器人实例 LoRA**：在 Action Expert 的 FFN 中引入每个机器人的低秩适配器，学习残差行为；训练时 hard‑routed by instance_id，推理时可合并至基座权重，保持统一计算图。
- **训练策略**：
  - 冻结 Qwen3‑VL 骨干，联合训练 VLM LoRA、共享 Action Expert 和实例 LoRA。
  - 采用掩码流匹配损失，仅监督有效动作槽；从混合机器人数据中以幂律采样。
  - 后训练阶段仅更新目标实例 LoRA，重用小样本适应。

### 2.3 部署优化
- **编译优化**：FlashAttention 风格 tiling 至 R1 的 SPM；算子融合（归一化、投影、残差等合并）；细粒度任务调度。
- **混合精度**：W8A16（INT8 权重量化 + FP16 激活），定制 GEMM 内核融合加载、反量化和计算，实现 1.69× 加速（以 up_proj 为例）。
- **并行视觉编码**：将三路相机图像批量输入 ViT，延迟从 34.52 ms 降至 24.31 ms。

## 六、主要结论与发现

1. **Token 数量是 VLA 部署延迟的关键因素**，通过 Qwen3‑VL 的 4× token 压缩可有效加速。
2. **统一接口（视图注册表 + 72 D 槽 + 实例 LoRA）有效支持异构机器人学习**，实例 LoRA 捕捉残差而不引入额外计算图。
3. **软硬件协同优化使 RhinoVLA 在慧溪 R1 上达到 11.69 Hz**，满足 10 Hz 实时闭环控制。
4. **性能与 π0.5 相当**：在 LIBERO 上平均 90.0%（低于 π0.5 的 96.9%），但在真实机器人上展现出竞争力（Galbot 任务 100% 匹配 π0.5，AgiBot G2 超越 π0.5）。

## 七、优点

- **问题定位准确**：深入分析算子级延迟，指出 token 数与 GEMM 计算量的线性关系，为设计提供直观依据。
- **新颖的算法‑系统协同设计**：从模型架构（token 压缩）到芯片特定优化（编译、W8A16、批处理）形成完整闭环。
- **统一跨机器人接口的设计创新**：视图注册表和 72 D 槽空间提供可扩展的标准化输入输出，实例 LoRA 保留个体差异而不破坏共享结构。
- **实验对比充分**：模拟对比 13 种基线，真实机器人在多个平台与 π0.5 直接比较，且包含 seen/unseen 设置评估泛化能力。
