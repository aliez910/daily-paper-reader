## 摘要
目前最广泛采用的机器人学习流程通过机器人演示或结构化人类数据来学习技能，但这些方法成本高昂且受限于特定具身。相比之下，非结构化人类视频提供了一种可扩展的替代方案。这些视频包含跨物体、场景和策略的多样化操作演示，但并未直接与机器人动作关联。我们提出LUCID，一个两阶段框架，从源自互联网规模数据集的非结构化人类视频中学习任务意图，并在大规模并行仿真中学习机器人控制。意图模型从当前观察的闭环中预测短期意图（场景中接下来应该发生什么）。特定具身的感觉运动策略将该意图转化为机器人动作。意图接口跨控制器共享，因此同一意图模型可应用于不同具身，从我们的主要灵巧手到平行夹爪。我们在五个真实世界操作任务上评估了LUCID：仅由互联网视频监督的搅拌、擦拭和分拣，零样本迁移到新颖场景和物体实例；以及各由1小时自行收集的智能手机视频监督的推T形件和电缆布线。项目页面：https://lucid-robot.github.io/。

## Abstract
The most widely-adopted robot learning pipelines today learn skills from robot demonstrations or structured human data, which are expensive to collect and tied to specific embodiments. In contrast, unstructured human videos provide a scalable alternative. They contain diverse manipulation demonstrations across objects, scenes, and strategies, but are not directly connected to robot action. We propose LUCID, a two-stage framework that learns task intent from unstructured human videos drawn from internet-scale datasets and learns robot control in massively-parallel simulation. The intent model predicts short-horizon intent (what should happen next in the scene) from the current observation in closed loop. An embodiment-specific sensorimotor policy converts this intent into robot actions. The intent interface is shared across controllers, so the same intent model can be applied to different embodiments, from our primary dexterous hand to a parallel-jaw gripper. We evaluate LUCID on five real-world manipulation tasks: stirring, wiping, and binning supervised by only internet video, with zero-shot transfer to novel scenes and object instances; and push-T and cable routing supervised by 1 hr each of self-collected smartphone video. Project page: https://lucid-robot.github.io/.

- **问题**：当前机器人操作策略大多依赖昂贵的机器人遥操作演示或结构化人类数据，这些数据收集成本高、规模受限，且往往绑定特定机器人本体（embodiment）。非结构化人类视频（如互联网视频）虽然规模大、内容丰富，但没有动作标签，无法直接用于训练机器人策略。另一方面，物理仿真可以大量生成带动作标签的数据，但需要为每个任务精心设计奖励函数，尤其对于高层意图的定义非常困难。
- **整体含义**：论文提出 LUCID，旨在通过解耦“意图理解”和“动作控制”，将人类视频的可扩展性与仿真训练的强泛化能力结合。核心是学习一个**与本体无关的意图模型**（从人类视频中预测场景短期变化），再为具体本体训练一个**感觉运动策略**（在仿真中生成动作来实现该意图）。此框架为无需遥操作的灵巧操作技能获取提供了新路径，并展示了同一意图模型跨不同机械手（灵巧手和平行夹爪）的迁移能力。

## 2. 方法论

### 核心思想
- **意图与控制解耦**：将任务分解为两个独立模块：
    1. **意图模型** $f_\theta$（embodiment-agnostic）：预测短期物体运动（object flow）和手掌参考位姿（palm pose），即“场景中接下来应该发生什么”。
    2. **感觉运动策略** $\pi$（embodiment-specific）：将意图参考映射为具体机器人的关节动作命令。
- **闭环预测**：在执行过程中，意图模型持续根据当前场景观测重新预测，而非一次性开环规划。

### 关键技术细节
- **意图模型**：
    - 输入：最近 $F$ 帧 RGB-D 图像 $I_t$，当前物体查询点（3D位置） $\{x_{n,0}^\text{trk}\}$，当前手掌位姿 $(p_0^\text{palm}, R_0^\text{palm})$。
    - 输出：未来 $T$ 步的物体点轨迹 $\{x_{n,\tau}^\text{trk}\}$ 和手掌位姿序列 $\{(p_\tau^\text{palm}, R_\tau^\text{palm})\}$（构成参考 $R$）。
    - 架构：基于 CoTracker3 的点标记转换器，修改点：(1) 条件于冻结的 DINOv3 RGB 特征 + 深度残差适配器；(2) 预测未来时间步；(3) 附加手掌位姿标记（与物体标记联合预测）。
    - 训练：在从互联网视频提取的监督信号上最小化均方误差（包括物体流、手掌位置、手掌旋转损失）。监督通过四阶段流水线提取：ViPE（相机与深度）、SAM 3.1（分割）、DenseTrack3Dv2（3D轨迹）、WiLoR+刚性配准（MANO手掌位姿）。
    - 强数据增强：特别是人类区域遮挡增强，防止意图模型依赖演示者手部外观。

- **感觉运动策略**：
    - 训练环境：Isaac Lab，大规模并行仿真。
    - 训练数据：程序化生成物体（原始形状组合）和参考轨迹（包含接近、手中运动、目标、脱离四个阶段），覆盖多种抓取策略。
    - 动作空间：臂关节位置增量 + 特征抓握基系数 + 手指残差（结合特征抓握偏置探索和精细操作）。
    - 教师策略 $\pi_T$：使用特权信息（完整物体表面点、物体位姿等），PPO训练，配合自适应领域随机化（课程从无重力到全重力、扰动增加）。
    - 学生策略 $\pi_S$：通过混合目标（PPO + 蒸馏）从教师策略学习，仅使用外部 RGB-D 相机可见的物体点、腕部深度图像和本体感觉。输入包括参考 $R$。
    - 奖励：单一任务无关奖励，包括物体追踪、手掌位姿跟踪、手指接触、动作正则化等。

- **真实部署**：
    - 每个意图周期（约1Hz）：使用 SAM 3.1 更新物体掩膜，采样查询点，查询意图模型 $f_\theta$ 获得新参考 $R$。
    - 其间：滑动窗口点跟踪器保持点一致性，策略 $\pi_S$ 以 50Hz 运行，使用当前观测和 $R$ 输出动作。

## 3. 实验设计

### 数据集与任务

## 6. 主要结论与发现

1. **闭环意图**对于鲁棒性关键：闭环 LUCID 在 web 任务上平均 73% 成功率，远高于开环基线的 28%（开环因执行偏离固定规划而失败）。
2. **意图与控制的解耦**使得同一意图模型可跨不同机器人本体（灵巧手和平行夹爪）工作，在手机任务上各达成 19/30 成功率差异不大（电缆布线更适夹爪）。
3. **训练视频数据量的增加**稳定提升了推理时的意图损失和真实世界成功率（从 1k 到 20k 片段，成功率从约 20% 升至约 80%+）。
4. **感觉运动策略设计选择**很重要：标记化编码器、特征抓握基、混合 PPO+蒸馏目标、腕部相机均贡献了性能。
5. **互联网视频足以训练出有效的操作意图模型**，零样本迁移到新场景、新物体、新视角。

## 7. 优点

- **创新性**：清晰解耦意图与控制，利用人类视频和仿真的互补优势，避开了昂贵的机器人遥操作数据。
- **可扩展性**：意图模型可从大规模非结构化视频训练；控制策略可在仿真中大规模并行训练，任务无关。
- **闭环属性**：使系统能适应执行过程中的变化，比现有开环视频规划器更强。
- **跨本体迁移**：展示了同一意图模型可驱动完全不同形态的机械手，为共享任务知识提供可能。
- **真实世界验证**：在 5 个任务上进行了实际系统验证，包括对可变形物体（布、纸巾）的操作，尽管策略仅在刚体物体上训练。
- **系统工作**：完整的从视频到部署的 pipeline（感知、意图预测、策略控制）在一个 GPU 上实时运行（目标 1Hz 意图更新，50Hz 控制）。

## 8. 不足与局限

- **流程脆弱**：系统高度模块化（SAM、ViPE、WiLoR、DenseTrack3Dv2 等），每个模块都可能是故障点（尤其是感知遮挡或无纹理物体时）。级联错误影响整个系统。作者建议端到端学习意图表示可缓解。
- **任务条件差距**：对于没有自然终止条件的任务（如搅拌），意图模型会无限循环，需要外部停止信号。每个新任务需要手动筛选训练视频，无法轻松扩展至大量任务。作者提议通过文本条件（如“搅拌”“放下勺子”）来扩展。
- **显式接口信息损失**：物体流+手掌位姿的接口丢弃了手指配置、精细接触等信息，可能限制了策略复现人类演示的精确性。作者建议学习隐式联合中间表示。
- **训练与部署匹配**：策略仅在刚体物体上训练，但在真实中能处理可变形物体（如布），说明有一定泛化，但可能不稳定。
- **实验统计有限**：真实试验数量较少（每场景 10 次），可能无法充分反映随机性和分布外情况。无置信区间或显著性检验。
- **对比基线单一**：仅比较了开环视频规划器（Veo 3.1）和一个蒸馏消融。缺乏与直接模仿学习（如从人类视频推断动作）或端到端 VLA 方法的比较（虽然作者指出这些需要机器人数据）。
- **资源细节不透明**：训练时间、GPU 总使用量未明确报告，影响可复现性评估。
