## 摘要
在机器人学习中，跨不同形态和环境扩展训练数据集已成为学习通用机器人策略的主流范式。这些策略通常通过行为克隆从预收集的演示中模仿动作进行训练。然而，由于机器人动作与数据采集机器人的动力学紧密相关，不同机器人可能需要不同的动作来实现相同的运动。这种差异阻碍了跨多种机器人的策略训练和部署。为了解决这一问题，我们提出使用笛卡尔状态增量作为跨机器人的通用动作表示，并引入状态预测与自适应命令执行（SPACE）框架。SPACE在三个层面处理机器人动力学变化：跨不同形态、同一形态的不同硬件单元以及单个机器人运行期间。它包含两个组件：(i) 笛卡尔状态增量策略，用于预测几何末端执行器位移；(ii) 动作适配器，将预测的笛卡尔状态增量转换为特定于机器人的控制命令。实验表明，当从跨不同形态和同一形态的不同硬件单元收集的数据中学习时，SPACE显著优于直接预测控制命令的策略。SPACE在部署时的动力学变化下也保持鲁棒性，包括控制频率、物体重量和控制器增益的变化。项目页面见http://haeone.site/space-website/。

## Abstract
In robot learning, scaling training datasets across diverse embodiments and environments has become a dominant paradigm for learning generalizable robot policies. These policies are commonly trained via behavior cloning to imitate actions from pre-collected demonstrations. However, since robot actions are tied to the dynamics of the data collection robot, different robots may require different actions to achieve the same motion. This discrepancy hinders both policy training and deployment across diverse robots. To address this, we propose using Cartesian state delta as a universal action representation across robots, and introduce State Prediction and Adaptive Command Execution (SPACE) framework. SPACE handles robot dynamics variation at three levels: across different embodiments, across hardware units of the same embodiment, and within a single robot during operation. It consists of two components: (i) a Cartesian state delta policy that predicts geometric end-effector displacement, and (ii) Action Adapter, which converts the predicted Cartesian state delta into robot-specific control commands. Experiments show that SPACE substantially outperforms policies that directly predict control commands when learning from data collected across different embodiments and across hardware units of the same embodiment. SPACE also remains robust under dynamics shifts at deployment, including changes in control frequency, object weight, and controller gains. - **核心问题**：在机器人模仿学习中，不同机器人的**动作表示存在不一致性**。由于机器人控制命令（如关节/笛卡尔增量）与数据采集**机器人的动力学（控制器增益、控制频率、磨损等）紧密绑定**，导致相同的控制指令在不同机器人上产生不同的实际运动。这种差异严重阻碍了跨本体、跨硬件单元的数据共享和策略泛化。
- **研究背景**：近年来，利用大规模跨机器人数据集训练通用策略（如Open X-Embodiment、DROID）成为主流。但现有方法通常直接预测数据集中记录的控制命令，忽略了不同机器人之间的动力学差异。此外，部分演示数据（如人类手持夹爪、拖拽示教）不包含传统控制指令，进一步限制了数据利用。
- **整体含义**：SPACE旨在通过**统一动作表示**和**在线动力学自适应**，在不同机器人间实现有效的策略学习和部署，为真正的通用机器人策略奠定基础。

## 二、论文提出的方法论

### 核心思想
SPACE包含两个核心组件：
1. **笛卡尔状态增量策略（Cartesian State Delta Policy）**：预测末端执行器的**实际位移**（`Δp_t = p_{t+1} - p_t`，包括位置和姿态增量），而非底层控制命令。该表示与机器人动力学和控制器无关，可从任何提供末端执行器位姿的机器人中直接提取，跨机器人一致。
2. **动作适配器（Action Adapter）**：将预测的笛卡尔状态增量转换为**特定目标机器人**的控制命令（如笛卡尔增量控制指令），以解决实际运动与命令之间的跟踪误差。

### 关键技术细节与公式

- **行为克隆目标**：  
  `max_θ E[log πθ(Δp_t | o_t)]`，其中观测`o_t`包含末端位姿、图像和语言指令。

- **Action Adapter初始化（离线校准）**：  
  使用随机脚本策略收集少量校准轨迹`D_cal`，通过最小二乘法拟合线性模型：
  ```
  min_{W0,b0} Σ ||W0 Δp + b0 - u||^2
  ```
  校准过程通常不到1分钟（FR3机器人，M=10条轨迹，K=50步）。

- **部署时在线自适应（LMS算法）**：  
  每一步执行后，利用观测到的实际位移与预测命令的误差，通过**最小均方（LMS）**更新适配器参数：
  ```
  e_t = W_t Δp_obs + b_t - u_hat
  W_{t+1} = W_t - μ e_t (Δp_obs)^T
  b_{t+1} = b_t - μ e_t
  ```
  这使适配器能够跟踪运行时动力学变化（如负载增加、接近奇异点等）。

- **算法流程（文字说明）**：  
  ① 用随机策略收集少量校准数据，拟合初始`W0, b0`；  
  ② 在部署时，策略每步输出目标笛卡尔增量`Δp_target`；  
  ③ Action Adapter将其转换为控制命令`u_hat`并执行；  
  ④ 观测实际运动`Δp_obs`，计算误差并在线更新适配器参数。

## 七、优点

- **方法简洁高效**：使用线性模型+LMS在线自适应，校准仅需一分钟，计算开销极低，易于部署。
- **通用表示解耦**：Cartesian state delta不依赖任何机器人的底层控制器，可从各类数据集（包括人类手持夹爪）中提取，自然支持跨本体数据共享。
- **实时在线自适应**：能够处理运行时动力学变化（负载、增益、频率等），无需重新训练策略，实用性强。
- **实验设计全面**：覆盖了从简单共训练到零样本迁移、跨硬件、多硬件数据集以及多种动力学偏移的真实场景，评估严谨。
- **显著性能提升**：几乎在所有实验中都大幅超越基准方法，尤其是跨硬件和负载变化场景，证明了方法的实用价值。
- **开源友好**：提供了项目页面，便于复现和推广。

## 八、不足与局限

1. **力感知缺失**：Cartesian state delta只反映末端位移，无法表达施加力的大小。对于力敏感任务（如拧紧、推拉），需要额外预测期望力。
2. **控制模态转换未探索**：仅将笛卡尔增量转换为笛卡尔增量控制命令。对于不支持笛卡尔增量控制（仅关节控制）的机器人，如何转换尚待研究。
3. **线性模型局限性**：Action Adapter假设从位移到命令的映射为线性，在强非线性动力学（如高柔性臂、非线性摩擦）下可能精度不足。
4. **实验平台单一**：所有实验仅在Franka Research 3上进行，未在UR5、其他工业臂或人形机器人上验证，跨本体评估也仅限于用FR3执行UR5数据。
5. **依赖末端位姿数据**：准确提取Cartesian state delta需要可靠的末端执行器位姿（通常来自正向运动学或跟踪系统），部分数据集可能缺失。
6. **校准需要可控操作**：初始化需要自动执行随机轨迹收集，在不能编程控制的遥操作机器人上可能困难，需要手动示范。
7. **补充实验细节不足**：资源与算力、超参数选择（如LMS学习率μ=0.2）的调优未详细展开；DROID实验仅使用单一关键词筛选，任务泛化性有限。
