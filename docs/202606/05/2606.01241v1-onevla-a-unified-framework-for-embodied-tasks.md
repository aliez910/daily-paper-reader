## 摘要
导航与操作是具身智能的基础能力，使机器人能够理解自然语言命令并与其物理环境进行交互。然而，当前的视觉-语言-动作（VLA）模型仍然受限于任务特定的架构，专门处理导航或操作，这阻碍了通用机器人智能体的发展。为弥合这一差距，我们提出了OneVLA，一种统一架构，将这些不同的任务整合到单个连贯的框架中。具体而言，我们设计了一个统一动作头，能够生成导航和操作动作，无需任务特定的变体。此外，我们提出了一种多阶段渐进训练策略——结合精心策划的数据构建和思维链（CoT）微调，促进了两个领域之间的强正迁移和相互强化。在模拟和真实环境中的大量实验表明，OneVLA实现了最先进的性能，显著优于专门的单任务模型和现有的跨任务模型。通过统一这些核心能力，OneVLA为真正通用的机器人系统铺平了道路。模型和源代码将公开发布。

## Abstract
Navigation and manipulation are fundamental capabilities of embodied intelligence, enabling robots to interpret natural language commands and interact physically with their surroundings. However, current Vision-Language-Action (VLA) models remain constrained by task-specific architectures, specializing in either navigation or manipulation, which hinders the development of general-purpose robotic agents. To bridge this gap, we introduce OneVLA, a unified architecture that integrates these distinct tasks into a single, cohesive framework. Specifically, we design a unified action head capable of generating both navigation and manipulation actions without requiring task-specific variants. Furthermore, we propose a multi stage progressive training strategy-incorporating curated data construction and Chain-of-Thought (CoT) fine-tuning that facilitates strong positive transfer and mutual reinforcement between the two domains. Extensive experiments in both simulated and real-world environments demonstrate that OneVLA achieves state-of-the-art performance, significantly outperforming both specialized single-task and existing cross-task models. By unifying these core capabilities, OneVLA paves the way for truly general-purpose robotic systems. The model and source code will be publicly released.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程（文字说明）

### 2.1 核心思想
- **统一架构**：使用单一的视觉-语言主干（Qwen2.5-VL-3B-Instruct）处理多模态输入，通过一个**统一动作头（Unified Action Head）**同时输出导航动作（离散的概率分布：前进、左转、右转、停止）和操作动作（连续的7自由度末端执行器控制：∆x, ∆y, ∆z, ∆roll, ∆pitch, ∆yaw, 夹爪）。
- **统一动作空间**：将导航动作（4维）和操作动作（7维）拼接成11维统一动作空间 `a_unified = [a_navi | a_mani] ∈ ℝ¹¹`，在训练时通过**任务特定权重掩码（task-specific weight mask）** 使得每个样本仅在自己相关的动作维度上计算损失，无关维度梯度接近零。
- **输出生成机制**：引入特殊标记 `<text>...</text><action>...</action>` 分离语言推理阶段和动作生成阶段，模型首先通过语言解释指令（chain-of-thought），再生成动作序列。
- **多阶段渐进训练策略**：将训练分为三个阶段，逐步引入任务复杂度和跨任务知识迁移。

### 2.2 关键技术细节

#### 统一视觉-语言编码器
- 视觉编码器：32层ViT，隐藏维度1280，输入多视图RGB图像（14×14块）。
- 投影层：将视觉特征线性投影到语言模型隐藏维度2048。
- 语言模型：36层Transformer，16个注意力头，2个KV头（GQA），输出上下文表示 `H ∈ ℝ^(N_v+N_t)×d_h`。

#### 统一动作头
- 采用**流匹配扩散过程（flow-matching diffusion）**：在训练时采样噪声ε～N(0,I)和时间步t～Beta(α,β)，构造噪声轨迹 `a_t = (1-t)ε + t*a_gt`，模型学习预测速度场 `v_θ = a_gt - ε`，最小化流匹配目标。
- 架构：动作编码器（正弦位置编码条件化）、DiT骨干网络（带交叉注意力融合动作与视觉语言特征H）、动作解码器（投影到11维统一空间）。
- 推理时使用欧拉积分逐步去噪，从随机噪声迭代到最终动作。
- 损失函数：使用任务特定掩码的均方误差 `L_action = mean( w_τ ⊙ (v̂_θ - v)² )`。

#### 损失函数设计
总损失：`L_total = λ_vlm L_vlm + λ_text L_text + λ_action L_action`
- `λ_vlm`：通用VQA数据的交叉熵损失（保持视觉语言对齐）。
- `λ_text`：具身任务样本中语言推理序列的下一词预测损失（包括边界标记）。
- `λ_action`：上述流匹配损失（对导航和操作分别使用权重掩码）。

#### 多阶段训练策略
- **阶段1**（建立操作能力）：仅使用操作数据集+通用VQA数据，只训练操作动作头。
- **阶段2**（发展导航能力）：加入导航数据，同时保留操作和VQA数据，使用统一动作头和掩码进行联合训练。
- **阶段3**（增强跨任务性能）：引入Chain-of-Thought（CoT）数据，同时监督语言推理损失，进一步优化推理和决策质量。
