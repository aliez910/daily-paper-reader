## 摘要
用于机器人模仿学习的人类示教通常包含错误和纠正行为，例如不精确的抓取、物体未对齐、不稳定的接触以及重复尝试。虽然这些片段通常被视为噪声或次优数据，但它们提供了有价值的证据，表明执行何时偏离期望路径以及如何恢复任务可行性。然而，现有的奖励和价值模型往往依赖于单调进展假设，这些假设捕捉了粗略的任务进展，但可能忽略了不完美示教中的局部执行错误和纠正行为。在这项工作中，我们提出了ReTVL（重试监督的价值学习），一个通过利用重试事件作为稀疏监督从混合质量机器人示教中学习对错误敏感的价值函数的框架。ReTVL通过结合全局进展校准与由稀疏标注的重试关键点诱导的局部成对偏好学习，捕捉错误周围的局部退化-恢复结构。然后，学习到的价值模型用于为下游行为克隆重新加权示教片段，减少有害执行错误的影响，同时保留有用的纠正行为。在真实机器人操作任务上的实验表明，ReTVL比基于进展的基线产生了更细粒度的价值估计，并改进了从不完美示教中的模仿学习。

## Abstract
Human demonstrations for robot imitation learning often contain mistakes and corrective behaviors, such as imprecise grasps, object misalignment, unstable contact, and repeated attempts. While these segments are commonly treated as noisy or suboptimal data, they provide valuable evidence about when execution deviates from a desirable path and how task feasibility can be restored. However, existing reward and value models often rely on monotonic progress assumptions, which capture coarse task advancement but may overlook local execution errors and corrective behaviors in imperfect demonstrations. In this work, we propose ReTVL (ReTry-Supervised Value Learning), a framework for learning mistake-sensitive value functions from mixed-quality robot demonstrations by leveraging retry events as sparse supervision. ReTVL captures the local degradation-and-recovery structure around mistakes by combining global progress calibration with local pairwise preference learning induced by sparsely annotated retry keypoints. The learned value model is then used to reweight demonstration chunks for downstream behavior cloning, reducing the influence of harmful execution errors while preserving useful corrective behaviors. Experiments on real-robot manipulation tasks show that ReTVL produces more fine-grained value estimates than progress-based baselines and improves imitation learning from imperfect demonstrations.

## 1. 核心问题与研究动机

- **问题**：机器人模仿学习中使用的人类演示数据常包含**错误和纠正行为**（如抓取偏移、重复尝试等）。现有价值/奖励模型大多假设任务进展**单调递增**，只能捕捉粗粒度进度，却忽略了局部执行错误和纠正确保能恢复的价值动态。
- **动机**：这些“重试”事件并非纯粹的噪声——它们蕴含着**执行偏离与恢复**的证据。若能利用重试事件作为监督信号，价值函数便能学会区分有害错误与有用的纠正行为，从而提升从不完美演示中学习的效果。

## 2. 方法：ReTVL（Retry-Supervised Value Learning）

### 核心思想
- 结合**全局进度校准**（利用手工进度标签或成功/失败信号）与**局部成对偏好学习**（利用稀疏标注的重试关键点），使价值函数不仅能估计宏观进度，还能在重试周围产生局部的**价值下降—回升**模式。

### 关键技术细节

1. **价值模型结构**：
   - 基于**Qwen3-VL-4B**（与Robometer相同）的VLM骨干，附加两层MLP离散价值头，预测64个进度区间的分布。
   - 输入：语言指令 + 8帧历史图像窗口（5Hz采样）。
   - 输出：期望进度值 \( V_\theta(h_t, \ell) \in [0,1] \)。

2. **全局进度监督**：
   - 对于成功轨迹，将时间步归一化为 \( v^\star_t = t/T \) 作为目标；失败轨迹终端设为0。
   - 使用交叉熵损失 \( L_{\text{abs}} \) 训练分布价值头。
   - 仅在**重试邻域之外**的帧上施加此损失，避免全局标签与局部重试冲突。

3. **局部重试偏好监督**：
   - 对每个重试关键点 \( r_{i,j} \)，定义三个时间窗口：
     - 前重试窗口 \( W_{\text{pre}} \)：离关键点稍远
     - 近重试窗口 \( W_{\text{near}} \)（被认为是局部低值区域）
     - 后重试窗口 \( W_{\text{post}} \)
   - 构造偏好对 \((h^+, h^-)\)，要求：
     - 近重试区域价值最低；前重试区域价值应递减（靠近重试的更低）；后重试区域价值应递增（远离重试的更高）。
   - 引入**软窗口权重**：
     - 对每个偏好对，基于较远的一端到关键点的距离 \( d(h^+; r) \) 赋予指数衰减权重 \( w = \exp(-d/\tau_w) \)，从而弱化边界模糊的影响。
   - 偏好损失（Pairwise Logistic Loss）：
     \[
     L_{\text{pref}} = -\mathbb{E}_{(h^+,h^-)} w \log \sigma\left( \frac{V_\theta(h^+)-V_\theta(h^-)}{T_{\text{pref}}} \right)
     \]
   - 总损失：\( L_{\text{value}} = \lambda_{\text{abs}} L_{\text{abs}} + \lambda_{\text{pref}} L_{\text{pref}} \)，采样比例 \( \rho_{\text{pref}} \)。

4. **下游行为克隆重加权**：
   - 对于动作块（chunk），计算价值改进 \( r_t = V_\theta(h_{t+\Delta_a}) - V_\theta(h_t) \)。
   - 根据改进值的均值和标准差将权重归一化到 [0,1]：\( \alpha_t = \text{clip}\left(\frac{r_t - (\mu-2\sigma)}{4\sigma+\epsilon}, 0, 1\right) \)。
   - 加权BC损失：\( L_{\text{wBC}} = \frac{\sum_t \alpha_t \ell_{\text{BC}}(\pi_\psi(h_t), a_t)}{\sum_t \alpha_t + \epsilon} \)。

## 3. 实验设计

### 数据集与场景
- **机器人平台**：AgileX Piper 机器人手臂（领导者-跟随者遥操作采集）。
- **任务**（4个真实操作任务）：
  - Pick up Spoon（简单拾取）
  - Stack Blocks（精度堆叠，三层）
  - Fold Towel（可变形物体折叠）
  - Open Drawer（铰接物体操作，含放物体再关门）
- **数据划分**：
  - 价值模型：每个任务30条标注轨迹（含重试关键点）+ 20条保留轨迹用于评估。
  - 下游策略：训练使用混合质量数据集（每任务200条，Pick up Spoon 80条）。

### 对比方法（Baselines）
- **TOPReward**：零样本VLM提示（“任务完成？”→ True token概率）。
- **Robometer**：通用机器人奖励模型，使用轨迹级比较与回退方法。
- **RECAP-Value**：基于蒙特卡洛步进目标的价值方法（与本文价值头结构一致）。

### 评估指标
- **全局指标**：VOC（进展排序相关性，只在干净成功轨迹上）、S/F Det.（终端值区分成功/失败分类准确率）。
- **局部指标**（在重试轨迹上）：
  - Drop AUC（价值下降是否聚集在重试附近）
  - Drop Probability（重试处是否检测到显著下降）
  - Pre > Retry（前窗均值高于重试点比例）
  - Post > Retry（后窗均值高于重试点比例）
- **下游策略**：成功率（20次独立实验），以及保留/抑制分析（成功块权重、后重试块权重、成功删除率、严格坏动作保留率）。

### 消融实验
- 对比三个变体：
  - w/o Preference Loss：直接用手动注入价值下降的进度监督替代偏好损失。
  - w/o Soft Window：去掉软权重，统一应用重试邻域监督。
  - w/o Absolute Calibration：去掉绝对进度损失。

## 6. 主要结论与发现

1. **ReTVL显著提升局部错误敏感性**：局部指标（Drop AUC、Pre > Retry等）远超所有基线，尤其Pre > Retry从最优基线的0.33提升至0.74，表明模型能准确把重试对应为局部价值最小值。
2. **全局性能保持竞争力**：VOC达0.987（仅次于Robometer的0.994），S/F检测达1.000（最优），说明引入偏好学习并未破坏全局进度结构。
3. **下游策略大幅改进**：ReTVL-BC平均成功率80%，相比Standard BC（41%）和RECAP-BC（63%）提升显著。坏动作块权重降低80%，同时高恢复块权重保持较高。
4. **消融关键性**：偏好损失是核心，无偏好损失时局部指标急剧下降；软窗口使全局价值更平滑；绝对校准提供全局锚点。

## 7. 优点

- **数据高效**：只需稀疏标注重试关键点（而非完整错误段），成本低（每任务约30分钟标注），且标注一致性高。
- **结合全局与局部信号**：同时保障对宏观进度和微观错误的感知。
- **软窗口设计巧妙**：缓解窗口边界歧义，提升训练稳定性。
- **应用价值高**：在不完美演示场景下有效提升策略性能，且不改变下游BC流程，易于部署。
