## 摘要
在机器人学习中，跨不同形态和环境的训练数据集扩展已成为学习通用机器人策略的主流范式。这些策略通常通过行为克隆来模仿预先收集的演示中的动作。然而，由于机器人动作与数据采集机器人的动力学相关，不同机器人可能需要不同的动作来实现相同的运动。这种差异阻碍了跨不同机器人的策略训练和部署。为了解决这个问题，我们提出使用笛卡尔状态增量作为跨机器人的通用动作表示，并引入了状态预测与自适应命令执行（SPACE）框架。SPACE在三个层级处理机器人动力学变化：跨不同形态、跨同形态的硬件单元以及单台机器人在运行过程中的变化。它包含两个组件：（i）笛卡尔状态增量策略，用于预测几何末端执行器位移；以及（ii）动作适配器，将预测的笛卡尔状态增量转换为机器人特定的控制命令。实验表明，当从跨不同形态和同形态硬件单元收集的数据中学习时，SPACE明显优于直接预测控制命令的策略。SPACE在部署时的动力学变化下也保持鲁棒性，包括控制频率、物体重量和控制器增益的变化。项目页面见http://haeone.site/space-website/。

## Abstract
In robot learning, scaling training datasets across diverse embodiments and environments has become a dominant paradigm for learning generalizable robot policies. These policies are commonly trained via behavior cloning to imitate actions from pre-collected demonstrations. However, since robot actions are tied to the dynamics of the data collection robot, different robots may require different actions to achieve the same motion. This discrepancy hinders both policy training and deployment across diverse robots. To address this, we propose using Cartesian state delta as a universal action representation across robots, and introduce State Prediction and Adaptive Command Execution (SPACE) framework. SPACE handles robot dynamics variation at three levels: across different embodiments, across hardware units of the same embodiment, and within a single robot during operation. It consists of two components: (i) a Cartesian state delta policy that predicts geometric end-effector displacement, and (ii) Action Adapter, which converts the predicted Cartesian state delta into robot-specific control commands. Experiments show that SPACE substantially outperforms policies that directly predict control commands when learning from data collected across different embodiments and across hardware units of the same embodiment. SPACE also remains robust under dynamics shifts at deployment, including changes in control frequency, object weight, and controller gains. ## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：机器人策略学习常采用行为克隆（Behavior Cloning）从预先收集的演示中训练。然而，机器人动作与采集时所用机器人的动力学紧密耦合；不同机器人（或同一型号的不同硬件单元）由于控制器、磨损、制造差异等，为实现相同运动需要不同的控制指令。
- **核心问题**：这种动作表示的不一致性导致跨机器人数据无法直接共享，训练和部署通用策略受到严重阻碍。
- **整体目标**：提出统一框架 **SPACE**，通过解耦动作与机器人动力学，实现从不同机械臂本体、同一本体不同硬件单元以及同一机器人在运行中动力学变化下的高效数据学习与鲁棒部署，迈向通用策略。

## 2. 论文提出的方法论

### 核心思想
- **用笛卡尔状态差（Cartesian state delta, Δp）作为通用动作表示**：Δp = p_{t+1} - p_t，即末端执行器的位姿增量。该量只反映实际运动，不依赖机器人控制器，可从任何提供末端位姿的轨迹中提取，适合跨机器人共享。
- **设计 Action Adapter**：一个轻量级（线性）模型，将预测的 Δp 转换为目标机器人能执行的特定控制指令，并能在部署时在线自适应。

### 关键技术细节
1. **笛卡尔状态差策略训练**：沿用 π0.5 VLA 模型，监督目标改为 Δp，输入为观测（位姿、图像、语言指令）。
2. **Action Adapter 初始化**：
   - 在目标机器人上执行 M=10 条随机脚本轨迹（长度 K=50），同时记录 Δp 和实际发送的控制指令 u。
   - 通过最小二乘线性回归拟合：û = W₀ Δp + b₀，得到初始参数。
   - 校准过程约 1 分钟。
3. **在线自适应（部署时）**：
   - 每步执行流程：策略输出 Δp^{target} → Adapter 生成命令 û_t = W_t Δp^{target}_t + b_t 并执行。
   - 观察实际运动 Δp^{obs}_t = p_{t+1} - p_t，计算预测误差 e_t = W_t Δp^{obs}_t + b_t - û_t（注意误差使用实际运动而非目标，因无真实指令可用）。
   - 使用最小均方算法（LMS）在线更新 Adapter 参数：
     - W_{t+1} = W_t - μ e_t (Δp^{obs}_t)ᵀ
     - b_{t+1} = b_t - μ e_t
     - 学习率 μ = 0.2
4. **算法流程**（文字描述）：
   - Step 1：用校准数据离线拟合初始 W₀, b₀。
   - Step 2：部署时循环：策略预测 Δp → Adapter 转命令 → 执行 → 观测实际运动 → 计算更新。
   - Step 3：每次 rollout 前重置 Adapter 为初始参数，确保独立。

## 6. 论文的主要结论与发现

- **笛卡尔状态差作为通用动作表示**有效缓解了跨机器人动作不一致问题。
- **跨本体学习**：SPACE 比控制命令策略平均提升约 30%（UR5+FR3 共训），50%（UMI+FR3 共训），并在零样本 UR5→FR3 上达到 92% 成功率（控制命令 0%）。
- **跨硬件单元学习**：控制命令策略在不同硬件间成功率从 98% 跌至 18%，而 SPACE 保持在 84%；在 DROID 多硬件数据上，SPACE 获得 84%/92% vs 基线的 4%/8%，提升超过 80 个百分点。
- **部署时动力学变化**：
  - 频率提升 2× 后，SPACE 成功率维持 94%，控制命令降至 50%。
  - 负载增加（90g→530g），SPACE 仍达 92%，控制命令完全失败（0%）。
  - 增益变化（0.5×/1.5×），控制命令成功率跌至 25%/0%，SPACE 分别保持 95%/100%。
- **Action Adapter 的关键作用**：离线初始化 + 在线更新缺一不可；随机校准轨迹比模式化轨迹更有效。

## 8. 不足与局限

- **力感知缺失**：笛卡尔状态差仅反映运动学位移，无法传递力信息，不适合需要力控的任务（如精密装配、软组织操作）。
