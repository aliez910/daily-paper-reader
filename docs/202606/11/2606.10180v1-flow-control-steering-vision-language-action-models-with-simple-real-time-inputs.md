## 摘要
我们介绍了视觉-语言-动作（VLA）模型的流控制，这是一种简单有效的方法，通过通用输入（例如键盘）实时操纵VLA动作。这种方法可以即用，无需重新训练或微调VLA。它允许相对粗略的用户输入来操纵VLA以符合用户意图。VLA将这些输入转换为动作样本，这些样本来自训练期间学到的VLA专家动作分布，因此生成的动作具有高质量（符合专家动作分布）和高保真度（反映用户意图）。我们证明了流控制具有许多理想的特性：（1）流控制通过用户输入准确且响应迅速地操纵机器人动作；（2）它对次优用户输入具有鲁棒性；（3）它使用户能够操纵VLA实现显著更高的成功率和更快的任务完成；（4）在流控制轨迹上微调VLA可以改进自主策略。这些结果共同为用户提供了一种简单直观的方式来帮助引导VLA动作，从而提高任务性能。

## Abstract
We introduce flow control of vision-language-action (VLA) models, a simple and effective way to steer VLA actions in real-time through generic inputs, such as a keyboard. This method can be used out-of-the-box and does not require retraining or fine-tuning VLAs. It enables relatively crude user inputs to steer a VLA to align with user intent. The VLA transforms these inputs into action samples drawn from the VLA expert action distribution learned during training, so that the generated actions are high quality (conformity to the action expert distribution) and high fidelity (reflecting the user's intent). We demonstrate that flow control has many desirable properties: (1) flow control accurately and responsively steers robot actions with user inputs, (2) it is robust to suboptimal user inputs, (3) it enables users to steer VLAs to achieve significantly higher success rates and faster task completion, and (4) fine-tuning a VLA on flow control trajectories improves the autonomous policy. Together, these results provide a simple and intuitive way for users to help steer VLA actions, increasing task performance.

- **整体含义**：本文旨在提供一种**简单、直观、低延迟且无需重新训练**的实时引导方法，允许用户通过低带宽输入（如键盘）来操控VLA的动作，从而提高任务的成功率和执行效率。这种方法被称为“流控制”，它利用VLA内部的流匹配生成式动作专家，将用户粗略的输入转化为高质量且符合用户意图的机器人动作，实现了零样本的共享自主性。

### 2. 方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
- **核心思想**：利用VLA中**流匹配动作专家**的特性。流匹配是一个确定性的常微分方程过程，它保留了初始条件中的信息（与扩散过程不同）。因此，通过直接修改流ODE（常微分方程）的初始条件为用户输入，再经由流匹配模型将其变换为专家动作分布中的样本，可以同时保证生成动作的**高质量**（符合专家分布）和**高保真度**（反映用户意图）。
- **关键技术细节**：
    - **输入映射**：用户通过键盘输入六个方向之一（上、下、左、右、前、后）的速度命令（`v_user`）。
    - **逆运动学**：将该3D笛卡尔末端执行器速度通过逆运动学转换为机器人关节速度（`x_user = IK(v_user)`）。此时，`x_user`通常处于VLA动作专家分布（`p_expert`）之外。
    - **初始条件注入**：将`x_user`进行归一化处理后，直接设置为流匹配ODE的初始条件（`x_0`）。然后执行标准的欧拉法积分（步数K=10，步长Δt=0.1），将`x_0`变换为最终的专家动作样本`x_1`。
    - **算法流程**：
        ```
        1. 循环开始:
        2.     观察上下文 c (图像、语言、状态)
        3.     获取用户键盘输入 v_user
        4.     计算 x_user = IK(v_user)
        5.     采样标准高斯噪声 x_0 ~ N(0, I)
        6.     将 x_0[0:τ] 替换为 归一化后的 x_user[0:τ] (τ代表动作块中受影响的步数)
        7.     for k = 0 to K-1:
        8.         x_{k+1} = x_k + Δt * v_θ(x_k | c)
        9.     end for
        10.    执行动作块 x_1 在机器人上
        11. 循环结束
        ```
    - **理论基础**（与扩散模型的对比）：
        - 扩散模型：每一步都注入噪声，形成马尔可夫链，初始条件信息逐渐丢失。
        - 流匹配：路径是确定性的，其流是一个微分同胚，初始条件和最终动作之间的互信息保持不变，因此用户输入在最终动作中得以保持。同时，仅当动作分布是多模态时，初始条件偏移才有效；当动作分布是单模态（即任务明确、无歧义）时，流匹配会将不一致的用户输入自动修正为策略内（on-policy）的动作，不会干扰任务执行。

### 3. 实验设计：数据集/场景、基准、对比方法
- **使用的VLA与数据集**：
    - VLA：`π0.5-DROID`，这是一个基于PaliGemma VLM并使用流匹配动作专家的模型，在DROID数据集上进行了微调。
    - 机器人硬件：Franka Panda七自由度机械臂，配置一个场景相机（ZED）和一个腕部相机（ZED Mini）。
- **任务场景**：
    1.  **两块拾放**（Two-Block pick-and-place）：两个红方块位于桌子左右，语言指令为“把方块放进洞”。该任务用于程序化控制测试。
    2.  **五块拾放**（Five-Block pick-and-place）：五个不同颜色方块沿直线放置，间距分别为0cm、2cm、4cm。用于评估控制精度和进行“偏离策略”的转向测试。
    3.  **Marker-in-Bowl**：三个不同颜色记号笔（红、黑、蓝）放在碗前，语言指令为“把红色记号笔放进碗里”。针对VLA的语言跟随缺陷进行测试。
    4.  **Cup-Stacking**：三个杯子放在桌上，语言指令为“堆叠杯子”。一个较长时域且更具挑战性的任务。
- **对比基准**：
    - **自主策略**（Autonomous）：运行未经过任何干预的`π0.5-DROID`。
    - **流控制引导**（Flow Control steering）：用户通过键盘实时引导同一VLA。
    - **遥操作**（Teleoperation）：用户使用Meta Quest 2 VR设备进行全身模仿式遥操作（作为用户研究中的对比，主要在时间效率上比较）。
    - **微调流控制**（Flow Control fine-tuned）：使用成功的流控制轨迹对`π0.5-DROID`进行微调后的模型（`π0.5-FC`）。

### 6. 主要结论与发现
1.  **可行性和有效性**：通过修改流匹配VLAs的初始条件，用户可以用简单的键盘输入实时、准确地引导机器人行为，即使这些输入很粗略。
2.  **鲁棒性**：当用户意图与VLA的固有单模态动作分布冲突时，流控制不会造成任务失败，而是会自动调整为用户行为专家分布内的动作，不影响任务完成。
3.  **性能提升**：
    - 在Marker-in-Bowl任务中，流控制将正确拾取目标记号笔的成功率从自主的**53.3%提升至99.4%**，平均任务时间缩短了8.3%。
    - 在Cup-Stacking任务中，成功率先从自主的**48.0%提升至87.7%**，更快（时间缩短30%），这得益于用户纠正了策略的典型错误（如拿起杯子不够高）。
4.  **微调价值**：流控制轨迹是策略内（on-policy）的，因此可以在其上微调VLA。微调后的模型`π0.5-FC`在Cup-Stacking任务上达到了**100%**的成功率，平均任务时间进一步降至22秒，说明该引导数据能有效提升自主策略能力。
5.  **公平性**：流控制能有效缩小不同用户在遥操作技能上的巨大差距，尤其对新手用户帮助显著，表现更加一致。

### 7. 优点
1.  **零样本可用（Out-of-the-box）**：无需任何数据收集、标注或模型微调，直接应用于任意基于流匹配动作专家的VLA。
2.  **简单直观**：用户界面极为简易（仅有六个方向的键盘输入），学习成本低。
3.  **低带宽且实时**：仅需输入一个方向命令，推理延迟几乎不增加（仅需改变初始条件）。
4.  **低风险引导**：引导动作自动保持在VLA的专家分布内（on-policy），避免了产生危险或无用动作的风险，甚至在用户输入与任务相悖时也能通过模型的“自修复”能力保持执行。
5.  **可训练增强**：收集的引导轨迹可作为高质量的正样本进行微调，从而提升原始模型的自主性能，解决了传统人工遥操作数据集难以获取或质量不稳定的问题。
6.  **共享自主的新范式**：流控制代表了共享自主性在VLA上的高效实现，特别是对于复杂高维机器人的控制，利用模型的先验知识将低维用户意图“解码”为高质量行为。

### 8. 不足与局限
1.  **适用范围有限**：该方法仅适用于具有**流匹配动作专家**的VLA模型。对于使用扩散动作专家（需通过SDE引导）或自回归离散动作token的VLA，该方法不适用或需要替代方案。
2.  **策略内的“顽固性”**：当VLA的专家动作分布在某个状态下是极度单模态的（例如，已决定抓取某个特定物体），仅仅修改初始条件可能不足以强行改变VLA的“决策”。虽然可以通过在流ODE积分过程中持续注入扰动（部分OOD）来纠正，但此文并未系统研究其最优策略，且这些动作可能退化为低质量的分布外动作。
3.  **控制分辨率限制**：在目标距离很近的情况下（如五块拾放任务的0cm间隔），流控制偶尔会选错目标（如误差一个目标），表明控制的精细程度受限于模型本身的技能偏好空间。
4.  **实验泛化性不足**：所有实验基于一种特定的机器人（Franka Panda）和一种VLA（`π0.5-DROID`），未在多种不同的VLA或机器人硬件上进行验证。任务种类也有限，主要是桌面拾放和堆叠，未涉及更动态、更复杂的操作。
5.  **用户输入依赖**：流控制完全依赖于用户的实时手动输入（键盘）。虽然它不断适应策略，但在用户注意力不集中或无法实时响应时（如脑机接口应用场景，文中提到了潜力但未证明），其效果会受限。此外，实验中的程序化输入是恒定且无噪声的，而真实用户输入存在变化和延迟，其鲁棒性仅在用户研究中得到一般性验证。
