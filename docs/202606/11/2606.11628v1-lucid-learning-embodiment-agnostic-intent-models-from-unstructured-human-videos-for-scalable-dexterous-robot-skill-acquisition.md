## 摘要
目前最广泛采用的机器人学习流程是从机器人演示或结构化人类数据中学习技能，这些数据收集成本高昂且与特定具身绑定。相比之下，非结构化人类视频提供了一种可扩展的替代方案。它们包含跨物体、场景和策略的多样化操作演示，但并未直接关联到机器人动作。我们提出了LUCID，这是一个两阶段框架，从互联网规模的数据集中提取的非结构化人类视频中学习任务意图，并在大规模并行仿真中学习机器人控制。意图模型从当前观测中闭环预测短视界意图（场景中接下来应发生什么）。一个具身特定的感觉运动策略将此意图转换为机器人动作。意图接口在不同控制器间共享，因此相同的意图模型可应用于不同具身，从我们的主要灵巧手到平行夹爪。我们在五项真实世界操作任务上评估了LUCID：仅由互联网视频监督的搅拌、擦拭和装箱，并零样本迁移到新场景和物体实例；以及由各自1小时自收集智能手机视频监督的推-T和电缆布线。项目页面：https://lucid-robot.github.io/。

## Abstract
The most widely-adopted robot learning pipelines today learn skills from robot demonstrations or structured human data, which are expensive to collect and tied to specific embodiments. In contrast, unstructured human videos provide a scalable alternative. They contain diverse manipulation demonstrations across objects, scenes, and strategies, but are not directly connected to robot action. We propose LUCID, a two-stage framework that learns task intent from unstructured human videos drawn from internet-scale datasets and learns robot control in massively-parallel simulation. The intent model predicts short-horizon intent (what should happen next in the scene) from the current observation in closed loop. An embodiment-specific sensorimotor policy converts this intent into robot actions. The intent interface is shared across controllers, so the same intent model can be applied to different embodiments, from our primary dexterous hand to a parallel-jaw gripper. We evaluate LUCID on five real-world manipulation tasks: stirring, wiping, and binning supervised by only internet video, with zero-shot transfer to novel scenes and object instances; and push-T and cable routing supervised by 1 hr each of self-collected smartphone video. ## 1. 论文的核心问题与整体含义（研究动机和背景）

- 当前机器人技能学习的主流范式依赖**机器人演示**或**结构化人类数据**（如动作捕捉、穿戴设备、手持夹爪），这些数据收集成本高、耗时，且与特定机器人本体绑定，难以扩展到多样化的任务和场景。
- 另一方面，**非结构化人类视频**（如互联网视频）数量巨大、内容丰富，包含各种物体、场景和操作策略，但缺少机器人的动作标签，无法直接用于训练。
- 物理仿真可以大规模生成带有动作标签的数据，但每个任务都需要手动设计奖励函数，尤其对于高层意图很难定义。
- **核心问题**：如何将这两种互补的数据源结合，实现从易获取的人类视频中学习与机器人无关的意图，并利用仿真训练鲁棒的控制策略，从而**摆脱对昂贵演示数据的依赖**，实现可扩展、跨本体的灵巧操作技能获取。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**意图-控制分离**。将机器人操作分解为：
  - **意图模型（Intent Model, fθ）**：从非结构化人类视频中学习，预测**短视界**（1秒）的物体运动轨迹（物体流）和手掌姿态（palm pose），不依赖任何机器人本体信息。
  - **感觉运动策略（Sensorimotor Policy, π）**：在仿真中通过**大规模并行强化学习**训练，以意图模型输出的参考轨迹为目标，输出机器人动作，实现具体本体的控制。
  - 两者通过**显式的短视界参考**（物体流 + 手掌姿态）通信，意图模型在部署时**闭环地**从实时场景中反复生成参考，而非一次性规划。

- **关键技术细节**：
  - **意图模型（fθ）**：
    - 基于 **CoTracker3** 点跟踪 Transformer，修改为**前向预测**（从过去帧预测未来帧的轨迹）并与 **DINOv3** 特征融合，通过残差适配器融合深度信息。
    - 输入：近期 RGB-D 帧堆叠、当前物体查询点3D位置、当前手掌姿态；输出：未来物体点轨迹和手掌姿态序列。
    - 训练数据从无结构人类视频中自动提取：ViPE（深度/相机位姿）、SAM 3.1（分割）、DenseTrack3Dv2（3D点跟踪）、WiLoR（手部网格）+ 刚性对齐（palm pose）。并施加人类区域增强，避免依赖演示者手部外观。
  - **感觉运动策略（π）**：
    - 在 **Isaac Lab** 中训练，使用**程序化生成的物体**（约1000个形状，覆盖 blob、工具、平板等）和**程序化生成的参考轨迹**（approach → in-hand motion → goal → disengage），模拟多样操作需求。
    - 分**教师（πT）**和**学生（πS）**训练：教师使用特权信息（物体完整表面点、位姿等）通过 PPO 训练；学生通过**混合目标（PPO + 蒸馏）**从教师蒸馏，输入为外置 RGB-D 相机可见点、腕部深度图像和本体感受。
    - 动作空间：包括手臂关节增量、**特征抓取基（Eigen-grasp）系数**（基于人类抓取数据降维）和手指残差，有效减小探索空间。
    - 奖励函数为**任务无关**，平衡物体跟踪、手掌姿态跟随、接触奖励和正则化。
    - 采用**自适应域随机化（ADR）**：随策略能力提升增加重力、扰动、精度要求，实现鲁棒的 Sim-to-Real 迁移。
  - **部署流程**：
    - 使用 SAM 3.1 实时更新物体掩码，DenseTrack3Dv2 进行滑动窗口点跟踪，每1秒重新查询意图模型，控制策略以50Hz运行。
    - 意图模型输出从外置相机帧转换到机器人基座帧。

## 8. 不足与局限

- **管道脆弱性**：系统高度模块化（SAM、DenseTrack3D、ViPE、WiLoR、policy），每个环节都是潜在故障点。部署中遮挡、纹理缺失等可导致级联错误。
- **任务条件差距**：缺乏明确终止条件的任务（如搅拌）需要外部强制停止。新任务需要手动过滤训练数据（按名词查询），难以扩展到上千个任务。
- **接口信息丢失**：3D流+手掌姿态的显式接口丢弃了手指配置、接触细节等有用信息，即使意图准确，策略也可能缺少关键线索复现人类演示。理想情况下应使用联合优化的潜在接口。
- **数据依赖**：意图模型仍需要每个任务对应的训练视频片段（20k片段/任务）；自采集任务需要1小时视频，虽然远少于演示数据，但仍有收集门槛。未探索零样本任务泛化（例如文本提示）。
- **环境泛化限制**：虽然在桌面上零样本迁移到新物体和场景，但涉及动态物体、复杂背景、不同光照等真实世界变异未广泛测试；失败模式（附录 C.2）显示不可恢复状态和感知丢失比例不低。
- **计算资源要求**：意图模型训练和策略训练均需高端 GPU（A100、RTX 5090），部署也需 RTX 5090，可能限制资源有限用户复现。
- **统计量**：真实实验每个条件10-15次试次，可能不足以完全消除随机性影响，但论文提供了失败模式分类辅助理解。
