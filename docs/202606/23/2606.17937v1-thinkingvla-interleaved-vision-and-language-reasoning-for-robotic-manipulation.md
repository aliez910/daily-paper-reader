## 摘要
大多数视觉-语言-动作（VLA）模型直接将观测映射到动作，缺乏显式推理，限制了它们在推理密集型长时程任务中的能力。为应对这一问题，现有方法采用思维链（CoT）推理，实现了子目标分解和空间预判。然而，这些方法缺乏用于高效跨模态推理的统一架构，且未能明确包含基于目标状态的逆推理能力。我们认为，操作规划自然地分解为预测（预见下一个视觉状态）和逆动力学（推断达到该状态的动作）。连接两者需要一种统一的、在单一生成过程中交错文本和视觉推理的自回归架构。我们提出ThinkingVLA，一种在统一混合变换器（Mixture-of-Transformers）架构中实现这一分解的生成模型。ThinkingVLA包括一个前向CoT，用于识别当前子目标并引导视觉预测；预测的图像作为目标状态，为逆向CoT提供基础，逆向CoT基于预测图像推理空间关系和动作意图；最终动作在完整的推理上下文中生成。在仿真和真实世界基准上的大量实验表明，ThinkingVLA一致地优于最先进的基线方法，在长时程操作任务上尤其表现出显著优势。

## Abstract
Most Vision-Language-Action (VLA) models map observations directly to actions without explicit reasoning, limiting their capacity for reasoning-intensive long-horizon tasks. To address this, existing approaches adopt Chain-of-Thought (CoT) reasoning to enable subgoal decomposition and spatial anticipation. However, those methods lack a unified architecture for effective cross-modal reasoning and fail to explicitly include inverse reasoning ability based on the target state. We argue that manipulation planning naturally decomposes into prediction, anticipating the next visual state, and inverse dynamics, inferring the actions to reach it. Bridging both requires a unified autoregressive architecture that interleaves textual and visual reasoning in a single generation process. We propose \textbf{ThinkingVLA}, a generative model that realizes this decomposition within a unified Mixture-of-Transformers architecture. ThinkingVLA consists of a forward CoT that identifies the immediate subgoal and guides the visual forecasting; the predicted image then serves as the target state, grounding an inverse CoT that reasons about spatial relationships and action intent based on the predicted image; and the final action is generated conditioned on this full reasoning context. Extensive experiments on simulation and real-world benchmarks demonstrate that ThinkingVLA consistently outperforms state-of-the-art baselines, with particularly large gains on long-horizon manipulation tasks.

- 现有视觉-语言-动作（VLA）模型直接将观测映射到动作，缺乏显式推理步骤，导致在推理密集型长时程操作任务中表现受限。
- 为引入推理能力，已有方法采用思维链（Chain-of-Thought, CoT）策略，分为三类：
  - **单模态 CoT**（纯文本或纯视觉）：仅提供子目标分解或空间预判之一，无法兼顾两者。
  - **解耦多模态 CoT**：分别生成文本和视觉推理，但缺乏跨模态一致性，且没有基于预测图像进行显式逆推理，逆动力学过程隐含。
- 针对以上不足，论文认为操作规划应自然分解为两部分：
  - **预测（Prediction）**：预判下一时刻的视觉状态。
  - **逆动力学（Inverse Dynamics）**：推断达到该状态所需的动作。
- 连接两者需要统一的**交错视觉-语言推理**架构，在单一自回归序列中交替生成文本推理和视觉预测，使得预测图像能作为桥梁，显式地连接推理与动作生成。

## 2. 方法论：核心思想、关键技术细节、公式与算法流程

### 2.1 核心思想
- 提出 **ThinkingVLA**，基于 **Mixture-of-Transformers (MoT)** 统一架构，将推理和动作生成集成在一个端到端可训练的模型中。
- 每个决策步骤生成一条交错推理链：**前向 CoT → 预测图像 → 逆 CoT**，然后基于完整上下文生成动作。

### 2.2 架构细节
- **统一分词器**：文本使用 PaliGemma 分词器，图像使用 SigLIP 编码器 + Cosmos 离散图像分词器，所有输入输出映射到共享词汇表。
- **思考专家（Thinking Expert）**：基于 PaliGemma（~2B 参数），负责生成前向 CoT、预测图像、逆 CoT。训练序列格式为：
  [BOS] {观测} {指令} [think] {前向 CoT} [/think] [gen] {预测图像} [/gen] [think] {逆 CoT} [/think] [EOS]
  文本与图像 token 在同一自回归过程中生成，实现跨模态直接条件化。
- **动作专家（Action Expert）**：基于 300M 参数的 Gemma 变压器，采用流匹配（Flow Matching）生成动作 chunk。
- **混合注意力机制**：每层两个专家共享注意力，但各自有自己的 QKV 投影和 FFN；注意力 mask 允许上下文双向、推理因果、动作专家关注完整推理前缀但对思考专家不可见。

### 2.3 训练目标
- **文本损失**：对前向 CoT 和逆 CoT 的 token 进行交叉熵。
- **图像损失**：对预测图像的离散 token 进行交叉熵。
- **动作损失**：流匹配条件下的均方误差（预测速度场）。
- 总损失：\(\mathcal{L} = \lambda_{\text{text}} \mathcal{L}_{\text{text}} + \lambda_{\text{img}} \mathcal{L}_{\text{img}} + \lambda_{\text{action}} \mathcal{L}_{\text{action}}\)

### 2.4 三阶段训练策略
1. **阶段 1（生成推理预训练）**：仅训练思考专家，生成前向 CoT 和未来图像，无动作预测。使用 Open X-Embodiment 子集（25 个数据集，约 2730 万帧）。
2. **阶段 2（端到端策略学习）**：引入动作专家，完整推理链 + 动作生成，权重 λ_text=1, λ_img=2, λ_action=10。
3. **阶段 3（目标机器人适配）**：在目标数据集（RoboTwin 2.0 或 ALOHA 真实演示）上微调，扩展为多视角图像预测，λ_text=λ_img=1, λ_action=10。

### 2.5 推理策略
- **分类器自由引导（CFG）**：仅应用于图像生成，融合无条件、仅图像条件、完整条件三条分支的对数几率，使用 s_I=1.5, s_T=2。
- **推理 Dropout**：训练时以 0.1 概率独立丢弃前向/逆 CoT，使模型能在推理时选择性跳过 CoT 以降低延迟。
