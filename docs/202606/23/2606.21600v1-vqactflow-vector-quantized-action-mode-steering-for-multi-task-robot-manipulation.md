## 摘要
多任务机器人操作策略从演示中学习是具有挑战性的，因为传统上单个网络必须根据语言和视觉上下文，从多模态演示分布中在性质不同的动作模式之间进行选择。错误的模式选择意味着执行错误的任务或场景中不可行的动作。将连续动作标记化为学习到的离散码本，在表示层面上将这些模式分开，为多任务学习提供了结构上的优势。我们提出了VQActFlow，一种多任务操作策略，它将动作块进行标记化，并通过变分流匹配生成代码序列。VQActFlow在整个生成过程中保持对动作模式的显式偏好。推理时的引导作用于这种偏好，以引导模式承诺。我们通过语言条件的无分类器引导实例化这一点，它将策略引导至所指示的动作模式，以及一个学习到的码本批评者，它提供补充的可行性信号。我们在三个平台上评估VQActFlow：LIBERO仿真基准测试、执行全身拾取和放置的Unitree G1人形机器人，以及执行接触丰富任务的ALOHA风格双臂平台。在这些基准测试中，VQActFlow优于连续和离散基线。

## Abstract
Multi-task robot manipulation policies are challenging to learn from demonstration because traditionally a single network must select among qualitatively different action modes from a multimodal demonstration distribution, conditioned on language and visual context. A wrong mode selection means executing the wrong task or an action infeasible in the scene. Tokenizing continuous actions into a learned discrete codebook separates these modes at the representation level, offering structural advantages for multi-task learning. We propose VQActFlow, a multi-task manipulation policy that tokenizes action chunks and generates code sequences via Variational Flow Matching. VQActFlow maintains an explicit preference over action modes throughout generation. Inference-time guidance acts on this preference to steer mode commitment. We instantiate this with classifier-free guidance over language conditioning, which steers the policy toward the instructed action mode, and a learned codebook critic that supplies a complementary feasibility signal. We evaluate VQActFlow on three platforms: the LIBERO simulation benchmarks, a Unitree G1 humanoid performing whole-body pick-and-place, and an ALOHA-style bimanual platform performing contact-rich tasks. Across these benchmarks, VQActFlow outperforms both continuous and discrete baselines.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：多任务机器人操作策略需要从语言指令和视觉观察中选择一个动作模式执行，但演示数据是多模态的——同一观察可能对应不同任务的不同动作模式。传统方法在连续动作空间建模，缺乏对离散动作模式的显式表示，导致推理时难以正确“承诺”于指令指定的模式，容易执行错误任务或不可行动作。
- **背景**：
  - 扩散策略 / 流匹配是当前主流，但它们在连续空间生成，模式选择被隐式地混合在单一连续分布中。
  - 离散动作表示（如 VQ-VAE）能将动作模式分开，但已有方法（如 Discrete Policy）在生成后缀才进行离散量化，丢失了生成过程中的离散结构。
- **整体含义**：VQActFlow 通过在整个生成过程中保持对离散动作模式的显式分类偏好（logits），并结合推理时引导机制（CFG 和码本批评者），使策略能够正确、可干预地选择任务指定的动作模式，从而提升多任务操作的成功率。

## 2. 论文提出的方法论

### 2.1 核心思想
1. **动作标记化**：使用 VQ-VAE 将连续动作块压缩为离散码本索引序列，每个索引对应一个动作模式。
2. **变分流匹配生成**：训练一个 DiT 骨干网络，在生成过程中产生每个码本位置上的 logits（显式偏好），并通过 logits 计算后验均值作为端点，导出速度场进行 ODE 积分。
3. **推理时引导**：通过两种机制修改生成过程中的 logits，从而引导模式承诺：
   - **CFG（无分类器引导）**：在 logits 层对语言条件进行外推，强化任务身份。
   - **码本批评者**：一个对比学习训练的小型 Transformer，输出可行性分数，其梯度调整当前分布，使其趋向场景可行的动作模式。

### 2.2 关键技术细节
- **VQ-VAE 动作标记化**（Sec. IV-A）
  - 编码器：时序 CNN，将动作块 `a1:H` 映射到 `L` 个连续潜变量（`L << H`，通过 S 个下采样阶段）。
  - 量化：最近邻搜索到码本 `C = {e_k}`，大小为 K。
  - 解码器：时序 CNN 重构动作块。
  - 冻结后为 VFM 提供监督目标（目标 embeddings `x1` 和 ground-truth 索引）。

- **变分流匹配策略**（Sec. IV-B）
  - **训练**：采样噪声 `x0 ~ N(0,I)`、时间 `t ~ U(0,1)`，构造 OT 插值 `xt = (1-t)x0 + t x1`。策略网络 `fθ(xt, t, o, c)` 输出每个位置上的 logits `h(l) ∈ R^K`。训练使用交叉熵损失（Eq.6）。
  - **推理**：通过 logits 软化后计算后验均值 `μ1 = Σ p_k·e_k`，速度场 `vθ = (μ1 - xt)/(1-t)`。从 `x0` 开始积分至 `t=1`，最终量化得到代码序列，解码为动作。

- **CFG 对语言条件的引导**（Sec. IV-C）
  - 同时计算条件 logits `hc` 和无条件 logits `h∅`（语言置为 ∅），外推得 `˜h = h∅ + w·(hc - h∅)`，用 `˜h` 计算分布和速度。

- **码本批评者**（Sec. IV-D）
  - 结构：小型 Transformer encoder，输入 `(o, c, p, t)`，输出标量可行性分数。
  - 训练：InfoNCE 损失，正样本为当前观测、语言、代码分布；负样本包括时序打乱、随机替换、错误观测/语言配对、跨演示交换等。
  - 推理引导：计算批评者关于 `p` 的梯度（保持切方向投影到单纯形）并更新分布：`˜p = Proj_Δ[p + λ(t)·Π_T[∇_p Cψ]]`。其中 λ(t) 随分布熵自适应，熵高时弱引导，熵低时强引导。

### 2.3 算法流程（文字说明）
- **Stage 1**：训练 VQ-VAE，编码器将动作块压缩为离散索引，解码器重构。
- **Stage 2**：训练 VFM 策略网络，输入加噪后的编码器输出（embeddings），输出每个码本位置上的 logits，用交叉熵拟合真实索引。
- **推理**：给定语言指令和观测，策略从噪声开始逐步积分，每一步:
  1. 计算条件 logits 和无条件 logits（CFG），若启用 CFG 则外推。
  2. 若启用码本批评者，则计算批评者分数对分布的梯度，按自适应权重调整。
  3. 用调整后的分布计算后验均值，导出速度。
  4. 执行 Euler 步。
  5. 积分结束，量化得到代码序列，解码执行。

## 6. 论文的主要结论与发现

- **显式分类接口使 CFG 更有效**：在 LIBERO-Goal 上 VQActFlow 在 w=4 达到 81.0%（峰值），而 Discrete Policy 在 w=6 只有 61.5%，且 VQActFlow 方差更小。
- **码本批评者提供互补信号**：在 LIBERO-90 上，批评者单独 +2.0%，与 CFG 结合再 +2.9% 达到 80.5%；在双臂任务上从 73.8% (CFG only) 提升到 77.5%。
- **CFG 主要减少错误任务，批评者改善执行精度**：人形实验 CFG 将错误任务率从 41.3% 降至 2.5%，批评者对插入/放置精度任务提升明显。
- **离散表示助力运动平滑性**：CFM 在双臂任务上产生高抖动（峰值 jerk ~10^4 rad/s³），而 VQActFlow 仅 ~2×10² rad/s³，平滑两个数量级。
- **最优代码本大小**：K=512 平衡了分辨率与分类难度，过大或过小均降低性能。

## 7. 优点

- **方法论创新**：首次在机器人策略中明确将离散动作模式表示与变分流匹配结合，并在生成全程维持显式偏好，打开了一个有效的推理时介入接口。
