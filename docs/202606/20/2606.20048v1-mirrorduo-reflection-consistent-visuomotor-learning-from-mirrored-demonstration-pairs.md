## 摘要
基于图像的行为克隆利用来自普遍存在的RGB摄像头的演示。然而，它仍然受到收集多样化演示成本的限制，特别是在跨工作空间变化进行泛化时。我们提出MirrorDuo，一种基于反射的公式，它处理图像、本体感受和完整的六自由度末端执行器动作元组，为每个原始演示生成镜像对应，有效实现“收集一个，免费获得一个”。它可以作为现有学习流程（如标准行为克隆或扩散策略）的数据增强策略，或作为反射等变策略网络的结构先验。通过利用原始域和镜像域之间的重叠，当演示在工作空间两侧均匀分布时，MirrorDuo在相同数据预算下实现了显著提高的性能。当演示仅限于一侧时，MirrorDuo能够高效地将技能迁移到镜像工作空间，在目标布置中只需零个或五个演示。

## Abstract
Image-based behaviour cloning leverages demonstrations captured from ubiquitous RGB cameras. However, it remains constrained by the cost of collecting diverse demos, especially for generalizing across workspace variations. We propose MirrorDuo, a reflection-based formulation that operates on image, proprioception, and full 6-DoF end-effector action tuples, generating a mirrored counterpart for each original demonstration, effectively achieving "collect one, get one for free". It can be applied as a data augmentation strategy for existing learning pipelines, such as standard behaviour cloning or diffusion policy, or as a structural prior for reflection-equivariant policy networks. By leveraging the overlap between the original and mirrored domains, MirrorDuo achieves significantly improved performance under the same data budget when demonstrations are evenly distributed across both sides of the workspace. When demonstrations are confined to one side, MirrorDuo enables efficient skill transfer to the mirrored workspace with as few as zero or five demos in the target arrangement.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：基于图像的模仿学习（Behaviour Cloning, BC）依赖从 RGB 相机采集的演示数据，但在需要跨工作空间布局（如对象位置、场景不对称）泛化时，收集足够多样化的演示成本很高。相比之下，基于 3D 表示（如点云）的方法可利用 SE(3) 等变性，通过空间变换扩大数据覆盖，而 2D 图像难以直接应用这些变换。
- **核心问题**：如何利用 **反射对称性**（左右镜像）这一在机器人操作任务中普遍存在的结构，来提升图像模仿学习的数据效率和泛化能力。
- **研究动机**：许多操作任务天然存在镜像变体（如左侧抓放对应右侧），若能借助反射变换将单侧演示“免费”转换为另一侧的可用样本，就能显著降低数据收集成本，并迁移技能到新的工作空间。

## 2. 方法论

### 核心思想
- **MirrorDuo**：一个通用框架，对原始演示中的 **RGB 图像（手眼相机 + 第三人称相机）**、**本体感知** 和 **完整 6‑DoF 末端执行器动作** 联合进行反射一致变换，生成语义和物理一致的镜像演示对。实现“收集一个，免费得到一个”。
- **两种实现形式**：
  - **MirrorAug**：作为数据增强策略，在批训练时随机替换部分样本为它们的镜像版本。
  - **MirrorDiffusion**：作为反射等变先验，在扩散策略中嵌入反射等变性，使噪声预测函数与反射操作对易。

### 关键技术细节

1. **姿态空间镜像操作**：
   - 在标准针孔相机模型下，镜像姿态 `C_X*_H = E * C_X_H * E`，其中 `E = diag([-1, 1, 1, 1])`。
   - 等价地，在局部坐标系（delta 姿态 / relative 姿态）中镜像操作消除对相机外参的依赖：`ΔX*_H = E * ΔX_H * E`, `δX*_H = E * δX_H * E`。

2. **SO(3) 不连续性问题处理**：
   - 镜像可能导致初始姿态发生突变。通过将轨迹初始姿态的旋转对齐到世界 X 轴（平均初始旋转的 x 方向），使镜像操作保持初始姿态不变。

3. **向量化镜像规则**：
   - 将 SE(3) 姿态展平为 9 维向量（位置 + 两列旋转向量），镜像等价于元素乘向量 `ρ = [-1, 1³, -1³, 1²]^T`，便于神经网络处理。

4. **MirrorDiffusion 实现**：
   - 使用 E(n)-等变 Steerable CNN 构建 ResNet（对 Flip2dOnR2 群等变），确保噪声预测函数与反射对易。
   - 然而，扩散过程每一步注入的独立噪声会破坏全局反射对称性；该偏差在实验中通过少量真实镜像演示（5 条）即可补偿。

5. **视觉不对称处理**：
   - 镜像可能引入机器人本体不对称（如手腕左右）、背景纹理差异等。采用 ImageNet 预训练的 ResNet-18 和 **Random Overlay** 数据增强来提升视觉鲁棒性。

## 6. 主要结论与发现

1. **数据效率大幅提升**：
   - 在双侧演示设置中，使用 MirrorAug 或 MirrorDiffusion 在 **低/中数据量**（50/100 条）时增益显著（如 Stack Three 从 20.7%→51.3%，Square D2 从 25.3%→32.7%）。
   - 在单侧演示中，只需 **0–5 条** 镜像域演示，即可实现接近原域的性能迁移。

2. **零样本迁移可行但需视觉泛化**：
   - 在近对称视觉条件下，Diffusion Policy + MirrorAug + Random Overlay + 预训练 可达到 93% 的原域成功率。
   - 在视觉不对称较严重时（广视角、不同背景），零样本迁移成功率大幅下降，但加入少量（5～10 条）镜像域演示即可恢复。

3. **MirrorDiffusion vs MirrorAug**：
   - MirrorDiffusion 在视觉简单场景中与 MirrorAug 表现相当，但计算成本更高。
   - MirrorDiffusion 在零样本迁移时因扩散噪声破坏等变性而失败，但加入极少量数据即可解锁等变性优势。
   - **推荐实际使用 MirrorAug + Diffusion Policy + Random Overlay + 预训练**，在不对称和有限数据下表现更好且训练更快。

4. **高数据量时的边际下降**：
   - 在 500 条演示时，MirrorAug 在部分任务（Square D2, 3-Part Assembly）出现约 2–4% 的成功率下降，分析认为镜像演示可能引入冲突的视觉/动作分布，使决策边界碎片化。

## 7. 优点

- **新颖的对称性利用**：首次系统地将反射对称性作为数据增强和等变先验应用于图像模仿学习，填补了 2D 操作学习中对 SE(3) 之外对称性的利用空白。
- **方法通用且即插即用**：MirrorAug 可无缝添加到现有 BC / 扩散策略流程中，无需修改模型架构。
- **实验设计全面**：覆盖仿真多个任务、多种视觉难度、多种演示预算；真实实验验证了方法的实际可行性。
- **实用取向**：综合性能、效率、易用性后给出清晰推荐（Diffusion + MirrorAug + O + P），而非坚持理论最优。
- **理论分析扎实**：详细推导了镜像在 SE(3) 上的代数表示、不连续性处理及与等变框架的结合。

## 8. 不足与局限

- **依赖视觉泛化技术**：直接零样本迁移在视觉不对称下几乎失败，必须配合 Random Overlay 和预训练编码器才能有所表现；推广性对视觉增强技术敏感。
- **高数据量时的副作用**：镜像可能引入多模态冲突（如不同旋转方向对应相似视觉输入），在数据充足时反而轻微伤害性能。
- **仅考虑反射对称**：未推广到其他离散对称群（如 90° 旋转、更一般的不变群）；也未考虑非对称任务（如只能单手操作的阀门）。
- **计算开销**：MirrorDiffusion 的等变架构显著增加训练耗时，而 MirrorAug 则几乎没有额外开销。
- **特定相机设置限制**：对侧方相机、过肩相机可能需要额外的居中预处理才能适用；仅进行了初步验证。
- **未提供详细算力资源**：无法复现所需计算量，不利于社区资源评估。
- **泛化假设**：要求任务本身具有左右对称性，且演示集中在某一侧；对于完全不对称场景不适用。
