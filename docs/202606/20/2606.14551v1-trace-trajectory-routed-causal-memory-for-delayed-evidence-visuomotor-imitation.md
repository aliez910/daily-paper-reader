## 摘要
自主运行的机器人可能需要基于不再可见的证据做出决策。我们研究“延迟证据”任务，其中早期线索在稍后的决策点之前消失，因此视觉上相似的观测可能需要不同的动作。在这些设置中，当前的观测不足以作为控制的状态。我们介绍了TRAjectory-routed Causal Evidence (TRACE)，一种用于视觉运动模仿策略的记忆框架。TRACE将任务相关的视觉和机器人状态证据（如物体身份、目标选择或路线相关状态）存储在固定大小的潜在记忆中，该记忆在长时间片段中保持有界。TRACE不是通过原始时间或手动提供的任务标签来索引记忆，而是使用“路径签名”：执行机器人状态轨迹的紧凑、顺序敏感的特征。这些签名不存储视觉线索本身；相反，它们提供轨迹条件化的键，用于写入和检索线索可见时存储的证据。当机器人后来遇到模糊的观测时，策略以TRACE记忆为条件，恢复缺失的上下文并选择正确的分支。TRACE通过轻量级适配器附加到策略上，而不改变策略主干、动作头或模仿目标。在具有视觉模糊分支点的真实世界长视域操作任务中，TRACE在分支选择和任务成功率上优于其他基线方法，包括短历史和循环记忆。项目页面：https://jeong-zju.github.io/trace

## Abstract
Robots under autonomous operation may require decisions based on evidence that is no longer visible. We study \emph{delayed-evidence} tasks, where an early cue disappears before a later decision point, so visually similar observations can require different actions. In these settings, the current observation is not a sufficient state for control. We introduce TRAjectory-routed Causal Evidence (TRACE), a memory framework for visuomotor imitation policies. TRACE stores task-relevant visual and robot-state evidence, such as object identity, target choice, or route-dependent state, in a fixed-size latent memory that remains bounded over long episodes. Instead of indexing memory by raw time or manually provided task labels, TRACE uses \emph{path signatures}: compact, order-sensitive features of the executed robot-state trajectory. These signatures do not store the visual cue itself; rather, they provide trajectory-conditioned keys for writing and retrieving the evidence stored when the cue was visible. When the robot later reaches an ambiguous observation, the policy conditions on TRACE memory to recover the missing context and choose the correct branch. TRACE attaches through lightweight adapters to policies, without changing the policy backbone, action head, or imitation objective. Across real-world long-horizon manipulation tasks with visually ambiguous branch points, TRACE improves branch selection and task success over alternative baselines, including short-history and recurrent memory. ## 2. 方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
TRACE 提出一种固定大小的因果记忆模块，其读写行为由机器人状态轨迹的路径签名（path signatures）控制。核心思想是将记忆内容（视觉-状态证据）与记忆寻址（轨迹派生键）分离。

- **路径签名**：对机器人状态轨迹进行分段线性插值后计算截断路径签名 \( \xi_t = \operatorname{Sig}^{\leq p}(S_t) \) 及其增量 \( \delta_t = \xi_t - \xi_{t-1} \)。签名是紧凑、对顺序敏感的轨迹描述子，对单调时间重参数化具有不变性，能反映轨迹的展开方式。
- **轨迹键生成**：将 \( \xi_t \) 和 \( \delta_t \) 分别通过学习到的映射 \( \phi_g, \phi_\Delta \) 编码为特征 \( g_t, \Delta g_t \)，再组合成轨迹键 \( q_t = \phi_q([g_t, \Delta g_t]) \)。
- **路由与读写**：轨迹键 \( q_t \) 通过路由状态 \( \rho_t = \phi_\rho(q_t) \) 与上一时间步的记忆槽 \( m_{t-1,k} \) 计算亲和度，得到软赋权向量 \( \omega_t \) 决定哪些槽被选中。写入时，当前观测编码 \( e_t \) 与轨迹键结合生成候选内容 \( \tilde{m}_{t,k} \)，通过门控更新 \( m_{t,k} = (1-\beta_{t,k})m_{t-1,k} + \beta_{t,k}\tilde{m}_{t,k} \)，其中 \( \beta_{t,k} \) 受路由权重调制。读取时，基于当前轨迹和证据生成查询，通过注意力从更新后的记忆槽中读出 \( z_t^{\text{mem}} \)。
- **策略适配器**：共享的记忆状态 \( R_t = (M_t, z_t^{\text{mem}}, g_t, \Delta g_t) \) 通过轻量适配器 \( A_\theta^{(e)} \) 转换为基策略的本地条件表示。对于回归策略（如 ACT），适配器将记忆槽和摘要令牌拼接到注意力记忆中；对于扩散策略，适配器通过零初始化的 MLP 输出一个全局条件向量。
- **训练**：在固定预算的因果历史序列上按时间步扫描，应用与部署相同的更新器，优化基策略的原始损失（如 chunk L1 损失或扩散去噪损失），并辅以平衡、熵和一致性三项辅助损失以稳定记忆读写。

## 6. 主要结论与发现
- TRACE 显著提升延迟证据操作任务的表现：TRACE Regression 平均进度 69.23%，TRACE Diffusion 平均进度 59.53%，远超对应的基策略（ACT 25.50%，Diffusion 25.00%）以及最强的 VLA 基线 π0.5（50.47%）。
- 增益跨策略家族迁移：TRACE 使用相同的共享记忆模块，仅更换轻量适配器，即可同时改进回归和扩散两种策略。
- 优于通用历史记忆：在相同的回归基策略下，TRACE 平均进度 69.23%，而 GRU（45.83%）、Transformer 上下文（52.10%）、LRU（53.93%）、检索提示（56.30%）均明显低于 TRACE，说明仅增加历史容量或通用循环不足以解决问题，需要轨迹导向的读写结构。
- 诊断确认 TRACE 依赖有序历史证据：保持顺序的历史变换（时间重采样、速度抖动等）下内存路由和分支读出一致性接近完整 TRACE 参考值（约 90-96%），而顺序反转（负对照）则大幅降至 37.8%（路由相似度）和 41.6%（分支一致性），证明 TRACE 提取并利用了因果顺序信息而非仅靠当前观测。

## 7. 优点
- **模块化设计**：TRACE 作为一个独立模块，通过轻量适配器集成到现有策略中，不修改策略主干、动作表示或模仿损失，易于在多种策略家族间迁移。
- **创新的记忆寻址**：利用路径签名作为轨迹描述子，提供紧凑、顺序敏感且对时间重参数化鲁棒的键，从而将记忆读写与机器人到达当前状态的“路径”绑定，而非与原始时间戳绑定。
- **固定预算长时程运行**：通过固定大小的记忆槽和在线更新，能在长时间段内保持内存有界且信息不会无限增长。
- **丰富的诊断与消融**：除了主实验外，还设计了历史变换诊断、留一任务分析、长时稳定性检查等，详细展示了记忆的行为模式和因果依赖关系，增强了结论的可靠性。
