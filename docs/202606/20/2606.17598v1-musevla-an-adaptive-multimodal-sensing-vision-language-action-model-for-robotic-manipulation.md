## 摘要
人类自然地利用多种感知模态与物理世界交互，而大多数面向机器人的视觉-语言-动作（VLA）模型仅依赖RGB观测。这限制了它们感知难以或无法从RGB相机推断的物理属性的能力，例如温度、声音或雷达响应。我们提出MuseVLA，一种适应性多模态感知VLA模型，将新型传感器集成为机器人操作的按需工具。给定任务指令和视觉上下文，MuseVLA首先生成一个传感器令牌和目标描述，选择要调用的感知模态和关注对象，类似于带参数的工具调用。然后将选定的传感器测量转换为接地传感器图像，这是一种统一的中间表示，编码异构读数以进行多模态融合和动作生成。这种设计将传感器特定处理与VLA主干解耦，从而能够高效集成多种模态。为了减少对昂贵的多传感器机器人数据集的需求，我们进一步引入了一个数据合成流水线，用接地传感器图像增强现有的RGB视频数据集，从而实现对未见过的传感器引导任务的泛化。我们在真实机器人上评估了MuseVLA，涉及需要多模态感知输入的挑战性灵巧手操作任务，包括温度引导的拾取与放置、音频驱动的物体搜索和雷达辅助的隐藏物体检索。MuseVLA平均成功率达到80.6%，显著优于仅RGB和多传感器VLA基线，并在未见过的任务上表现出强大的零样本能力。

## Abstract
Humans naturally leverage diverse sensing modalities to interact with the physical world, while most Vision-Language-Action (VLA) models for robotics rely solely on RGB observations. This limits their ability to perceive physical properties that are difficult or impossible to infer from RGB cameras, such as temperature, sound, or radar response. We present MuseVLA, an adaptive multimodal sensing VLA model that integrates novel sensors as on-demand tools for robotic manipulation. Given a task instruction and visual context, MuseVLA first generates a sensor token and target description that select the sensing modality to invoke and what to attend to, analogous to a tool call with arguments. It then converts the selected sensor measurement into a grounded sensor image, a unified intermediate representation that encodes heterogeneous readings for multimodal fusion and action generation. This design decouples sensor-specific processing from the VLA backbone, enabling efficient integration of diverse modalities. To reduce the need for expensive multisensory robot datasets, we further introduce a data synthesis pipeline that augments existing RGB video datasets with grounded sensor images, enabling generalization to unseen sensor-guided tasks. We evaluate MuseVLA on a real-world robot across challenging dexterous hand manipulation tasks that require multimodal sensing inputs, including temperature-guided pick-and-place, audio-driven object search, and radar-assisted hidden object retrieval. MuseVLA achieves 80.6% success rate on average, outperforming RGB-only and multisensory VLA baselines significantly, and exhibits strong zero-shot capabilities on unseen tasks.

## 2. 方法论核心思想与关键技术细节

- **整体框架**：MuseVLA 将操作任务分解为三个步骤：
  1. 根据语言指令和 RGB 图像，由 VLM 生成**传感器令牌**（`<thermal>` / `<acoustic>` / `<mmWave>` / `<none>`）和**目标描述**（如“hot drink”），相当于工具调用中的参数传递。
  2. 调用选定的传感器获取原始读数，并将其转换为**接地传感器图像**（Grounded Sensor Image）：通过 SAM3 分割目标描述对应的区域，将传感器热力图（温度图、声强图、雷达反射图）仅叠加在该区域上，其余部分保留 RGB 值。这种表示统一了异构传感器数据到 RGB 空间，允许重用预训练的视觉编码器。
  3. 将接地传感器图像、传感器令牌、目标描述与原始 RGB 图像、语言指令一起送入 VLM 主干，输出条件特征，再由扩散动作专家（Diffusion Transformer）生成动作序列。

- **自适应传感器选择**：引入四个可学习的传感器令牌，模型通过分类任务直接预测需调用的传感器，避免了固定输入所有传感器带来的计算开销和噪声干扰。

- **训练损失**：
  - VLM 损失：传感器令牌的分类交叉熵损失 + 目标描述的生成交叉熵损失。
  - 扩散损失：MSE 噪声预测损失。
  - 两部分联合优化（λ=1e-2），保证 VLM 输出质量和动作生成能力同时提升。

- **数据合成流水线**：
  - 从已有 RGB 机器人数据集（MolmoAct、AgiBotWorld-Alpha、VITRA）中提取视频片段，随机选择一种传感器及其对应物理属性关键词（如 “hot” → 热像传感器）。
  - 利用 GPT-5.2 修改任务指令（如 “Pick up the hot mug”），SAM3 分割目标物体，并在目标区域叠加代表该物理属性的颜色（如黄色代表热），模拟传感器读数。
  - 使用数据增强：克隆错位区域并给予错误的传感器颜色，强制模型依赖传感器信息而非空间先验。
  - 合成 9.6K episodes / 1.05M 帧，覆盖 1000+ 物体。

## 3. 实验设计

- **机器人平台**：12-DoF Robotera XHand 灵巧手 + 传感器套件（RGB-D、2×热像仪、60GHz 毫米波雷达、6+1 麦克风阵列），所有传感器 2D 热力图与 RGB 图像对齐。

- **真实数据集**：遥操作收集 720 episodes，包含三类任务：
  - **温度引导取放**（热/冷/常温饮品）
  - **音频驱动物体搜索**（拨开覆盖物找到响铃电话）
  - **雷达辅助隐藏物体检索**（透视封闭盒子并打开有物品的盒子）
  - 以及它们的多阶段组合任务。

- **对比方法**：
  - 基线 VLA 模型：基于 π0 和 π0.5 的 RGB-only 变体（`π0-RGB`、`π0.5-RGB`）和原始传感器热力图输入变体（`π0-Raw`、`π0.5-Raw`）。
  - 自身消融变体：
    - `MuseVLA-RGB`：完全不使用任何传感器
    - `MuseVLA-Raw`：输入全部传感器热力图，无自适应选择

## 6. 主要结论与发现

- MuseVLA（无合成数据预训练）在感知操作任务上平均成功率达 **76.4%**，显著优于 RGB-only 基线（20.8%–22.2%）和原始传感器热力图基线（27.8%–33.3%）。
- 自适应传感器选择的准确率高：训练任务上传感器选择 100%、目标描述 93.5%；未见任务上传感器选择 100%、目标描述 82.0%（经过合成数据预训练后）。
- **数据合成预训练**带来了显著的零样本泛化提升：未见任务平均成功率从 27.1% 提升至 **66.7%**，其中热感知、声感知、雷达感知模态分别提升至 75.0%、56.3%、68.8%。
- 在多阶段任务（雷达→RGB 和雷达→热）中也取得 66.7%–75.0% 端到端成功率，证明了长序列中自适应切换传感器能力。
- 推理时内存占用从固定融合模式的 13.23 GB 降至 6.61 GB（仅处理一个传感器），效率和可扩展性增强。

## 7. 优点

- **自适应感知**：首次在 VLA 模型中实现任务驱动的传感器动态选择，既提高了性能（排除无关传感器噪声）又降低了计算开销。
- **统一中间表示**：接地传感器图像将异质传感器数据转化为 RGB 格式，避免了设计专用编码器，允许复用预训练视觉骨干，并自然兼容数据合成。
- **高效数据扩展**：数据合成流水线从纯 RGB 数据中自动生成多传感器训练数据，大幅降低真实多传感器数据集采集成本，且有效提升零样本泛化能力。
- **综合验证**：在灵巧手这种高难度操作平台上，使用真实机器人对三个完全不同传感模态的多个任务进行系统评测，消融和基线对比设计完善，实验规模充足。

## 8. 不足与局限

- **真实数据规模有限**：仅 720 episodes，虽然通过合成数据辅助，但更大规模真实数据可能进一步提升性能和鲁棒性。论文自身也承认这一局限。
- **对分割模块的依赖**：接地传感器图像的质量依赖 SAM3 的语义分割能力，在目标描述模糊或复杂背景时可能出错，从而影响后续操作。
- **声学任务成功率偏低**（58.3%），部分原因是抓取柔软布料（衣服、毛巾）的灵巧操作本身难度大，模型在该子任务上仍有提升空间。
- **传感器对齐简化**：使用离线单应性矩阵进行 2D 映射，假设工作深度恒定，对于大范围 3D 工作空间或移动传感器可能不够精确。
- **数据合成模拟的逼真度**：仅用颜色覆盖模拟传感器读数（如黄色表示热），无法建模真实传感器的噪声、时空分布和非线性特性，模型在遇到真实噪声时表现可能有所下降。
- **算力需求较高**：训练需要 64 块 A100 GPU，对大多数研究团队而言成本高昂，虽然推理时可由于自适应选择而减少内存占用，但总训练开销仍在。
