## 摘要
视觉-语言-动作模型（VLAs）在机器人操作中展现了强大的任务理解和泛化能力，然而全模型推理的高计算成本限制了其在低延迟、高频率闭环控制中的部署。我们提出了一种异步语义-动作解耦框架，该框架沿着现有VLA的内部语义-动作接口将语义理解与动作生成分离，无需重新设计视觉-语言主干或引入外部规划器。低频理解模块异步更新可复用的语义条件，而高频动作模块则持续输出控制动作，无需重复调用完整模型。为缓解过时语义与当前执行状态之间的时间错位，我们进一步引入了历史动作条件和时间错位训练，这提供了短程执行上下文，并提高了在过时语义条件下的反馈控制鲁棒性。在LIBERO上使用π0.5和UniVLA进行的实验，以及使用UniVLA的真实机器人部署表明，所提框架实现了高达35.6 Hz的服务器端动作模块推理吞吐量，并提供了一条低侵入性途径，无需以控制频率运行完整VLA推理即可实现高频率闭环控制。

## Abstract
Vision-Language-Action models (VLAs) have demonstrated strong task understanding and generalization in robotic manipulation, yet the high computational cost of full-model inference limits their deployment in low-latency, high-frequency closed-loop control. We propose an asynchronous semantic-action decoupling framework that separates semantic understanding from action generation along the internal semantic-action interface of existing VLAs, without redesigning the vision-language backbone or introducing an external planner. A low-frequency understanding module asynchronously updates reusable semantic conditions, while a high-frequency action module continuously outputs control actions without repeatedly invoking the full model. To mitigate the temporal mismatch between stale semantics and the current execution state, we further introduce historical action conditioning and time-misalignment training, which provide short-horizon execution context and improve feedback control robustness under stale semantic conditions. Experiments on LIBERO with $π_{0.5}$ and UniVLA, together with real-robot deployment using UniVLA, show that the proposed framework achieves up to 35.6 Hz server-side action-module inference throughput and offers a low-intrusion path to high-frequency closed-loop control without running full VLA inference at control rate.

- 现有方案多采用动作分块（action chunking），即一次性预测未来一段序列并由底层控制器顺序执行。但这并未真正解耦语义理解与动作生成：在执行整块动作期间，场景语义和任务意图是固定不变的，导致当环境变化或执行推进时，机器人仍依赖过时的语义信息。
- 先前工作多通过模型加速、动作块修正或异步推理系统来缓解，但往往仍需全模型调用或引入额外执行模块。

## 2. 论文提出的方法论

### 核心思想
- 将预训练VLA的内部计算拆分为两个异步角色：
  - **低频理解模块**：处理视觉与语言输入，定期更新可复用的语义条件（如UniVLA的最后一层隐藏状态或π0.5的全KV缓存）。
  - **高频动作模块**：基于最新缓存的语义条件、当前机器人状态和近期动作历史，在每个控制周期内连续输出动作。

### 关键技术细节
- **异步解耦**：理解模块以约3–5 Hz频率更新语义缓存；动作模块以控制频率（20 Hz）独立运行，无需等待全模型推理完成。
- **历史动作调节**：为补偿陈旧语义与当前执行状态的临时错位，将最近m步历史动作作为额外输入注入动作模块（UniVLA用轻量投影层，π0.5改为动作头条件嵌入）。
- **时间错位训练**：训练时模拟推理时的语义延迟——对时间步t的动作预测，随机使用t–Δ时刻的语义条件（Δ均匀采样自[0, H-1]，H=动作块长度）。这使动作模块学会在过时语义下保持稳定控制。
- **动作块重规划与时间集成**：每个控制周期动作模块重新规划完整的动作块，并通过ACT风格的时间加权（衰减系数0.1）合并重叠预测，输出最终执行动作。

### 公式/算法流程（文字说明）
- 训练目标：$\mathcal{L} = \mathbb{E}_{t, \Delta}[\ell_{\text{base}}(G_\phi(z_{t-\Delta}, s_t, h_t), A_t)]$，其中$\ell_{\text{base}}$为原VLA的损失函数（token预测、流匹配或动作回归）。
- 推理时：理解模块按低频$\tau_k$更新语义条件$z_{\tau_k}$；动作模块每步读取$z_{\tau_k}$、当前状态$s_t$和历史$h_t$，输出动作块，经时间集成后发送给机器人。

## 6. 主要结论与发现

1. **异步解耦可行**：LIBERO上异步变体（UniVLA-Async, π0.5-Async）成功率与同步版本基本持平（UniVLA: 93.8 vs 95.2；π0.5-Async-15k: 98.1 vs 官方96.9）。
2. **吞吐量显著提升**：服务器端动作模块推理可达35.6 Hz（UniVLA-Async，同步仅3.4 Hz），π0.5-Async达19.7 Hz（同步8.7 Hz）。
3. **历史动作调节与时间错位训练至关重要**：单独缓存复用导致成功率从92.0下降到85.2；仅加历史动作恢复到86.4；仅加延迟训练达89.0；两者结合达91.2（同步UniVLA为92.0），说明显式模拟训练-测试时间错配是核心。
4. **真实机器人验证**：UniVLA-Async在SO100上平均完成时间从17.4s降至13.1s，六任务平均成功率93.3%（同步UniVLA为78.3%），且优于SmolVLA和A2C2。
5. **低侵入性**：不需要重新设计VLA主干或添加外部规划器，仅需在动作侧进行轻量修改。

## 7. 优点

- **方法层面**
  - 利用VLA内部语义-动作接口解耦，具有普遍适用性（在π0.5和UniVLA上验证）。
  - 历史动作调节和延迟训练共同缓解陈旧语义带来的控制不稳定，设计简洁且有效。
  - 不引入额外规划器或复杂流水线，改动限定在动作侧，易于集成到现有VLA中。
- **实验层面**
  - 仿真+真实机器人双验证，真实部署覆盖两平台、六任务，含长时间序列和扰动场景。
  - 消融实验清晰拆解各组件贡献，证明提出的两个机制不可或缺。
  - 参数设置接近实际（20 Hz控制环、常用硬件），结果具有实践参考价值。

## 8. 不足与局限

- **依赖可重用中间条件**：框架要求VLA内部存在可缓存复用的语义表示（如隐藏状态或KV缓存），不是所有VLA架构都自然具备此接口。
- **陈旧语义的固有风险**：当场景发生剧烈变化或任务阶段快速转换时，缓存语义可能严重过时，当前固定刷新策略不是最优的。
- **实验覆盖范围有限**：
  - 主要针对单臂操作任务，未涉及移动操作、双臂协作、动态抓取等。
  - 未在高速动态或接触丰富的任务上验证。
- **π0.5异步版本未真机部署**：仅报告服务器端吞吐量，真实闭环可行性未完全证明。
- **通信延迟影响**：真实机器人端到端延迟包含10–15 ms通信，在控制环接近20 Hz时可能成为瓶颈。
- **训练预算差异**：π0.5异步使用的训练步数远少于全训练基线，对比时需谨慎解释。
- **未进行超参数敏感性分析**：历史窗口长度、语义刷新频率、时间错位采样范围等未深入消融。
