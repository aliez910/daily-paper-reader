## 摘要
在现实世界中，鲁棒的机器人操作不仅需要理解当前观测，还需要记忆和动力学建模。世界动作模型（WAMs）通过联合建模基于当前和历史观测的视觉预见和动作来拥有这些能力，使其成为机器人操作中有前景的范式。然而，现有的WAMs面临一个基本的权衡：具有高效推理的方法通常仅基于最近观测的有限窗口进行条件建模，因此在非马尔可夫环境中表现不佳；而保留长历史记录的方法则会产生时间和空间成本，这些成本随着序列长度大幅增长。为了解决这一挑战，我们提出了MemoryWAM，一种具有高效持久记忆的世界动作模型。MemoryWAM采用了一种混合记忆设计，结合了最近帧、事件边界锚定帧以及总结长程历史的紧凑gist令牌。一个定制的注意力机制能够检索详细的短期上下文和压缩的长期上下文，支持依赖记忆的决策制定，同时减少推理延迟和GPU内存使用。在模拟和现实世界的长周期、依赖记忆的操作任务中，MemoryWAM优于强大的视觉-语言-动作（VLA）和WAM基线模型，同时保持了良好的计算效率。

## Abstract
Robust robotic manipulation in the real world requires not only an understanding of the current observation, but also memory and dynamics modeling. World action models (WAMs) possess these capabilities by jointly modeling visual foresight and actions conditioned on both current and historical observations, making them a promising paradigm for robotic manipulation. However, existing WAMs face a fundamental trade-off: methods with efficient inference typically condition only on a bounded window of recent observations and therefore struggle in non-Markovian environments, whereas methods that preserve long histories incur time and space costs that grow substantially with sequence length. To address this challenge, we introduce MemoryWAM, a world action model with efficient persistent memory. MemoryWAM uses a hybrid memory design that combines recent frames, event-boundary anchor frames, and compact gist tokens that summarize long-range history. A tailored attention mechanism enables retrieval of both detailed short-term context and compressed long-term context, supporting memory-dependent decision-making with reduced inference latency and GPU memory usage. Across long-horizon, memory-dependent manipulation tasks in both simulation and the real world, MemoryWAM outperforms strong vision-language-action (VLA) and WAM baselines while maintaining favorable computational efficiency.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：机器人操作需要同时理解当前观测、记忆历史信息以及建模物理世界动态。世界动作模型（WAM）通过联合建模视觉预见和基于历史观测的动作预测具备这些能力，被视为有前景的范式。
- **核心问题**：现有 WAM 面临**记忆-效率权衡**：
  - 高效的 WAM（如滑动窗口）只能利用有限的历史帧，在非马尔可夫环境中（任务关键信息位于窗口外）表现差；
  - 保留完整历史的 WAM（如全 KV 缓存）虽然能记住长程上下文，但推理延迟和 GPU 内存随轨迹长度线性增长，难以实用。
- **整体含义**：论文旨在**同时实现高效的持久记忆**，在不显著增加计算开销的前提下使 WAM 能利用长程历史信息，从而在长期、依赖记忆的操作任务上达到更好性能。

## 2. 方法论

### 核心思想
- **混合记忆设计**：受人类认知三阶段记忆启发，将系统记忆分为三部分，并以统一的注意力接口供动作模型查询。
- **技术实现**：使用 Mixture-of-Transformers (MoT) 架构，包括一个视频 DiT（预测视觉动态，训练时使用）和一个动作 DiT（去噪生成动作块，推理时使用），并维护一个高效的混合记忆 KV 缓存。

### 关键技术细节
- **短期记忆**：保留最近的 `N_recent` 帧的完整视觉 token（滑动窗口），提供高保真局部上下文。
- **事件边界记忆**：保留任务开始时的少量锚定帧（`N_init` 帧）的完整 token，捕获对任务至关重要的初始场景配置。
- **gist 记忆**：对每个历史帧附加 `M` 个可学习的 **gist token**（每个帧原本有 `L` 个视觉 token，`M << L`），后续帧不直接注意该历史帧的视觉 token，而是注意其 gist token 的 KV 缓存。这实现了对长程历史的紧凑压缩。
- **压缩率**：`d = L/M`，本文中 `L=120`，`M=8`，压缩率 15×。因此长程记忆的复杂度从 O(NL) 降至 O(NL/d) = O(N·M)。
- **定制注意力掩码**（图 3）：保证训练时与推理时注意力模式一致：近期帧和锚帧可被完全注意；历史帧的视觉 token 被 evict，仅保留 gist token；动作 token 注意所有上下文。
- **推理流程**：
  1. 视频 DiT 仅对当前观测进行一次前向，更新视频侧 KV 缓存（包含短期、锚定、gist 缓存）。
  2. 动作 DiT 通过注视频侧缓存去噪生成动作块。
  3. 不进行视频生成，视频预测仅作为训练监督，降低成本。

## 6. 主要结论与发现

1. **混合记忆设计有效**：通过结合短期窗口、事件边界锚帧和压缩 gist token，MemoryWAM 在保留长程历史信息的同时，将推理复杂度从 O(N) 降至 O(N/d)（d=15），并显著减少 GPU 内存和推理延迟。
2. **性能超越基线**：在仿真 RMBench 上，MemoryWAM 平均成功率 83.0%，高于全历史 WAM 的 78.2% 和 VLA/有界 WAM 的 <11%。真实世界任务也取得同样优势。
3. **记忆组件各有侧重**：gist token 对长程历史最关键（去除后 Press Button 成功率从 87% 降至 5%）；锚帧和滑动窗口提供互补信息。
4. **持久记忆可压缩**：并非所有历史信息都需要保真存储；紧凑 gist token 配合高保真关键帧已足够，且全历史可能引入冗余噪音。
5. **效率优势显著**：即使在 1600 帧的长轨迹下，MemoryWAM 的延迟和内存仍低于 TTT/RNN 方法，而全注意力已不可接受。

## 7. 优点

- **方法创新**：首次将认知科学的混合记忆思想系统引入世界动作模型，提出 gist token + 锚帧 + 滑动窗口的机制，有效解决了记忆-效率权衡问题。
- **高效且实用**：提出后可直接在现有 WAM 框架中实现，推理时无需视频生成，适合闭环控制。压缩率 15× 显著节省显存和计算。
- **实验全面**：不仅有标准仿真基准，还包含真实世界机器人实验；对比多种记忆表达（全注意/TTT/RNN），消融充分，结果清晰。
- **开源友好**：提供项目页，使用公开模型（Wan2.2）和基准（RMBench），便于复现与后续研究。
- **兼容性强**：可适用于多种 WAM 后端（视频 DiT），动作 DiT 可与视频 DiT 共享注意力缓存，直接替换记忆层。

## 8. 不足与局限

- **继承视频扩散模型的局限**：视频 DiT 在语义理解和复杂推理上具有固有限制，可能影响需要高级常识的任务（如工具使用、多步推理）。论文在结论中明确承认这一点。
- **算力隐藏假设**：虽然训练成本通过压缩降低，但基础视频模型（5B）仍较大，对研究人员配置有门槛。训练未给出全时长（但说明是 8 GPU），且动作 DiT 初始化通过插值可能降低收敛时间，但总体仍较高。
- **真实世界任务有限**：仅两个任务、各 20 次试验，样本量较小（虽然典型的机器人论文数量如此），在更复杂或动态环境中可能需更多验证。
- **对其他类型记忆任务泛化性未检验**：只在 RMBench 和特定真实任务上测试，未在更广泛的操作基准或非操纵场景上验证记忆机制的通用性。
- **未深入讨论灾难性遗忘或状态坍缩**：虽然 gist token 压缩可能丢失部分细节，但论文未量化信息损失或与全注意力在细粒度任务（如精确放置）上的边界性能差异。
- **耦合性**：混合记忆的设计高度依赖于任务结构（锚帧依赖于感知事件边界，在连续任务中可能需自适应，但现为简单固定初期帧）。长期看可能需要动态锚帧选择机制。
