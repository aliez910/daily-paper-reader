## 摘要
视觉-语言-动作模型在机器人操作中展现出巨大潜力，然而原始语言主要用于指定任务意图，而非在高速底层执行过程中反复处理。基于这一分离，我们提出了一种受小脑-丘脑启发的视觉-动作模型（CT-VAM），用于高效的任务条件视觉运动控制。CT-VAM作为一个紧凑的局部执行策略，从双视角视觉观察、本体感觉和轻量级任务条件中预测动作块，可能实现一种实用的云边范式，其中高层语义推理由大模型处理，而快速闭环控制则在本地硬件上运行。为了有效融合异构输入，CT-VAM引入了TARS（丘脑动作路由流），一种流分离条件注意力解码器，独立路由动作、视觉和任务流，防止密集的感知令牌淹没紧凑的任务相关条件。仅凭68M参数，CT-VAM在LIBERO上取得了与更大规模VLA模型竞争的成功率，同时降低了推理延迟。结合用于异步块执行的流一致修复，CT-VAM支持高频控制，并在资源受限的机器人平台上展示了鲁棒的现实世界部署能力。

## Abstract
Vision-language-action models have shown strong promise for robot manipulation, yet raw language is primarily needed to specify task intent rather than to be repeatedly processed during high-frequency low-level execution. Motivated by this separation, we propose a cerebello-thalamic-inspired vision-action model (CT-VAM) for efficient task-conditioned visuomotor control. CT-VAM acts as a compact local execution policy that predicts action chunks from dualview visual observations, proprioception, and a lightweight task condition, potentially enabling a practical cloud-edge paradigm in which high-level semantic reasoning can be handled by large models while fast closed-loop control runs on local hardware. To fuse heterogeneous inputs effectively, CT-VAM introduces TARS (Thalamic Action Routing Stream), a stream-separated conditional attention decoder that independently routes action, visual and task streams, preventing dense sensory tokens from overwhelming compact task-relevant conditions. With only 68M parameters, CT-VAM achieves LIBERO success rates competitive with substantially larger VLA models, while reducing inference latency. Together with flow-consistent inpainting for asynchronous chunk execution, CT-VAM supports high-frequency control and demonstrates robust realworld deployment on resource-constrained robotic platforms.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：
  1. **意图‑执行分离**：将原始语言通过接地（grounding）过程转化为紧凑的任务条件（如 one‑hot 任务标识），低层策略仅根据该条件、视觉观察和本体状态生成动作块，不再重复处理语言。
  2. **受小脑‑丘脑功能类比启发**：高层处理（大脑）负责语义推理，低层模块（小脑‑丘脑）负责快速感觉运动执行；CT‑VAM 中的 VAM 模块对应小脑，TARS 路由模块对应丘脑。

- **关键技术细节**：
  - **输入与表示**：双视角 RGB 图像（第三视角 + 手腕视角）通过冻结的 DINOv3 骨干编码，生成层级的密集视觉记忆；本体状态通过 MLP 编码；任务标识通过可学习嵌入矩阵得到任务 token。
  - **TARS（丘脑动作路由流）**：流分离的条件注意力解码器。动作 token 作为查询，分别对四个独立流（自流、动作查询流、密集视觉‑本体流、任务流）执行注意力，每个流独立归一化后通过可学习门控聚合，防止密集视觉 token 淹没任务等紧凑条件。
    - 关键公式：每层中，动作 token 分别与自流（Xℓ）、动作查询（Eaq）、密集流（[Dℓ; S]）、任务流（{r}）计算注意力，输出经门控（γ）加权求和。
  - **动作查询初始化**：从 DINOv3 的寄存 token（register tokens）投影并扩展至 32 个，可学习，作为动作精化的隐性槽。
  - **动作生成框架**：基于 rectified‑flow 的流匹配，预测噪声→动作的线性速度场。训练时构造插值状态并回归目标速度（MSE 损失）；推理时从均匀噪声通过 8 步积分生成动作块。
  - **FCI（流一致修补）**：实现异步块执行。下一块的开始部分（重叠区域）按训练相同的噪声‑数据插值路径约束，保证边界连续且解码器输入分布内；计算开销极小。

- **算法流程**（文字描述）：
  1. 编码：双视角图像经 DINOv3 得到层级 token，通过手腕‑视角交叉注意力融合，构成密集空间记忆 Dℓ；本体状态编码为 S；任务 token 嵌入为 r。
  2. 初始化动作 token：将噪声动作块展平后通过动作编码器与可学习位置编码得到 X0。
  3. TARS 解码：对每一层，动作 token 分别与自流、动作查询流、密集流、任务流做注意力（Q 来自动作 token，K/V 来自各流），各自归一化后经门控相加，再经时间条件 FFN 更新，得到下一层 token。
  4. 输出速度：最后一层 token 通过线性投影预测速度场。
  5. 推理时通过 ODE 积分（8 步）生成干净动作块；启用 FCI 时，当前块执行到剩余 Kov 步时异步启动下一块推理，重叠区域强制沿着流一致路径。

## 6. 论文的主要结论与发现

1. **语言分离可行**：一旦任务意图接地为紧凑表示，低层策略可丢弃原始语言，仅依赖视觉‑本体与任务条件实现高效执行。
2. **CT‑VAM 在 LIBERO 上达到与大型 VLA 模型相当甚至更好的成功率（82.1% avg）**，但模型参数仅 68M，推理延迟也显著更低。
3. **TARS 优于简单缩放**：在相同流匹配框架下，CT‑VAM 的性能增益来自流分离注意力和动作查询设计，而非单纯增大模型。

5. **可在资源受限平台部署**：在 Jetson Orin NX 上经 TensorRT 优化后，CT‑VAM 成功运行并保持良好性能。
