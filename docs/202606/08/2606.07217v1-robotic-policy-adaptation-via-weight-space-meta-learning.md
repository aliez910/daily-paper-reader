## 摘要
视觉-语言-动作(VLA)模型正成为机器人操作的一种有前景的范式，能够从大规模演示和动作标签数据中训练通用策略。然而，将这些模型适应新任务通常仍需要特定于任务的演示、动作标注和额外的微调，使得部署成本高昂且难以扩展。我们提出WIZARD，一种权重空间元学习框架，通过为冻结的VLA策略生成特定于任务的LoRA参数，从而避免任务特定的微调。仅凭一条语言指令和一段简短的演示视频，WIZARD单次前向传播就能预测相应的自适应权重，无需目标任务的动作标签或测试时优化。在元训练期间，WIZARD学习将任务证据直接映射到专家级的LoRA更新，在权重空间中捕捉任务之间的关系。在LIBERO上的实验表明，WIZARD在未见过的数据集集合上性能提升高达约2倍，在未见过的任务上提升高达约14倍。在Franka Emika Panda上，WIZARD始终优于经过真实域适应训练的基线，表明生成的适配器能够实现超越模拟的任务级专业化。

## Abstract
Vision-Language-Action (VLA) models are emerging as a promising paradigm for robotic manipulation, enabling general-purpose policies trained from large corpora of demonstrations and action labels. However, adapting these models to new tasks still typically requires task-specific demonstrations, action annotations, and additional fine-tuning, making deployment costly and difficult to scale.   We propose WIZARD, a weight-space meta-learning framework that sidesteps task-specific fine-tuning by generating task-specific LoRA parameters for a frozen VLA policy. Given only a language instruction and a short demonstration video, WIZARD predicts the corresponding adaptation weights in a single forward pass, without target-task action labels or test-time optimization. During meta-training, WIZARD learns to map task evidence directly to expert LoRA updates, capturing relationships between tasks in weight space.   Experiments on LIBERO show that WIZARD improves performance by up to ~2x on unseen dataset collections and up to ~14x on unseen tasks. On a Franka Emika Panda, WIZARD consistently improves over a real-domain adapted baseline, showing that generated adapters provide task-level specialization beyond simulation.

# 通过权重空间元学习的机器人策略适应（WIZARD）总结

## 1. 核心问题与研究背景

- **问题**：当前视觉‑语言‑动作（VLA）模型在机器人操作中展现出通用能力，但将其适应新任务时，通常需要收集大量任务特定的演示数据、动作标注，并进行额外的微调（如 LoRA）。这导致部署成本高昂、难以扩展，尤其是任务数量增长时，每个新任务都需要独立的优化及存储。
- **动机**：现有解决方案（如多任务微调、LoRA 微调、检索式适配）仍然依赖动作监督或测试时优化。本文提出一种**零样本、无需微调**的适应方式，仅依靠语言指令及一段简短视频，直接生成适应策略的参数更新。

## 2. 方法论

- **核心思想**：将机器人策略适应视为**权重空间中的参数推断问题**。通过元学习训练一个元网络（meta‑network），将任务证据（语言 + 视频）直接映射为冻结 VLA 策略的 LoRA 适配参数，跳过微调过程。
- **关键技术细节**：
  - **元数据集构建**：为每个任务训练一个专家 LoRA 适配器（在 π0.5 模型上通过两阶段 LoRA 微调获得），同时利用 VLA 编码器提取任务嵌入（多段演示的平均）。构成 pair（嵌入，专家权重更新）。
  - **多模态权重结构**：将 LoRA 更新按视觉、语言、动作三个模块组织成结构化张量，保留架构边界，便于联合建模。
  - **尺度感知参数生成**：元网络同时预测每层的归一化更新及其均值和标准差，并在输出时进行重新缩放，保证连续控制中的参数尺度稳定性。
  - **对齐导向监督**：损失函数包含三项：MSE 重建损失、尺度损失（L2）、余弦方向一致性损失。三者共同保证生成更新在权向量空间与专家更新方向一致且量级匹配。
- **算法流程**（文字说明）：
  1. 对于每个元任务，采样少量演示 → 用 VLA 编码器提取嵌入 → 聚合为任务表示。
  2. 元网络（3D 卷积架构）以该表示为输入，输出预测的 LoRA 参数及每层尺度。
  3. 预测参数经尺度归一化后注入冻结的 VLA 模型，形成适应后的策略。
  4. 元训练时优化三项损失；测试时只进行一次前向传播。

## 3. 实验设计

- **基准与数据集**：
  - **LIBERO 套件**（仿真环境，MuJoCo）：包含四个子集——Spatial（空间布局）、Object（物体辨识）、Goal（目标导向）、10（长程组合任务）。每个子集含 10 个任务，每个任务 50 个固定初始状态。
  - **真实机器人**：Franka Emika Panda（7‑DoF），配备三台 Intel RealSense 相机（一手腕 + 两外部），执行 5 个真实操作任务（pick banana/apple/marker/cup，以及 apple→cup 复合任务）。
- **对比方法**：
  - **专家**：每个任务独立微调的 π0.5 LoRA 模型（上界）。
  - **最近邻（NN）**：从训练集中选择最接近的适配器。
  - **MT‑VLA**：在元训练集上通过监督微调 (OpenVLA‑OFT 或 π0.5) 得到的多任务策略。

- **零样本适应能力**：WIZARD 在 LIBERO‑Spatial 上达到 0.40 平均成功率，是 MT‑VLA (0.19) 的两倍多；在 LIBERO‑Goal 上 0.22 vs 0.14，在 LIBERO‑Object 上 0.03 vs 0.01，大幅优于无需微调的基线。NN 和直接部署 π0.5（≈0%）均远低于 WIZARD。

- **结构化处理多模态参数**：将 LoRA 更新按照视觉、语言、动作模块组织成结构化张量，引入尺度预测与余弦对齐损失，保证了参数生成的稳健性。
