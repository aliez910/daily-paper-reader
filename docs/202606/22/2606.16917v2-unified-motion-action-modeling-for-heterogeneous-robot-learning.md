## 摘要
我们提出了统一运动-动作（UMA）模型，该方法利用3D物体运动轨迹作为共享接口，连接视觉运动控制与动力学建模。UMA将物体运动与机器人动作视为在掩码生成目标下共同演化的变量，其中掩码模式既决定了预训练期间的监督机制，也决定了部署时的推理模式。通过利用事后重新标注的运动上下文以及将任务意图与场景几何解耦的对比目标，UMA实现了跨异构数据源的多任务预训练，且无需人工标注的任务指令。在部署时，相同的预训练参数支持运动条件视觉运动控制、基于运动的动力学建模以及从少量演示中学习任务适应。通过在机器人演示、人类视频和模拟数据的混合数据上进行预训练，UMA在每种推理模式下均始终优于专门设计的现有最强基线。

## Abstract
We present Unified Motion-Action (UMA) Model, an approach that uses 3D object motion trajectories as a shared interface to bridge visuomotor control and dynamics modeling. UMA treats object motion and robot actions as co-evolving variables under a masked generative objective, in which the mask pattern determines both the supervision regime during pretraining and the inference mode at deployment. Using hindsight-relabeled motion contexts and a contrastive objective that disentangles task intent from scene geometry, UMA enables multi-task pretraining across heterogeneous data sources without requiring manually annotated task instructions. At deployment, the same pretrained parameters support motion-conditioned visuomotor control, motion-based dynamics modeling, and task adaptation from few-shot demonstrations. Pretrained on a mixture of robot demonstrations, human videos, and simulated data, UMA consistently outperforms state-of-the-art baselines specialized for each inference mode.

## 1. 论文的核心问题与整体含义（研究动机和背景）

机器人学习面临异构数据源难以统一建模的问题。现有方法通常专用于单一推理模式：运动条件策略需要配对动作标签，无法利用无动作视频；世界模型预测运动但需要独立控制器，无法直接吸收动作标签。不同数据源（人类视频、真实机器人数据、模拟数据）格式差异大，缺乏共享表示。因此，亟需一种统一框架，既能从无监督视频学习运动动力学，又能利用动作标签进行控制，同时支持多种推理模式（控制、预测、适应），且无需手动标注任务指令。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 以**3D物体运动轨迹**（物体表面关键点的3D位置序列）作为共享接口，将物体运动与机器人动作视为**共同演化的变量**，在**掩码生成目标**下联合建模。
- 通过**事后重新标注的运动上下文**和**对比学习**将任务意图与场景几何解耦，实现无需手工标注的多任务预训练。

### 关键技术细节
- **表示**：从RGB视频提取稠密3D关键点轨迹（使用TAPIP3D等工具），与动作同处3D相机坐标系。
- **掩码扩散变压器（Masked DiT）**：基于DiT的联合去噪模型，对目标变量（带掩码的运动/动作令牌）施加去噪，对条件变量（观测、任务令牌）保持清洁。采用空间（同时间步运动-动作耦合）、时间（同关键点跨时间步）、上下文（目标到观测）分解注意力。
- **任务编码器**：从参考运动（部分稀疏方式点）和初始观测提取**任务潜在表示c**。使用PointNet++处理点云与图像特征，对时间归一化（0~1）实现时间步不变性。
- **对比学习（SimCLR风格）**：对（初始观测，参考运动）对施加关键点子采样、时间子采样、随机SE(3)变换，使c捕获与场景绝对位姿无关的抽象意图。
- **公式**：  
  `p_θ(x^{Px}, a^{Pa} | o_t, c, x^{Gx}, a^{Ga})`，通过掩码模式（P，G）确定预测/条件。  
  损失函数：`L = λ_x Lx + λ_a La + λ_c Lc`，其中Lx、La为流匹配损失，Lc为对比损失。  
- **训练流程**：随机掩码运动/动作令牌，联合预测；无动作数据仅贡献Lx和Lc。
- **推理模式**：
  - **运动条件控制**：给定目标参考运动，预测未来动作，闭式执行并重新规划。
  - **动力学预测**：给定候选动作序列，预测物体运动，用于模型预测控制或诊断。
  - **软提示调优**：将c视为自由优化变量（固定其他参数），利用少量演示优化c，实现场景迁移。

## 6. 论文的主要结论与发现

- **统一模型超越专用基线**：零样本下，UMA在插入、清扫、折叠上比最強基线（COIL或UVA）高出20-25个百分点。
- **联合建模有益**：动作-运动联合预测优于仅动作预测或独立解码头（模拟消融提升约20%）。
- **数据互补**：模拟数据驱动运动-动作耦合（无模拟时MSE增加5倍），人类视频提供任务多样性（提升折叠5%成功率）。
- **对比学习关键**：去掉对比损失后，少样本适应成功率下降30-60个百分点。
- **动力学预测改善**：UMA的MSE（0.042）低于PointWorld（0.054），即使后者是专用模型。
- **结构化注意力优于稠密注意力**：在相同预算下，结构化注意力表现更佳。

## 7. 优点

- **统一框架**：一种预训练模型支持三种推理模式（控制、预测、适应），且无需重新训练。
- **异构融合**：通过物体运动接口同时利用无动作视频、带动作机器人数据、模拟数据，无需标注。
- **鲁棒的任务表示**：对比学习产生场景位姿不变的任务潜在表示，支持跨场景迁移和少样本适应。
- **高效的适应**：软提示调优仅优化低维任务令牌，无需微调大量参数，比全参数微调更轻量。
- **可解释性**：可通过预测的运动与任务对比诊断行为意图。

## 8. 不足与局限

- **实施例依赖**：在新机器人上需要配套动作头，仍需相容的机器人数据。
- **相机坐标依赖**：预测位于相机坐标系，部署需要标定相机-机器人基底外参。
- **仅用单步观测**：未利用多步历史观测和稠密点对应，这些天然适合该掩码框架。
- **评估范围有限**：仅在三个桌面任务上评估，未涉及非刚性交互或多机器人场景，泛化性验证不够充分。
- **计算成本**：模型参数较大（最高600M），训练需8块A100 GPU，资源门槛较高。
- **失败分析显示执行误差为主**（82%），说明运动-动作联合建模的精度仍需提升，尤其在接触任务中。
