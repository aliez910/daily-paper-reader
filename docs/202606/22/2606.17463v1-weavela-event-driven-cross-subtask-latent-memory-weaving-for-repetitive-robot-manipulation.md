## 摘要
视觉-语言-动作(VLA)策略已在单步操作中取得了显著成果，但在每一步骤依赖于刚完成的任务时就变得脆弱。核心问题是结构性的：短窗口VLA缺少一个跨子任务边界路由信息的显式通道，而现有的记忆增强变体要么在每一帧写入，从演示时间阶段检索，或者在子目标事件时触发，但没有执行显式的子任务到子任务的移交到动作专家。我们将子目标完成事件识别为跨子任务记忆移交的自然时间单位，并提出WeaveLA（为视觉-语言-动作策略编织隐式记忆），一个跨子任务记忆接口，在冻结的VLA主干之上，通过查询驱动的注意力池化将每个完成的段压缩成潜在令牌，并将它们直接路由到下一个子任务的动作生成路径中。这种事件触发、动作侧的设计保留了基础策略的短窗口接口，同时添加了一个轻量级的跨子任务通道。通过在RoboMME上使用$π_{0.5}$骨干网络进行分层评估，WeaveLA的增益恰好落在需要通道的地方：在最难的重复切片（SwingXtimes, $N{=}3$）上，成功率从$0\%$上升到$47.8\%$，而单次执行回合保持不变。逐回合配对分析证实，增益仅限于那些因果结构需要跨子任务信息的任务。

## Abstract
Vision-Language-Action (VLA) policies have achieved remarkable single-step manipulation, yet they remain brittle precisely where each stage depends on what was just completed. The core issue is structural: short-window VLAs lack an explicit channel for rouxting information across sub-task boundaries, and existing memory-augmented variants either write at every frame, retrieve from demonstration-time stages, or fire at sub-goal events without performing an explicit sub-task-to-sub-task hand-off into the action expert. We identify the sub-goal completion event as the natural temporal unit for cross-subtask memory hand-off, and present WeaveLA (Weave Latent memory for Vision-Language-Action policies), a cross-subtask memory interface that, on top of a frozen VLA backbone, compresses each completed segment into latent tokens via query-driven attention pooling and routes them directly into the action-generation path of the next sub-task. This event-triggered, action-side design preserves the base policy's short-window interface while adding a lightweight cross-subtask channel. Through stratified evaluation on RoboMME with a $π_{0.5}$ backbone, WeaveLA's gains land exactly where the channel is needed: on the hardest repetition slice (SwingXtimes, $N{=}3$), success rises from $0\%$ to $47.8\%$, while single-execution episodes remain unchanged. Per-episode paired analysis confirms the gains are confined to tasks whose causal structure requires cross-subtask information.

- **研究动机**：现有视觉-语言-动作（VLA）策略在单步操作中表现出色，但在**重复性操作**（如装配、包装、质检中反复执行同一基本动作）中表现脆弱。其原因在于VLA通常使用短时的观测窗口，缺乏一个显式通道来跨子任务边界传递信息（例如已完成多少次重复），导致策略在子任务衔接时丢失关键上下文。
- **整体含义**：该问题本质上是**时间依赖的结构性缺失**。子任务的完成结果必须被压缩并传递给下一子任务，而现有方法（逐帧写入、检索示范阶段、或在子目标处触发但未传递到动作专家）均未有效解决这一“跨子任务移交”。本文提出WeaveLA，通过**事件驱动、动作侧注入**的轻量记忆接口，填补了这一空白，显著提升了重复操作的成功率。

## 2. 方法论

- **核心思想**：在冻结的VLA骨干上，增加一个“记忆编织器”（Memory Weaver），该编织器**仅在子目标完成事件触发时**，通过**查询驱动的注意力池化**将刚完成的子任务段压缩为一组固定大小的潜在令牌，并**直接注入到下一子任务的动作专家**，而不改动语言提示或视觉上下文。
- **关键技术细节**：
  - **事件触发**：子目标完成时触发写入，而非每帧写入；事件可由仿真器提供（oracle）或由模型内部检测（latent-shift）。

## 6. 主要结论与发现

- WeaveLA在**需要跨子任务通道的任务**上带来显著提升：SwingXtimes N=3从0%提升至47.8%，StopCube从8%提升至22%（6任务规模）；PickXtimes N≥2也有2-3倍相对提升。
- **单次执行（N=1）完全不变**，说明增益来自记忆机制而非通用能力提升。
- 事件触发设计优于逐帧写入或示范检索，其轻量化（1.4%参数）和定位准确性得到验证。
- 触发源可脱离oracle，用对比学习训练的latent-shift替代，恢复大部分性能（潜在偏移在StopCube上仍偏弱）。
- 注意力池化提取器优于Q-Former，后者导致高超时率（56.3%）。
- 瓶颈宽度8 tokens足够，过小（4）性能崩溃，过大（16）无增益。

## 7. 优点

- **机制轻盈高效**：仅增加1.4%参数，事件触发而非逐帧写，推理开销可忽略。
- **设计原则清晰**：将子目标完成作为自然时间单位，动作侧注入避免信息被VLM稀释，保留原VLA短窗口接口。
- **实验证据多维度且定位精准**：通过重复分层、难度分层、逐回合配对等分析，严格证明增益来源。对比条件严格控制（共享训练数据、优化器等）。
- **触发源可迁移**：提出latent-shift方法，使机制可脱离仿真oracle，实用性更强。
- **与现有方法正交**：指出可与FrameSamp等密度缓冲机制组合，预留结合空间。

## 8. 不足与局限

- **StopCube上的符号计数缺陷**：latent-shift触发在此任务上仅2%（vs. oracle 12%），说明对于纯符号计数任务，需要更专门的边界检测器。
- **Q-Former提取器不稳定**：导致大部分回合超时，训练稳定性较差，尚未深入分析原因。
- **未与FrameSamp等最强者相结合**：论文指出正交性但未实际实现与验证，可能错过最优性能。
- **子目标边界标注依赖**：虽然latent-shift可缓解，但训练阶段仍需要子目标文本描述，增加了标注成本。
- **真实实验规模小**：仅一个任务、单臂、20回合/条件，泛化性有待更大规模硬件验证。
- **计算资源需求较高**：191 GPU小时训练时间对一般研究团队可能较为昂贵，但相对骨干模型本身已算轻量。
- **未在更复杂的长期任务上测试**：论文聚焦重复操作，但很长的操纵序列（＞4子任务）未覆盖，Long-VLA场景下的效果未知。
