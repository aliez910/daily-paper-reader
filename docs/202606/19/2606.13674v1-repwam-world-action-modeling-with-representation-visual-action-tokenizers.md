## 摘要
本文提出了RepWAM，一种以表征为中心的世界动作模型（WAM），它建立在表征视觉-动作分词器之上。现有的WAM通常从预训练的视频生成模型中继承面向重建的视频分词器。尽管这些分词器保持了视觉保真度，但仅靠像素重建为学习连接未来预测与机器人控制的指令遵循动态提供的指导有限。为了解决这个问题，我们探索了一个语义视觉-动作潜在空间，用于以表征为中心的世界动作建模。具体来说，我们训练了一个表征视觉-动作分词器，将视觉输入映射为对齐的视觉和潜在动作令牌。然后，我们在语言指令下预训练我们的WAM，以联合建模未来视觉状态和连接它们的潜在动作，之后适应真实机器人轨迹以进行闭环操作。在真实世界操作任务和模拟基准上的实验表明，RepWAM在多种操作设置中均表现出色，而消融实验则突显了语义视觉-动作分词化相比面向重建的替代方案的价值。这些结果确立了表征视觉-动作分词化作为世界动作模型的一个有前景的基础，以及向通用机器人策略迈进的一步。代码和权重将发布在 https://github.com/wdrink/RepWAM。

## Abstract
This work presents RepWAM, a representation-centric world action model (WAM) built on representation visual-action tokenizers. Existing WAMs typically inherit reconstruction-oriented video tokenizers from pretrained video generation models. Although these tokenizers preserve visual fidelity, pixel reconstruction alone provides limited guidance for learning instruction-following dynamics that connect future prediction with robot control. To address this, we explore a semantic visual-action latent space for representation-centric world action modeling. Specifically, we train a representation visual-action tokenizer that maps visual inputs into aligned visual and latent action tokens. We then pretrain our WAM to jointly model future visual states and the latent actions that connect them under language instructions, followed by adaptation to real robot trajectories for closed-loop manipulation. Experiments on real-world manipulation tasks and simulation benchmarks show that RepWAM delivers strong performance across diverse manipulation settings, while ablations highlight the value of semantic visual-action tokenization over reconstruction-oriented alternatives. These results establish representation visual-action tokenization as a promising foundation for world action models and a step toward generalist robot policies. Code and weights will be available at https://github.com/wdrink/RepWAM.

## 2. 方法论：RepWAM

- **核心思想**：建立一个语义化的视觉-动作潜空间，使视觉状态和引起状态变化的潜在动作在同一空间内联合表示，在此空间上预训练世界动作模型，再适应到具体机器人控制。

### 关键技术细节

#### (1) 表征视觉-动作分词器（RepViTok）
- **视觉分词**：使用视觉Transformer（ViT）自编码器，输入视频帧（第一帧拆分为 16×16 块，后续帧拆分为 4×16×16 管状块），编码器产生视频潜变量 $z$。训练时除重建损失（L1、感知、对抗损失）外，引入特征对齐损失：
  \[
  \mathcal{L}_{\text{align}} = \left\| \operatorname{avg}(W_{\text{align}} z) - \operatorname{avg}(G(o)) \right\|_2^2
  \]
  其中 $G$ 是冻结的视觉基础模型（Perception Encoder），通过该损失使潜变量向语义特征空间靠拢。
- **潜在动作分词**：冻结视觉分词器，训练由逆动力学模型（IDM）$q_\phi$ 和前向动力学模型（FDM）$f_\psi$ 组成的潜在动作分词器。对相邻潜帧 $z_t, z_{t+1}$，IDM 压缩出紧凑潜动作 $\ell_t$，FDM 将其实现为传输矩阵 $K_t$ 和残差 $\delta_t$，重建下一帧潜变量：
  \[
  \ell_t = q_\phi(z_t, z_{t+1}), \quad K_t, \delta_t = f_\psi(z_t, \ell_t), \quad \hat{z}_{t+1} = K_t z_t + \delta_t
  \]
  训练采用前向预测损失和反向一致性损失。

#### (2) 因果世界动作模型
- **模型结构**：因果扩散 Transformer，将语言指令嵌入为条件 token，将视觉潜变量和潜在动作 token 按时间窗口分组为块（chunk），采用块因果掩码，图文混合注意力，视觉与动作模态共享注意力权重但使用独立的前馈网络。
- **训练目标**：流匹配（flow matching）。对每个块采样高斯噪声，构造线性插值路径，网络回归速度向量，Loss 分为视觉部分和动作部分（动作权重 $\lambda_a=1$）：
  \[
  \mathcal{L}_{\text{FM}} = \mathbb{E}\left[ \|F^v_\theta - \dot{x}^v_\alpha\|^2 + \lambda_a \|F^a_\theta - \dot{x}^a_\alpha\|^2 \right]
  \]
- **训练流程**：先在大型机器人数据集（AgiBot）上预训练 WAM（约 100G token，潜在动作由 RepViTok 生成），再用混合真实机器人语料（AgiBot, RoboMIND, RoboCOIN, InternA1；约 300G token）进行体现适应，最后针对具体任务微调。

## 6. 主要结论与发现

1. **表征视觉-动作分词（RepViTok）显著优于重建导向分词**：在生成质量（gFVD 降低 9.5%–13.2%）、开放环动作精度（OLS 提升）和闭环成功率上均表现更好。
2. **两阶段训练（先预训练潜在动作再适应真实动作）优于无潜在动作或联合预测**：两阶段在生成质量和下游成功率上全面领先，表明潜在动作作为中间表示的有效性。
3. **RepWAM 在真实和模拟操作中均具竞争力**：
   - 真实世界：RepWAM-5B 在三项任务上均达到或超越 Lingbot-VA，特别是在长程和精细任务上提升明显。
   - 仿真 RoboTwin 2.0：虽略低于使用预训练视频生成的 Lingbot-VA，但超过 Motus 和 π0.5。
4. **语义对齐降低了视频分类器自由引导（CFG）的依赖**：RepWAM 在 CFG=1.0（无额外引导）时表现最佳，而 Lingbot-VA 需要 CFG>1，说明语义空间本身已较好对齐语言指令。
5. **潜在动作可在机器人动作空间高效转移**：RepViTok 学习的潜在动作被逆动力学模型解码时损失更低，可视化显示聚焦于操作相关变化。

## 7. 方法亮点

- **语义对齐视觉分词**：通过蒸馏冻结基础模型，使潜空间富含语义信息，无需大量标注。
- **可解释的潜在动作**：基于传输矩阵和残差的分解隐含了光流式移动，动作 token 维数很小（dℓ=4），且与视觉 token 共享同一空间，便于联合建模。
- **双阶段因果扩散 WAM**：将视觉和动作建模统一在流匹配框架下，块因果掩码支持视频流自回归生成，且训练后动作分支可直接用于推理。
- **灵活适应**：从大规模预训练到体现适应再到任务微调，流程清晰，可扩展。

## 8. 不足与局限

- **从头训练与预训练基线的差距**：在 RoboTwin 2.0 上表现逊于继承大规模视频生成预训练的 Lingbot-VA，说明从零训练的 WAM 在数据量和泛化上仍有不足，需更大规模预训练或初始化。
- **真实世界验证有限**：仅测试了三种任务、单一机器人平台（Franka dual-arm），未覆盖更多场景或不同形态机器人，通用性待验证。
- **潜在动作的可迁移性未充分跨体现测试**：虽理论上跨体现，但实验仅在同一个机器人设置验证，未验证迁移到不同硬件或操作空间。
- **未涉及安全性与鲁棒性分析**：未讨论失败模式、碰撞避免、错误恢复等关键安全方面。
- **计算资源需求大**：1.3B 和 5B 模型需大量 GPU（64/128 H20），且预训练数据量大（100G tokens+300G tokens），论文未提供推理效率对比。
