## 摘要
自主运行的机器人可能需要基于不再可见的证据做出决策。我们研究延迟证据任务，其中早期提示在后续决策点之前消失，因此视觉上相似的观察可能需要不同的行动。在这些设置中，当前观察不足以作为控制状态。我们提出了轨迹路由因果证据（TRACE），一种用于视觉运动模仿策略的记忆框架。TRACE将任务相关的视觉和机器人状态证据（如物体身份、目标选择或路径依赖状态）存储在固定大小的潜在记忆中，该记忆在长时间片段内保持有界。TRACE不使用原始时间或手动提供的任务标签来索引记忆，而是使用路径签名：执行机器人状态轨迹的紧凑、顺序敏感特征。这些签名不存储视觉提示本身；相反，它们提供轨迹条件键，用于写入和检索在提示可见时存储的证据。当机器人后续遇到模糊观察时，策略基于TRACE记忆恢复缺失的上下文并选择正确的分支。TRACE通过轻量级适配器附加到策略上，而不改变策略主干、动作头或模仿目标。在具有视觉模糊分支点的真实世界长周期操作任务中，TRACE在分支选择和任务成功率上优于其他基线方法，包括短历史和循环记忆。项目页面：https://jeong-zju.github.io/trace

## Abstract
Robots under autonomous operation may require decisions based on evidence that is no longer visible. We study delayed-evidence tasks, where an early cue disappears before a later decision point, so visually similar observations can require different actions. In these settings, the current observation is not a sufficient state for control. We introduce TRAjectory-routed Causal Evidence (TRACE), a memory framework for visuomotor imitation policies. TRACE stores task-relevant visual and robot-state evidence, such as object identity, target choice, or route-dependent state, in a fixed-size latent memory that remains bounded over long episodes. Instead of indexing memory by raw time or manually provided task labels, TRACE uses path signatures: compact, order-sensitive features of the executed robot-state trajectory. These signatures do not store the visual cue itself; rather, they provide trajectory-conditioned keys for writing and retrieving the evidence stored when the cue was visible. When the robot later reaches an ambiguous observation, the policy conditions on TRACE memory to recover the missing context and choose the correct branch. TRACE attaches through lightweight adapters to policies, without changing the policy backbone, action head, or imitation objective. Across real-world long-horizon manipulation tasks with visually ambiguous branch points, TRACE improves branch selection and task success over alternative baselines, including short-history and recurrent memory. ## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 TRAjectory-routed Causal Evidence (TRACE)，一种固定大小的潜在记忆框架。它将任务相关的视觉和机器人状态证据存储于多个潜在槽中，并使用**路径签名（path signatures）** 作为轨迹条件键来索引记忆的写入和读取，而不是用原始时间戳或任务标签。
- **关键技术细节**：
  - **路径签名**：对执行过的机器人状态轨迹的紧凑、顺序敏感的摘要。深度 \(p\) 的路径签名包含从1到p重迭代积分的张量，对轨迹展开顺序敏感，且对时间重参数化不变。TRACE流式维护签名 \(\xi_t\) 和增量 \(\delta_t\)，作为地址特征。
  - **签名路由的槽记忆**：通过轨迹键 \(q_t\) 计算路由权重 \(\omega_{t,k}\)，确定哪些槽被当前证据写入。写入采用门控更新，读取使用注意力机制从槽中提取上下文。
  - **轻量适配器**：TRACE 通过适配器将记忆状态传递给下游策略，不改变策略主干、动作头或模仿损失。对动作块回归策略采用记忆 token 拼接；对扩散策略采用全局条件向量。
  - **辅助损失**：为防槽坍塌、写入弥散和读写不匹配，加入平衡、熵和一致性辅助损失。
  - **训练和推理一致性**：训练时使用固定预算的历史因果扫描，推理时每次更新同一记忆状态。不使用未来信息或分支标签。

## 7. 优点

- **模块化设计**：TRACE 通过轻量适配器即可附加到不同策略家族，不改动策略主干、动作头、模仿损失，容易集成。
- **固定预算**：使用固定数量潜在槽，不随运行时间增长，适合长期部署。
- **利用轨迹签名**：路径签名提供顺序敏感、时间重参数化不变的紧凑轨迹描述符，适合在线流式更新，无需存储完整历史。
- **多种辅助损失**：平衡、熵、一致性损失有效防止记忆坍塌、写入弥散和读写不匹配，增强训练稳定性。
- **诊断验证机制**：通过可解释的历史变换（如顺序反转）对比，为“使用历史证据”提供了直接证据，增强了可信度。
- **实际有效性**：在多个真实世界任务上取得显著提升，尤其对明确基于记忆的分支任务（Book、Laundry、Medicine）提升巨大。

## 8. 不足与局限

- **签名维度增长**：原始签名维度随机器人状态维度和深度指数增长（p=3 时 5219，p=4 时 88740），在更高维状态或更深签名下，计算和压缩成本可能显著增加，是当前主要局限。
- **缺乏长期覆盖保证**：固定槽数量可能无法避免在极长序列中早期证据被覆盖或模糊；辅助损失提供偏置但无理论保证。
- **任务覆盖有限**：仅在 5 个桌面操作任务上评估，缺乏对移动操作、多机器人协调等更复杂场景的验证。
- **训练算力不透明**：未明确报告训练使用的 GPU 型号、数量、总 GPU 小时，影响复现和公平性比较。
- **状态维度相关假设**：签名计算依赖机器人状态路径，对状态表示和归一化敏感；零掩码等处理方式可能未覆盖所有任务变体。
- **与其他方法比较**：VLA 基线虽微调，但可能未进行最优提示工程或超参数搜索；仅通用任务提示，可能限制了其利用语言线索的能力。
