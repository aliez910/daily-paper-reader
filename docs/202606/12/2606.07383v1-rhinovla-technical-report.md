## 摘要
视觉-语言-动作（VLA）模型在机器人操作中展现出巨大潜力，但在边缘硬件上实时部署仍面临挑战。在这项工作中，我们发现VLM的视觉和上下文令牌是部署延迟的主要来源：对于以GEMM为主的投影算子，在模型维度固定的情况下，计算量随输入令牌数量线性增长。基于这一观察，我们提出了RhinoVLA，这是一个与Huixi R1边缘SoC协同设计的面向部署的VLA模型。RhinoVLA采用令牌高效的Qwen3-VL骨干网络和连续动作专家，在保留预训练多模态能力的同时，减轻VLM侧的令牌和计算负担。为支持跨机器人学习，RhinoVLA进一步引入了一个统一接口，该接口结合了视图注册表、72维物理状态-动作槽位空间和机器人实例LoRA，使得异构机器人观测数据和动作模式能够在共享策略下对齐。在部署方面，RhinoVLA通过硬件感知编译、混合精度执行和并行视觉编码进行了优化。实验表明，在相似参数规模下，RhinoVLA的下游性能可与π0.5相媲美，同时在Huixi R1上实现了11.69 Hz的端到端推理，满足10 Hz实时闭环控制目标。该项目将在https://github.com/HuixiAI/RhinoVLA开源。

## Abstract
Vision-Language-Action (VLA) models have shown strong potential for robotic manipulation, but real-time deployment on edge hardware remains challenging. In this work, we identify VLM visual and context tokens as a major source of deployment latency: for GEMM-dominated projection operators, computation grows linearly with the number of input tokens when model dimensions are fixed. Motivated by this observation, we propose RhinoVLA, a deployment-oriented VLA model co-designed with the Huixi R1 edge SoC. RhinoVLA adopts a token-efficient Qwen3-VL backbone and a continuous Action Expert, reducing the VLM-side token and computation burden while preserving pretrained multimodal capability. To support cross-robot learning, RhinoVLA further introduces a unified interface that combines View Registry, 72D physical state-action slot space, and robotinstance LoRA, allowing heterogeneous robot observations and action schemas to be aligned under a shared policy. On the deployment side, RhinoVLA is optimized through hardware-aware compilation, mixed-precision execution, and parallel visual encoding. Experiments show that RhinoVLA achieves downstream performance comparable to π0.5 at a similar parameter scale, while reaching 11.69 Hz end-to-end inference on Huixi R1, meeting the 10 Hz real-time closedloop control target. The project will be open-sourced at https://github.com/HuixiAI/RhinoVLA.

# 1. 核心问题与整体含义
- **背景**：VLA（视觉-语言-动作）模型在机器人操作中能力突出，但在边缘硬件（如NVIDIA Jetson）上实时部署困难，闭环控制通常要求 ≥10 Hz 的推理频率。
- **核心问题**：VLM骨干中的视觉和上下文token是主要延迟来源，因为MLP投影算子（GEMM）的计算量与输入token数成正比，导致即使模型参数接近，不同token组织方式的VLA推理速度差异很大。
- **整体目标**：通过算法-系统协同设计，构建一个**部署导向的VLA模型（RhinoVLA）**，在保持任务性能的同时在边缘SoC上达到实时推理，并支持跨异构机器人学习。

# 2. 方法论
## 核心思想
- **Token压缩优先**：采用令牌高效的Qwen3‑VL骨干（单图仅64个合并token，对比PaliGemma的256个），从源头降低MLP的GEMM计算量。
- **统一接口解耦异构**：通过视图注册表、72维物理槽位空间和机器人实例LoRA，使不同视觉布局、动作定义的机器人数据能在统一策略下训练。
- **硬件-软件协同优化**：针对Huixi R1 SoC定制编译、混合精度执行和并行编码，将峰值算力转化为实际推理速度。

## 关键技术细节
- **架构**：分为VLM骨干（2.13B Qwen3‑VL）和连续动作专家（0.40B，基于流匹配）。动作专家以Qwen3‑VL最后18层KV缓存、72D状态、状态/动作掩码、噪声动作块和时间步为输入，预测流速度。
- **统一接口**：
  - **视图注册表**：给每张图像加入固定角色-模态标签（如`[head|rgb]`），消解数据集间相机顺序歧义。
  - **72D物理槽位空间**：固定70个槽的物理含义（臂关节D0‑D6、夹爪D14‑D15、手部D16‑D47等），状态和动作共享该定义，并附带二进制掩码指示有效维度。
  - **机器人实例LoRA**：在动作专家每层的FFN中插入低秩适配器，通过实例ID硬路由选择，保留共享结构的同时学习本体特定残差。
- **训练**：
  - 预训练：冻结Qwen3‑VL骨干，同时优化VLM LoRA、共享动作专家和各实例LoRA。损失为**掩蔽流匹配损失**（仅监督掩码标为有效的动作槽），采用幂律平衡采样多数据集。
  - 后训练/迁移：冻结VLM和大部分动作专家，仅更新目标机器人的实例LoRA，实现小样本适配。
- **部署优化**：
  - **硬件感知编译**：算子级（FlashAttention式tiling适配R1的SPM）、图级（融合归一化/线性/残差）、运行时级（细粒度任务调度）。
  - **混合精度**：W8A16（INT8权重+FP16激活），自定义GEMM内核将权重复位、反量化与矩阵乘融合，在R1上达到1.69×加速。
  - **并行视觉编码**：三路图像打包为batch同时编码，延迟从34.52 ms降至24.31 ms。

# 6. 主要结论与发现
- **性能相当**：在LIBERO上RhinoVLA取得90.0%平均成功率，虽低于π0.5，但考虑训练数据差异，可认为在类似参数规模下竞争性。
- **推理达标**：在Huixi R1上达到11.69 Hz，超过10 Hz实时闭环要求（π0.5在Orin上约1.17 Hz）。
- **跨机器人有效**：实例LoRA确实捕捉了本体残差，提升了动作预测精度；在AgiBot G2上unseen设置中超越π0.5（24% vs 18%）。
- **部署优化显著**：编译、混合精度、并行编码分别贡献了加速，最终帧率较基线提升约2×。

# 7. 优点
- **起点新颖**：明确将VLM的token数作为部署瓶颈，从模型架构层面对症下药。
- **系统化方案**：同时解决推理效率与跨本体训练两大难题，统一接口设计精巧（视图注册+72D槽+实例LoRA），避免了为每个机器人设计单独输出头的负担。
- **硬件-软件深度结合**：与R1 SoC的协同优化（FlashAttention-like、W8A16定制GEMM、并行ViT）展示出显著的工程价值。
- **开源性**：项目计划开源，利于社区复现与推进。
