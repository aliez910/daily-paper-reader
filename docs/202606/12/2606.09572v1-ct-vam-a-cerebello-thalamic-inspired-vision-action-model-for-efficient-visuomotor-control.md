## 摘要
视觉-语言-动作模型在机器人操作中展现出巨大潜力，然而原始语言主要用于指定任务意图，而非在高频低层级执行过程中反复处理。受此分离的启发，我们提出了一种小脑-丘脑启发的视觉-动作模型（CT-VAM），用于高效的任务条件化视觉运动控制。CT-VAM作为一个紧凑的局部执行策略，从双视角视觉观察、本体感觉和一个轻量级任务条件中预测动作块，这有可能实现一种实用的云边范式，其中高层语义推理由大型模型处理，而快速闭环控制在本地硬件上运行。为了有效融合异构输入，CT-VAM引入了TARS（丘脑动作路由流），一种流分离的条件注意力解码器，它独立路由动作、视觉和任务流，防止密集的感官令牌淹没紧凑的任务相关条件。仅凭68M参数，CT-VAM在LIBERO上取得了与更大规模VLA模型相当的成功率，同时降低了推理延迟。结合用于异步块执行的流一致性修补，CT-VAM支持高频控制，并在资源受限的机器人平台上展示了鲁棒的现实世界部署能力。

## Abstract
Vision-language-action models have shown strong promise for robot manipulation, yet raw language is primarily needed to specify task intent rather than to be repeatedly processed during high-frequency low-level execution. Motivated by this separation, we propose a cerebello-thalamic-inspired vision-action model (CT-VAM) for efficient task-conditioned visuomotor control. CT-VAM acts as a compact local execution policy that predicts action chunks from dualview visual observations, proprioception, and a lightweight task condition, potentially enabling a practical cloud-edge paradigm in which high-level semantic reasoning can be handled by large models while fast closed-loop control runs on local hardware. To fuse heterogeneous inputs effectively, CT-VAM introduces TARS (Thalamic Action Routing Stream), a stream-separated conditional attention decoder that independently routes action, visual and task streams, preventing dense sensory tokens from overwhelming compact task-relevant conditions. With only 68M parameters, CT-VAM achieves LIBERO success rates competitive with substantially larger VLA models, while reducing inference latency. Together with flow-consistent inpainting for asynchronous chunk execution, CT-VAM supports high-frequency control and demonstrates robust realworld deployment on resource-constrained robotic platforms.

### 2. 论文提出的方法论：核心思想、关键技术细节

CT-VAM是一个紧凑的视觉-动作模型，其核心思想是：**在任务意图（由任务条件向量表示）已确定后，仅依靠视觉观察和本体感觉来生成低层级动作块**，从而避免重复的语言处理。

- **整体架构**：CT-VAM包含三个主要组件：
    1.  **双视角视觉编码器**：使用冻结的DINOv3 ViT骨干网络，处理来自主摄像机和腕部摄像机的图像，并产生层级的视觉记忆，再通过跨视角交叉注意力融合。
    2.  **任务编码器**：将任务ID（如独热向量）嵌入为一个紧凑的任务令牌。
    3.  **动作解码器**：基于Transformer的整流流模型，是整个方法的核心。

- **关键技术1：TARS（Thalamic Action Routing Stream，丘脑动作路由流）** - 流分离的条件注意力解码器
    - **核心思想**：TARS模仿丘脑的信息路由功能，设计了一种流分离的注意力机制，用以处理并融合来自**四个独立流**的信息：
        - **自流（Self Stream）**：当前的动作令牌。
        - **动作查询流（Action-Query Stream）**：一组可学习的动作查询向量，初始化为视觉骨干网络的注册令牌。它们作为动作细化的“槽位”，但其贡献很弱（门控值≈0.02），主要起辅助作用。
        - **密集流（Dense Stream）**：融合后的视觉特征图块和本体感觉特征（令牌数量多）。
        - **任务流（Task Stream）**：紧凑的任务条件向量（仅一个令牌）。
    - **关键创新**：与传统的“共享Softmax”注意力不同，TARS对**每个流独立进行归一化（Stream-wise Normalization）**。这防止了由于视觉等密集流包含更多令牌，而在Softmax竞争中自然占据主导地位，从而保护了紧凑但至关重要的任务流信息不被淹没。
    - **优势**：消除了由于内存流长短不一导致的结构性偏差，增强了模型对不同令牌数量的鲁棒性，并能更清晰地控制不同信息流对最终动作生成的贡献。

- **关键技术2：流一致修补** - 异步块执行策略
    - **问题**：推理延迟仍然是实时控制的瓶颈。传统同步执行中，推理时间会直接叠加到执行时间上。
    - **核心思想**：在机器人执行当前动作块（Chunk）的同时，异步地预测下一个动作块，从而实现计算与执行的重叠，隐藏推理延迟。
    - **FCI方案**：为保证相邻块之间的动作连续性，FCI在生成下一个块时，强制其开头部分（重叠段）与当前块的尾部在**流的轨迹上**保持一致。
        - **约束方式**：不简单地“硬裁剪”最终动作值，而是让重叠段的中间状态始终遵循整流流训练时的线性插值路径，从而保证解码器输入分布的一致性。
        - **效果**：在大部分推理延迟可以被重叠执行窗口（`Kov * Δt`）“隐藏”的条件下，异步方案的实际等效频率非常接近目标控频率。

- **其他细节**：
    - 使用整流流（Rectified Flow）进行动作生成训练。
    - 策略输入为历史帧的双视角图像、本体感觉和任务标签；输出为一个动作块。

### 6. 论文的主要结论与发现

1.  **分离是可行的、高效的**：将语言理解（任务条件化）与视觉动作执行解耦，仅用68M参数的紧凑模型即可在LIBERO上达到与大规模VLA模型（数B参数）相当的性能，推理延迟显著降低。
