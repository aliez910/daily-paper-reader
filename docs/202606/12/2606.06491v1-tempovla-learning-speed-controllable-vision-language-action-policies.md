## 摘要
机器人操作交替出现低风险过渡阶段（需要快速执行）和高风险接触阶段（需要慢速精确运动）。然而，现有的视觉-语言-动作模型（VLA）仅从训练演示中继承单一固定速度。先前通过模型压缩、KV缓存重用或强化学习加速VLA的努力仅将策略从一个固定速度转换为另一个，而几乎未探索减速。我们观察到每个预测动作的幅度已经决定了机器人移动的快慢，这为可控执行速度开辟了直接途径。我们基于这一观察提出了TempoVLA，一个执行速度由显式条件控制的单一VLA。TempoVLA结合了两个耦合组件：（1）数据端的变速度轨迹增强（VSTA），通过合并或拆分动作将演示重新定时到任意目标速度，同时保持其运动语义；（2）模型端的条件机制，将速度输入策略。统计表明，VSTA以可忽略的运动误差达到所请求的速度。仿真和真实世界任务的实验表明，TempoVLA实现了两个方向的灵活速度控制，而VSTA通过更好的数据利用额外提升了默认$1\times$性能。此外，通过与大型多模态模型协作，TempoVLA实现了动态速度控制，在低风险阶段加速并在高风险阶段减速。

## Abstract
Robot manipulation alternates between low-risk transit phases that call for fast execution and high-risk contact stages that demand slow, precise motion. Yet existing Vision-Language-Action models (VLAs) only inherit a single fixed speed from training demonstrations. Prior efforts to accelerate VLAs through model compression, KV-cache reuse, or reinforcement learning only shift the policy from one fixed speed to another, and leave deceleration almost unexplored. We observe that the magnitude of each predicted action already governs how fast the robot moves, opening a direct route to controllable execution speed. We turn this observation into TempoVLA, a single VLA whose execution speed is controlled by an explicit condition. TempoVLA combines two coupled components. (1) A data-side Variable-Speed Trajectory Augmentation (VSTA) that re-times demonstration to any target speed by merging or splitting actions while preserving its motion semantics. (2) A model-side conditioning mechanism that feeds the speed to the policy. Statistics show that VSTA reaches the requested speed with negligible motion error. Experiments in simulation and on real-world tasks demonstrate that TempoVLA achieves flexible speed control in both directions, while VSTA additionally boosts the default $1\times$ performance via better data utilization. Furthermore, by cooperating with a large multimodal model, TempoVLA realizes dynamic speed control, accelerating through low-risk phases and decelerating for high-risk ones.

## 1. 核心问题与整体含义（研究动机和背景）
- 现有视觉-语言-动作模型（VLA）在训练时从演示数据中继承单一固定执行速度，无法适应机器人操作过程中不同阶段对速度的不同需求（低风险过渡阶段需快速、高风险接触阶段需慢速精确）。
- 先前的加速方法（模型压缩、KV缓存复用、强化学习等）仅将策略从一个固定速度切换到另一个，缺乏显式的按需双向速度控制，且几乎未涉及减速。
- **核心问题**：如何在不对基础架构进行重新训练的前提下，赋予单个 VLA 显式的、连续可调的速度控制能力。

## 2. 论文提出的方法论
- **核心思想**：利用已预测动作的幅度天然控制机器人移动速度这一观察，从数据端和模型端协同设计，实现速度可控的 VLA。
- **关键技术细节**：
  - **数据端：变速度轨迹增强（VSTA）**
    - 在线将任意演示重新定时到目标速度 \(s\)，同时保持运动语义。
    - 流程：对演示进行运动一致分割（基于运动模式与方向变化） → 将目标速度写为 \(s = q/p\)（\(q>p\) 加速，\(q<p\) 减速） → 在每个分段内按 \(q\) 帧为一组累积总运动 \(\Delta = \sum_{i=1}^{q} a_i\)，再重新平分为 \(p\) 步（线性插值累积运动） → 支持末端执行器平移、轴角旋转增量，夹爪离散处理 → 在线训练时随机化块起始偏移，避免丢弃观测。
  - **模型端：速度条件机制**
    - 提供三种轻量注入方案：文本前缀（在指令中加入“以 \(s\) 倍速执行”）、动作专家调制（通过 MLP 嵌入 \(s\) 并添加到流匹配时间步嵌入中，通过 adaRMSNorm 调节）、软提示（为各速度锚点学习可调 tokens）。
    - 实验表明文本前缀最简单且效果相当，被选为默认方案。
  - **动态速度控制**：与高级视觉语言模型（VLM，如 GPT-4o）结合，VLM 根据当前场景预测下一阶段的期望速度，TempoVLA 按命令速度执行，实现低风险阶段加速、高风险阶段减速的自动调度。

## 6. 论文的主要结论与发现
- VSTA 能可靠地将演示重定时至目标速度，且运动误差可忽略（小于 \(5\times10^{-8}\)）。
- 速度控制可轻松植入各种 VLA，三种条件机制性能几乎一致（文本前缀最简）。
- VSTA 训练本身作为数据增强，提升了默认 \(1\times\) 成功率（仿真 +8 点，真实 +8 点）。
- TempoVLA 在 \(0.5\times\) 到 \(1.5\times\) 范围内实现紧密的速度命令与实现匹配；超出后性能受限于低级控制器带宽。
- 最佳部署速度略高于演示速度（\(1.25\times\) 或 \(1.5\times\)），可消除演示中节奏冗余。
- 与 VLM 结合动态调度实现更高成功率（96%）和效率，自动完成相位感知调速。

## 7. 优点
- **轻量灵活**：无需额外数据收集、无需重训基础架构，直接适用于现有 VLA。
- **面向实际需求**：同时支持加速与减速，填补了减速研究的空白。
- **数据与模型协同**：VSTA 既是速度控制工具又是数据增强手段，一举两得。
- **条件注入简单有效**：文本前缀无需架构改动，方便部署。
- **动态调度能力强**：与 VLM 结合将执行速度变为高层推理器的控制通道，提升整体性能。
- **实验扎实**：从仿真到真实，多角度消融和压力测试，分析深入。

## 8. 不足与局限
- **高端速度饱和**：加速比超过约 \(1.5\times\) 后实现速度受限于固定低级控制器跟踪能力，策略潜力未充分发挥（论文提到未来可协同调优控制器）。
- **动作空间限制**：VSTA 要求动作空间对加法封闭（如轴角、平移），无法直接用于四元数、旋转矩阵等表示，需额外映射。
- **动态调度延迟**：VLM 同步调用增加墙钟时间，异步并行可优化但未实施。
- **速度参考未归一化**：默认 \(1\times\) 基于演示原始粒度，不同演示/阶段可能存在不一致，未来可归一化以更精确对应命令。
- **泛化性有限**：仅测试 LIBERO 和五个真实任务，更多场景与多样本评估有待验证。
- **缺少与同类型方法的直接对比**（如 RL 微调加速），但速度控制目标本身不同。
- **安全问题未讨论**：极端速度下策略执行的安全性及对控制器的影响未涉及。
