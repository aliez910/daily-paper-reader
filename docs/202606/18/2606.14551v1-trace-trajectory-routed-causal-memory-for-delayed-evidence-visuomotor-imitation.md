## 摘要
自主运行的机器人可能需要基于不再可见的证据做出决策。我们研究“延迟证据”任务，即早期线索在后续决策点之前消失，因此视觉上相似的观测可能需要不同的动作。在这些设置中，当前观测不足以作为控制状态。我们提出轨迹路由因果记忆（TRACE），一种用于视觉运动模仿策略的记忆框架。TRACE将任务相关的视觉和机器人状态证据（如物体身份、目标选择或路由依赖状态）存储在固定大小的潜在记忆中，该记忆在长时间段内保持有界。TRACE不使用原始时间或手动提供的任务标签来索引记忆，而是使用“路径签名”：执行的机器人状态轨迹的紧凑、顺序敏感特征。这些签名不存储视觉线索本身；相反，它们为写入和检索线索可见时存储的证据提供了轨迹条件键。当机器人后来遇到模糊的观测时，策略以TRACE记忆为条件，以恢复缺失的上下文并选择正确的分支。TRACE通过轻量级适配器附加到策略上，而不改变策略主干、动作头或模仿目标。在具有视觉模糊分支点的真实世界长时域操作任务中，TRACE在分支选择和任务成功率上优于替代基线，包括短历史和循环记忆。项目页面：https://jeong-zju.github.io/trace

## Abstract
Robots under autonomous operation may require decisions based on evidence that is no longer visible. We study \emph{delayed-evidence} tasks, where an early cue disappears before a later decision point, so visually similar observations can require different actions. In these settings, the current observation is not a sufficient state for control. We introduce TRAjectory-routed Causal Evidence (TRACE), a memory framework for visuomotor imitation policies. TRACE stores task-relevant visual and robot-state evidence, such as object identity, target choice, or route-dependent state, in a fixed-size latent memory that remains bounded over long episodes. Instead of indexing memory by raw time or manually provided task labels, TRACE uses \emph{path signatures}: compact, order-sensitive features of the executed robot-state trajectory. These signatures do not store the visual cue itself; rather, they provide trajectory-conditioned keys for writing and retrieving the evidence stored when the cue was visible. When the robot later reaches an ambiguous observation, the policy conditions on TRACE memory to recover the missing context and choose the correct branch. TRACE attaches through lightweight adapters to policies, without changing the policy backbone, action head, or imitation objective. Across real-world long-horizon manipulation tasks with visually ambiguous branch points, TRACE improves branch selection and task success over alternative baselines, including short-history and recurrent memory. ## 1. 论文的核心问题与整体含义（研究动机和背景）
- 机器人自主操作中常遇到**“延迟证据”**任务：早期出现的线索（如起始位置、物体身份）在后续决策点之前消失，导致视觉上相似的观测需要执行不同的动作。
- 现有视觉运动模仿策略通常只基于当前观测或短时间窗口，隐式假设当前输入是动作选择的充分统计量。延迟证据任务违反了这一**马尔可夫假设**，因为当前观测无法区分需要不同动作的历史轨迹。
- 因此，需要一种**因果的、固定预算的记忆机制**，能够在线存储早期线索并在后期模糊决策点恢复，从而正确选择分支动作。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 提出**TRACE (TRAjectory-routed Causal Evidence)** 记忆框架，通过机器人执行的**轨迹路径签名**作为条件键，在固定大小的潜在记忆槽中写入和检索视觉与状态证据。
- 将记忆内容（当前观测编码）与记忆存取（轨迹派生的键）分离：内容决定“存什么”，轨迹决定“存在哪/从哪取”。
- 通过轻量适配器附加到现有模仿策略上，不改变策略主干、动作头或模仿损失。

### 关键技术细节
1. **路径签名（Path Signature）**：
   - 对机器人状态轨迹 \(S_t\)（分片线性插值）计算截断深度 \(p\) 的路径签名 \(\xi_t = \text{Sig}^{\leq p}(S_t)\) 和签名增量 \(\delta_t = \xi_t - \xi_{t-1}\)。
   - 签名是紧凑、顺序敏感的轨迹描述符，对单调时间重参数化和常数偏移具有不变性。
   - 通过投影得到 \(g_t = \phi_g(\xi_t)\) 和 \(\Delta g_t = \phi_\Delta(\delta_t)\)，组合为轨迹键 \(q_t = \phi_q([g_t, \Delta g_t])\)。

2. **签名路由槽记忆（Signature-Routed Slot Memory）**：
   - 固定 \(K\) 个槽 \(M_t \in \mathbb{R}^{K \times d}\)，在每个时间步，用轨迹键 \(q_t\) 和路由状态 \(\rho_t\) 计算路由权重 \(\omega_{t,k}\)，决定写入哪些槽。
   - 写操作：将当前证据编码 \(e_t\) 与轨迹键组合为写输入 \(u_t\)，通过门控更新写入选中的槽。
   - 读操作：基于当前证据和路由状态形成查询，从更新后的槽中读出记忆读出 \(z_t^{\text{mem}}\)。
   - 使用辅助损失（平衡损失、熵损失、一致性损失）防止槽坍塌、扩散写入和读写不匹配。

3. **策略适配器**：
   - 对于回归策略（如ACT），将记忆槽和读出投影为记忆token，拼接到策略的注意力记忆中。
   - 对于扩散策略，将记忆槽按读出权重池化，与 \(g_t, \Delta g_t\) 等拼接后通过零初始化MLP变为全局条件向量。
   - 策略的动作表示、动作头和模仿损失完全不变。

1. **TRACE显著提升延迟证据操作**：相比基础策略（ACT 25.50%，Diffusion 25.00%），TRACE达到69.23%（回归）和59.53%（扩散）。

## 7. 优点
- **创新性**：首次将**路径签名**用于机器人记忆的索引，提供对轨迹顺序敏感的确定性键。
- **模块化与轻量**：通过适配器附加，不改变策略主干、动作头或损失，易于集成到不同策略家族。
- **固定预算记忆**：槽数量和维度固定，随任务时长不增长，适合在线长时域执行。
- **严格因果**：训练和部署均只使用当前及过去信息，无未来窥视或任务标签泄漏。
- **充分的真实世界实验**：五项具代表性的延迟证据任务，多种基线和消融，物理滚动评估，诊断验证机制。
- **辅助损失稳定记忆**：三个辅助损失有效防止槽坍塌、扩散写入和读写不匹配。

## 8. 不足与局限
- **签名维度缩放问题**：标准截断签名维度随状态维数和深度指数增长（\(d_s=17, p=3\) 时已达5219维），高维状态或更深签名会导致更大的存储与计算开销。论文未探索更高效的log签名变体。
- **固定预算的长期覆盖**：虽然实验验证了中等时长（最多2×）稳定性，但未提供极长任务（如数百步）的理论保证；多个信息可能被迫共享槽，可能被覆盖或模糊。
- **辅助损失仅经验有效**：坍缩和扩散的抑制依赖算法设计，缺乏严格容量上限或信息保持的保证。
- **状态表示选择**：签名需要设计合适的机器人状态表示（哪些通道被屏蔽），且依赖固定任务先验，通用性仍受人工设定影响。
- **推理开销**：对于快速策略（如ACT），TRACE增加约35.8%的每步延迟；对于慢速扩散策略则仅1.8%，但在实时控制中可能成为瓶颈。
- **未探索经典序列记忆方法**：如NMN、可微分神经计算机等更复杂记忆结构未纳入对比，仅与GRU、LRU等常见变体比较。
